# <a name="classification-of-standard-query-operators-by-manner-of-execution-visual-basic"></a><span data-ttu-id="fee0c-102">Classification des opérateurs de requête standard par mode d’exécution (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="fee0c-102">Classification of Standard Query Operators by Manner of Execution (Visual Basic)</span></span>
<span data-ttu-id="fee0c-103">Les implémentations LINQ to Objects des méthodes d’opérateur de requête standard s’exécutent de deux manières principales : immédiate ou différée.</span><span class="sxs-lookup"><span data-stu-id="fee0c-103">The LINQ to Objects implementations of the standard query operator methods execute in one of two main ways: immediate or deferred.</span></span> <span data-ttu-id="fee0c-104">En outre, les opérateurs de requête qui utilisent l’exécution différée peuvent être divisés en deux catégories : ceux prenant en charge la diffusion en continu et ceux ne la prenant pas en charge.</span><span class="sxs-lookup"><span data-stu-id="fee0c-104">The query operators that use deferred execution can be additionally divided into two categories: streaming and non-streaming.</span></span> <span data-ttu-id="fee0c-105">Le fait de connaître le mode d’exécution des différents opérateurs de requête peut vous aider à comprendre les résultats obtenus à partir d’une requête donnée.</span><span class="sxs-lookup"><span data-stu-id="fee0c-105">If you know how the different query operators execute, it may help you understand the results that you get from a given query.</span></span> <span data-ttu-id="fee0c-106">Ceci est particulièrement vrai si la source de données change ou si vous générez une requête sur une autre requête.</span><span class="sxs-lookup"><span data-stu-id="fee0c-106">This is especially true if the data source is changing or if you are building a query on top of another query.</span></span> <span data-ttu-id="fee0c-107">Cette rubrique classe les opérateurs de requête standard selon leur mode d’exécution.</span><span class="sxs-lookup"><span data-stu-id="fee0c-107">This topic classifies the standard query operators according to their manner of execution.</span></span>  
  
## <a name="manners-of-execution"></a><span data-ttu-id="fee0c-108">Modes d’exécution</span><span class="sxs-lookup"><span data-stu-id="fee0c-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="fee0c-109">Immédiat</span><span class="sxs-lookup"><span data-stu-id="fee0c-109">Immediate</span></span>  
 <span data-ttu-id="fee0c-110">L’exécution immédiate signifie que la source de données est lue et que l’opération est effectuée au point où la requête est déclarée dans le code.</span><span class="sxs-lookup"><span data-stu-id="fee0c-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="fee0c-111">Tous les opérateurs de requête standard qui retournent un résultat unique et non énumérable s’exécutent immédiatement.</span><span class="sxs-lookup"><span data-stu-id="fee0c-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="fee0c-112">Différé</span><span class="sxs-lookup"><span data-stu-id="fee0c-112">Deferred</span></span>  
 <span data-ttu-id="fee0c-113">L’exécution différée signifie que l’opération n’est pas effectuée au point où la requête est déclarée dans le code.</span><span class="sxs-lookup"><span data-stu-id="fee0c-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="fee0c-114">L’opération est effectuée uniquement quand la variable de requête est énumérée, par exemple à l’aide d’une instruction `For Each`.</span><span class="sxs-lookup"><span data-stu-id="fee0c-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="fee0c-115">Cela signifie que les résultats de l’exécution de la requête dépendent du contenu de la source de données au moment de l’exécution de la requête, et non au moment de sa définition.</span><span class="sxs-lookup"><span data-stu-id="fee0c-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="fee0c-116">Si la variable de requête est énumérée plusieurs fois, les résultats peuvent différer chaque fois.</span><span class="sxs-lookup"><span data-stu-id="fee0c-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="fee0c-117">Presque tous les opérateurs de requête standard dont le type de retour est <xref:System.Collections.Generic.IEnumerable%601> ou <xref:System.Linq.IOrderedEnumerable%601> s’exécutent de manière différée.</span><span class="sxs-lookup"><span data-stu-id="fee0c-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="fee0c-118">Les opérateurs de requête qui utilisent l’exécution différée peuvent également être classés comme prenant en charge la diffusion en continu ou non.</span><span class="sxs-lookup"><span data-stu-id="fee0c-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="fee0c-119">Diffusion en continu</span><span class="sxs-lookup"><span data-stu-id="fee0c-119">Streaming</span></span>  
 <span data-ttu-id="fee0c-120">Les opérateurs prenant en charge la diffusion en continu n’ont pas à lire toutes les données sources avant de générer des éléments.</span><span class="sxs-lookup"><span data-stu-id="fee0c-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="fee0c-121">Au moment de l’exécution, un opérateur prenant en charge la diffusion en continu effectue son opération sur chaque élément source à mesure qu’il est lu et génère l’élément si nécessaire.</span><span class="sxs-lookup"><span data-stu-id="fee0c-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="fee0c-122">Ce type d’opérateur continue à lire des éléments source jusqu’à ce qu’un élément de résultat puisse être produit.</span><span class="sxs-lookup"><span data-stu-id="fee0c-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="fee0c-123">Cela signifie que plusieurs éléments source peuvent être lus pour produire un élément de résultat.</span><span class="sxs-lookup"><span data-stu-id="fee0c-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="fee0c-124">Sans diffusion en continu</span><span class="sxs-lookup"><span data-stu-id="fee0c-124">Non-Streaming</span></span>  
 <span data-ttu-id="fee0c-125">Les opérateurs ne prenant pas en charge la diffusion en continu doivent lire toutes les données source avant de générer un élément de résultat.</span><span class="sxs-lookup"><span data-stu-id="fee0c-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="fee0c-126">Les opérations telles que le tri ou le regroupement font partie de cette catégorie.</span><span class="sxs-lookup"><span data-stu-id="fee0c-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="fee0c-127">Au moment de l’exécution, les opérateurs de requête ne prenant pas en charge la diffusion en continu lisent toutes les données source, les placent dans une structure de données, puis effectuent l’opération et génèrent les éléments de résultat.</span><span class="sxs-lookup"><span data-stu-id="fee0c-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
