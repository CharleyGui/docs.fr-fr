# <a name="how-to-hide-a-variable-with-the-same-name-as-your-variable-visual-basic"></a><span data-ttu-id="54ed7-102">Procédure : Masquer une Variable portant le même nom que votre Variable (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="54ed7-102">How to: Hide a Variable with the Same Name as Your Variable (Visual Basic)</span></span>
<span data-ttu-id="54ed7-103">Vous pouvez masquer une variable par *occultation* il, autrement dit, en le redéfinissant avec une variable du même nom.</span><span class="sxs-lookup"><span data-stu-id="54ed7-103">You can hide a variable by *shadowing* it, that is, by redefining it with a variable of the same name.</span></span> <span data-ttu-id="54ed7-104">Vous pouvez occulter la variable que vous souhaitez masquer de deux manières :</span><span class="sxs-lookup"><span data-stu-id="54ed7-104">You can shadow the variable you want to hide in two ways:</span></span>  
  
- <span data-ttu-id="54ed7-105">**Occultation par portée.**</span><span class="sxs-lookup"><span data-stu-id="54ed7-105">**Shadowing Through Scope.**</span></span> <span data-ttu-id="54ed7-106">Vous pouvez occulter via étendue en redéclarant dans une sous-région de la région contenant la variable que vous souhaitez masquer.</span><span class="sxs-lookup"><span data-stu-id="54ed7-106">You can shadow it through scope by redeclaring it inside a subregion of the region containing the variable you want to hide.</span></span>

## <a name="two-ways-to-hide-a-variable"></a><span data-ttu-id="54ed7-109">Deux méthodes pour masquer une Variable</span><span class="sxs-lookup"><span data-stu-id="54ed7-109">Two Ways to Hide a Variable</span></span>  
  
#### <a name="to-hide-a-variable-by-shadowing-it-through-scope"></a><span data-ttu-id="54ed7-110">Pour masquer une variable par elle occultation par portée</span><span class="sxs-lookup"><span data-stu-id="54ed7-110">To hide a variable by shadowing it through scope</span></span>  
  
1. <span data-ttu-id="54ed7-111">Déterminer la région de définition de la variable que vous souhaitez masquer et déterminez une sous-région dans laquelle la redéfinir avec votre variable.</span><span class="sxs-lookup"><span data-stu-id="54ed7-111">Determine the region defining the variable you want to hide, and determine a subregion in which to redefine it with your variable.</span></span>  
  
    |<span data-ttu-id="54ed7-112">Région de variable</span><span class="sxs-lookup"><span data-stu-id="54ed7-112">Variable's region</span></span>|<span data-ttu-id="54ed7-113">Sous-région autorisée pour la redéfinir</span><span class="sxs-lookup"><span data-stu-id="54ed7-113">Allowable subregion for redefining it</span></span>|  
    |-----------------------|-------------------------------------------|  
    |<span data-ttu-id="54ed7-114">Module</span><span class="sxs-lookup"><span data-stu-id="54ed7-114">Module</span></span>|<span data-ttu-id="54ed7-115">Une classe dans le module</span><span class="sxs-lookup"><span data-stu-id="54ed7-115">A class within the module</span></span>|  
    |<span data-ttu-id="54ed7-116">Classe</span><span class="sxs-lookup"><span data-stu-id="54ed7-116">Class</span></span>|<span data-ttu-id="54ed7-117">Une sous-classe de la classe</span><span class="sxs-lookup"><span data-stu-id="54ed7-117">A subclass within the class</span></span><br /><br /> <span data-ttu-id="54ed7-118">Une procédure au sein de la classe</span><span class="sxs-lookup"><span data-stu-id="54ed7-118">A procedure within the class</span></span>|  
  
     <span data-ttu-id="54ed7-119">Vous ne pouvez pas redéfinir une variable de procédure dans un bloc au sein de cette procédure, par exemple dans un `If`... `End If` construction ou une `For` boucle.</span><span class="sxs-lookup"><span data-stu-id="54ed7-119">You cannot redefine a procedure variable in a block within that procedure, for example in an `If`...`End If` construction or a `For` loop.</span></span>  
  
2. <span data-ttu-id="54ed7-120">Créez la sous-région si elle n’existe pas déjà.</span><span class="sxs-lookup"><span data-stu-id="54ed7-120">Create the subregion if it does not already exist.</span></span>  
  
3. <span data-ttu-id="54ed7-121">Dans la sous-région, écrivez un [instruction Dim](../../../../visual-basic/language-reference/statements/dim-statement.md) déclare la variable occultante.</span><span class="sxs-lookup"><span data-stu-id="54ed7-121">Within the subregion, write a [Dim Statement](../../../../visual-basic/language-reference/statements/dim-statement.md) declaring the shadowing variable.</span></span>  
  
     <span data-ttu-id="54ed7-122">En cas de code à l’intérieur de la sous-région fait référence au nom de variable, le compilateur résout la référence à la variable occultante.</span><span class="sxs-lookup"><span data-stu-id="54ed7-122">When code inside the subregion refers to the variable name, the compiler resolves the reference to the shadowing variable.</span></span>  
  
     <span data-ttu-id="54ed7-123">L’exemple suivant illustre l’occultation par portée, ainsi qu’une référence qui outrepasse l’occultation.</span><span class="sxs-lookup"><span data-stu-id="54ed7-123">The following example illustrates shadowing through scope, as well as a reference that bypasses the shadowing.</span></span>  
  
    ```  
    Module shadowByScope  
        ' The following statement declares num as a module-level variable.  
        Public num As Integer  
        Sub show()  
            ' The following statement declares num as a local variable.  
            Dim num As Integer  
            ' The following statement sets the value of the local variable.  
            num = 2  
            ' The following statement displays the module-level variable.  
            MsgBox(CStr(shadowByScope.num))  
        End Sub  
        Sub useModuleLevelNum()  
            ' The following statement sets the value of the module-level variable.  
            num = 1  
            show()  
        End Sub  
    End Module  
    ```  
  
     <span data-ttu-id="54ed7-124">L’exemple précédent déclare la variable `num` à la fois au niveau du module et au niveau de la procédure (dans la procédure `show`).</span><span class="sxs-lookup"><span data-stu-id="54ed7-124">The preceding example declares the variable `num` both at module level and at procedure level (in the procedure `show`).</span></span> <span data-ttu-id="54ed7-125">La variable locale `num` occulte la variable de niveau module `num` dans `show`, de sorte que la variable locale est définie sur 2.</span><span class="sxs-lookup"><span data-stu-id="54ed7-125">The local variable `num` shadows the module-level variable `num` within `show`, so the local variable is set to 2.</span></span> <span data-ttu-id="54ed7-126">Toutefois, il n’existe aucune variable locale pour les clichés instantanés `num` dans le `useModuleLevelNum` procédure.</span><span class="sxs-lookup"><span data-stu-id="54ed7-126">However, there is no local variable to shadow `num` in the `useModuleLevelNum` procedure.</span></span> <span data-ttu-id="54ed7-127">Par conséquent, `useModuleLevelNum` définit la valeur de la variable au niveau du module à 1.</span><span class="sxs-lookup"><span data-stu-id="54ed7-127">Therefore, `useModuleLevelNum` sets the value of the module-level variable to 1.</span></span>  
  
     <span data-ttu-id="54ed7-128">Le `MsgBox` appeler `show` ignore le mécanisme d’occultation en qualifiant `num` avec le nom du module.</span><span class="sxs-lookup"><span data-stu-id="54ed7-128">The `MsgBox` call inside `show` bypasses the shadowing mechanism by qualifying `num` with the module name.</span></span> <span data-ttu-id="54ed7-129">Par conséquent, il affiche la variable au niveau du module au lieu de la variable locale.</span><span class="sxs-lookup"><span data-stu-id="54ed7-129">Therefore, it displays the module-level variable instead of the local variable.</span></span>  
  
#### <a name="to-hide-a-variable-by-shadowing-it-through-inheritance"></a><span data-ttu-id="54ed7-130">Pour masquer une variable par l’occultation par héritage</span><span class="sxs-lookup"><span data-stu-id="54ed7-130">To hide a variable by shadowing it through inheritance</span></span>  
  
1. <span data-ttu-id="54ed7-131">Assurez-vous que la variable que vous souhaitez masquer est déclarée dans une classe et au niveau de la classe (en dehors de toute procédure).</span><span class="sxs-lookup"><span data-stu-id="54ed7-131">Be sure the variable you want to hide is declared in a class, and at class level (outside any procedure).</span></span> <span data-ttu-id="54ed7-132">Sinon, vous ne peut pas la masquer par héritage.</span><span class="sxs-lookup"><span data-stu-id="54ed7-132">Otherwise you cannot shadow it through inheritance.</span></span>  
  
2. <span data-ttu-id="54ed7-133">Définissez une classe dérivée à partir de la classe de la variable si celle-ci n’existe pas.</span><span class="sxs-lookup"><span data-stu-id="54ed7-133">Define a class derived from the variable's class if one does not already exist.</span></span>  
  
3. <span data-ttu-id="54ed7-134">À l’intérieur de la classe dérivée, écrivez une `Dim` instruction déclare votre variable.</span><span class="sxs-lookup"><span data-stu-id="54ed7-134">Inside the derived class, write a `Dim` statement declaring your variable.</span></span> <span data-ttu-id="54ed7-135">Inclure le [Shadows](../../../../visual-basic/language-reference/modifiers/shadows.md) mot clé dans la déclaration.</span><span class="sxs-lookup"><span data-stu-id="54ed7-135">Include the [Shadows](../../../../visual-basic/language-reference/modifiers/shadows.md) keyword in the declaration.</span></span>  
  
     <span data-ttu-id="54ed7-136">En cas de code dans la classe dérivée fait référence au nom de variable, le compilateur résout la référence à votre variable.</span><span class="sxs-lookup"><span data-stu-id="54ed7-136">When code in the derived class refers to the variable name, the compiler resolves the reference to your variable.</span></span>  
  
     <span data-ttu-id="54ed7-137">L’exemple suivant illustre l’occultation par héritage.</span><span class="sxs-lookup"><span data-stu-id="54ed7-137">The following example illustrates shadowing through inheritance.</span></span> <span data-ttu-id="54ed7-138">Il effectue deux références, qui accède à la variable occultation et une qui outrepasse l’occultation.</span><span class="sxs-lookup"><span data-stu-id="54ed7-138">It makes two references, one that accesses the shadowing variable and one that bypasses the shadowing.</span></span>  
  
    ```  
    Public Class shadowBaseClass  
        Public shadowString As String = "This is the base class string."  
    End Class  
    Public Class shadowDerivedClass  
        Inherits shadowBaseClass  
        Public Shadows shadowString As String = "This is the derived class string."  
        Public Sub showStrings()  
            Dim s As String = "Unqualified shadowString: " & shadowString &  
                 vbCrLf & "MyBase.shadowString: " & MyBase.shadowString  
            MsgBox(s)  
        End Sub  
    End Class  
    ```  
  
     <span data-ttu-id="54ed7-139">L’exemple précédent déclare la variable `shadowString` dans la classe de base et la masque dans la classe dérivée.</span><span class="sxs-lookup"><span data-stu-id="54ed7-139">The preceding example declares the variable `shadowString` in the base class and shadows it in the derived class.</span></span> <span data-ttu-id="54ed7-140">La procédure `showStrings` dans la classe dérivée, affiche la version de masquage de la chaîne lorsque le nom `shadowString` n’est pas qualifié.</span><span class="sxs-lookup"><span data-stu-id="54ed7-140">The procedure `showStrings` in the derived class displays the shadowing version of the string when the name `shadowString` is not qualified.</span></span> <span data-ttu-id="54ed7-141">Il affiche ensuite la version ombrée lorsque `shadowString` est qualifié avec le `MyBase` mot clé.</span><span class="sxs-lookup"><span data-stu-id="54ed7-141">It then displays the shadowed version when `shadowString` is qualified with the `MyBase` keyword.</span></span>
