# <a name="overload-resolution-visual-basic"></a><span data-ttu-id="ba91d-102">Résolution de surcharge (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="ba91d-102">Overload Resolution (Visual Basic)</span></span>

<span data-ttu-id="ba91d-103">Lorsque le compilateur Visual Basic rencontre un appel à une procédure qui est définie dans plusieurs versions surchargées, le compilateur doit décider de la surcharge à appeler.</span><span class="sxs-lookup"><span data-stu-id="ba91d-103">When the Visual Basic compiler encounters a call to a procedure that is defined in several overloaded versions, the compiler must decide which of the overloads to call.</span></span> <span data-ttu-id="ba91d-104">Pour ce faire, il effectue les étapes suivantes :</span><span class="sxs-lookup"><span data-stu-id="ba91d-104">It does this by performing the following steps:</span></span>  
  
1. <span data-ttu-id="ba91d-105">**Accessibilité.**</span><span class="sxs-lookup"><span data-stu-id="ba91d-105">**Accessibility.**</span></span> <span data-ttu-id="ba91d-106">Elle élimine toute surcharge avec un niveau d’accès qui empêche le code appelant de l’appeler.</span><span class="sxs-lookup"><span data-stu-id="ba91d-106">It eliminates any overload with an access level that prevents the calling code from calling it.</span></span>  
  
2. <span data-ttu-id="ba91d-107">**Nombre de paramètres.**</span><span class="sxs-lookup"><span data-stu-id="ba91d-107">**Number of Parameters.**</span></span> <span data-ttu-id="ba91d-108">Elle élimine toutes les surcharges qui définissent un nombre de paramètres différent de ceux fournis dans l’appel.</span><span class="sxs-lookup"><span data-stu-id="ba91d-108">It eliminates any overload that defines a different number of parameters than are supplied in the call.</span></span>  
  
3. <span data-ttu-id="ba91d-109">**Types de données de paramètre.**</span><span class="sxs-lookup"><span data-stu-id="ba91d-109">**Parameter Data Types.**</span></span> <span data-ttu-id="ba91d-110">Le compilateur fournit la préférence des méthodes d’instance sur les méthodes d’extension.</span><span class="sxs-lookup"><span data-stu-id="ba91d-110">The compiler gives instance methods preference over extension methods.</span></span> <span data-ttu-id="ba91d-111">Si vous trouvez une méthode d’instance qui requiert que seules les conversions étendues correspondent à l’appel de procédure, toutes les méthodes d’extension sont supprimées et le compilateur continue uniquement avec les candidats de méthode d’instance.</span><span class="sxs-lookup"><span data-stu-id="ba91d-111">If any instance method is found that requires only widening conversions to match the procedure call, all extension methods are dropped and the compiler continues with only the instance method candidates.</span></span> <span data-ttu-id="ba91d-112">Si aucune méthode d’instance n’est trouvée, elle se poursuit avec les méthodes d’instance et d’extension.</span><span class="sxs-lookup"><span data-stu-id="ba91d-112">If no such instance method is found, it continues with both instance and extension methods.</span></span>  
  
     <span data-ttu-id="ba91d-113">Dans cette étape, il élimine toute surcharge pour laquelle les types de données des arguments d’appel ne peuvent pas être convertis en types de paramètres définis dans la surcharge.</span><span class="sxs-lookup"><span data-stu-id="ba91d-113">In this step, it eliminates any overload for which the data types of the calling arguments cannot be converted to the parameter types defined in the overload.</span></span>  
  
4. <span data-ttu-id="ba91d-114">**Conversions restrictives.**</span><span class="sxs-lookup"><span data-stu-id="ba91d-114">**Narrowing Conversions.**</span></span> <span data-ttu-id="ba91d-115">Elle élimine toute surcharge qui requiert une conversion restrictive des types d’arguments d’appel aux types de paramètres définis.</span><span class="sxs-lookup"><span data-stu-id="ba91d-115">It eliminates any overload that requires a narrowing conversion from the calling argument types to the defined parameter types.</span></span> <span data-ttu-id="ba91d-116">Cela est vrai, que le commutateur de vérification de type ([Option Strict Statement](../../../language-reference/statements/option-strict-statement.md)) soit `On` ou `Off` .</span><span class="sxs-lookup"><span data-stu-id="ba91d-116">This is true whether the type checking switch ([Option Strict Statement](../../../language-reference/statements/option-strict-statement.md)) is `On` or `Off`.</span></span>  
  
5. <span data-ttu-id="ba91d-117">**Le moins étendu.**</span><span class="sxs-lookup"><span data-stu-id="ba91d-117">**Least Widening.**</span></span> <span data-ttu-id="ba91d-118">Le compilateur considère les surcharges restantes par paires.</span><span class="sxs-lookup"><span data-stu-id="ba91d-118">The compiler considers the remaining overloads in pairs.</span></span> <span data-ttu-id="ba91d-119">Pour chaque paire, il compare les types de données des paramètres définis.</span><span class="sxs-lookup"><span data-stu-id="ba91d-119">For each pair, it compares the data types of the defined parameters.</span></span> <span data-ttu-id="ba91d-120">Si les types de l’une des surcharges s’étendent aux types correspondants dans l’autre, le compilateur élimine ce dernier.</span><span class="sxs-lookup"><span data-stu-id="ba91d-120">If the types in one of the overloads all widen to the corresponding types in the other, the compiler eliminates the latter.</span></span> <span data-ttu-id="ba91d-121">Autrement dit, elle conserve la surcharge qui requiert le moins d’élargissement.</span><span class="sxs-lookup"><span data-stu-id="ba91d-121">That is, it retains the overload that requires the least amount of widening.</span></span>  
  
6. <span data-ttu-id="ba91d-122">**Candidat unique.**</span><span class="sxs-lookup"><span data-stu-id="ba91d-122">**Single Candidate.**</span></span> <span data-ttu-id="ba91d-123">Il continue à considérer les surcharges par paires jusqu’à ce qu’une seule surcharge soit conservée et résout l’appel à cette surcharge.</span><span class="sxs-lookup"><span data-stu-id="ba91d-123">It continues considering overloads in pairs until only one overload remains, and it resolves the call to that overload.</span></span> <span data-ttu-id="ba91d-124">Si le compilateur ne peut pas réduire les surcharges à un seul candidat, il génère une erreur.</span><span class="sxs-lookup"><span data-stu-id="ba91d-124">If the compiler cannot reduce the overloads to a single candidate, it generates an error.</span></span>
