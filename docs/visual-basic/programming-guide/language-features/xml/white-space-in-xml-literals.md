## <a name="significant-and-insignificant-white-space"></a><span data-ttu-id="b958d-105">Espaces blancs significatifs et non significatifs</span><span class="sxs-lookup"><span data-stu-id="b958d-105">Significant and Insignificant White Space</span></span>  
 <span data-ttu-id="b958d-106">Caractères d’espace blanc dans les littéraux XML sont significatifs que dans trois zones :</span><span class="sxs-lookup"><span data-stu-id="b958d-106">White space characters in XML literals are significant in only three areas:</span></span>  
  
- <span data-ttu-id="b958d-107">Lorsqu’ils sont dans une valeur d’attribut.</span><span class="sxs-lookup"><span data-stu-id="b958d-107">When they are in an attribute value.</span></span>

- <span data-ttu-id="b958d-109">Lorsqu’ils sont dans une expression incorporée pour le contenu de texte d’un élément.</span><span class="sxs-lookup"><span data-stu-id="b958d-109">When they are in an embedded expression for an element's text content.</span></span>  
  
 <span data-ttu-id="b958d-110">Sinon, le compilateur traite les caractères d’espace blanc comme insignifiants et ne les inclut pas dans le [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] objet pour le littéral.</span><span class="sxs-lookup"><span data-stu-id="b958d-110">Otherwise, the compiler treats white space characters as insignificant and does not include then in the [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] object for the literal.</span></span>  
  
 <span data-ttu-id="b958d-111">Pour inclure des espaces blancs non significatifs dans un littéral XML, utilisez une expression incorporée qui contient un littéral de chaîne avec les espaces blancs.</span><span class="sxs-lookup"><span data-stu-id="b958d-111">To include insignificant white space in an XML literal, use an embedded expression that contains a string literal with the white space.</span></span>
