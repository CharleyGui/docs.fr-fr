<span data-ttu-id="5fb60-103">Une expression accède à un élément de programmation défini dans une classe, une structure, un module ou une interface qui contient plusieurs membres portant le même nom.</span><span class="sxs-lookup"><span data-stu-id="5fb60-103">An expression accesses a programming element defined in a class, structure, module, or interface that contains more than one member with the same name.</span></span>  
  
 <span data-ttu-id="5fb60-104">Le *respect de la casse*est probablement à l’origine de cette erreur.</span><span class="sxs-lookup"><span data-stu-id="5fb60-104">The most likely cause of this error is *case sensitivity*.</span></span> <span data-ttu-id="5fb60-105">Les noms Visual Basic ne respectent pas la casse, ce qui signifie que vous pouvez les capitaliser différemment à différents endroits dans votre code.</span><span class="sxs-lookup"><span data-stu-id="5fb60-105">Visual Basic names are case-insensitive, which means you can capitalize them differently at different places in your code.</span></span> <span data-ttu-id="5fb60-106">Par exemple, si vous définissez une variable avec le nom `XYZ` et que vous y accédez par la suite en utilisant le nom `xyz`, le compilateur considère les deux noms comme étant identiques.</span><span class="sxs-lookup"><span data-stu-id="5fb60-106">For example, if you define a variable with the name `XYZ` and later access it as `xyz`, the compiler considers the two names to be equivalent.</span></span>  
  
 <span data-ttu-id="5fb60-107">Toutefois, d’autres langages, tels que [C#](../../csharp/index.yml) et [Visual C++](/cpp/index), respectent la casse.</span><span class="sxs-lookup"><span data-stu-id="5fb60-107">However, other languages, such as [C#](../../csharp/index.yml) and [Visual C++](/cpp/index), are case-sensitive.</span></span> <span data-ttu-id="5fb60-108">Dans ces langages, `XYZ` et `xyz` sont considérés comme des noms différents.</span><span class="sxs-lookup"><span data-stu-id="5fb60-108">In such a language, `XYZ` and `xyz` are not considered to be the same name.</span></span> <span data-ttu-id="5fb60-109">Par conséquent, une classe écrite dans ces langages peut définir une variable nommée `XYZ` et une propriété nommée `xyz`.</span><span class="sxs-lookup"><span data-stu-id="5fb60-109">Therefore, a class written in such a language could define a variable named `XYZ` and a property named `xyz`.</span></span> <span data-ttu-id="5fb60-110">Le Common Language Runtime (CLR) maintient le respect de la casse dans les assemblys.</span><span class="sxs-lookup"><span data-stu-id="5fb60-110">The common language runtime (CLR) preserves case sensitivity in assemblies.</span></span> <span data-ttu-id="5fb60-111">Toutefois, si une application Visual Basic accède à un assembly portant les noms `XYZ` et `xyz`, ceux-ci apparaissent avec le même nom.</span><span class="sxs-lookup"><span data-stu-id="5fb60-111">However, if a Visual Basic application accesses an assembly with names `XYZ` and `xyz`, they appear as the same name.</span></span>
