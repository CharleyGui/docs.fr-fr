# <a name="unable-to-generate-a-reference-to-file-filename-use-tlbimp-utility-to-reference-com-dlls-error-message"></a>Impossible de générer une référence au fichier '\<filename>' (utilisez l’utilitaire TLBIMP pour référencer les DLL COM) : \<error message>

Le compilateur Visual Basic appelle Assembly Linker (Al.exe, également appelé ALink) pour générer un assembly avec un manifeste. L’éditeur de liens a signalé une erreur lors de la recherche ou de la validation d’un fichier DLL COM+.  
  
 **ID d’erreur :** BC30142  
  
## <a name="to-correct-this-error"></a>Pour corriger cette erreur  
  
1. Examinez le message d’erreur cité et consultez la rubrique  [Al.exe (Assembly Linker)](../../framework/tools/al-exe-assembly-linker.md) pour obtenir plus d’explications et de conseils.  
  
2. Si la référence souhaitée concerne une DLL COM plutôt qu’une DLL COM+, utilisez l' [Tlbimp.exe (importateur de bibliothèques de types)](../../framework/tools/tlbimp-exe-type-library-importer.md) pour générer la référence.  
  
3. Si l'erreur persiste, rassemblez des informations sur ses circonstances et avertissez les services de support technique Microsoft.
