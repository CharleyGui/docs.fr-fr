# <a name="type-or-with-expected"></a><span data-ttu-id="aded1-102">Type ou 'With' attendu</span><span class="sxs-lookup"><span data-stu-id="aded1-102">Type or 'With' expected</span></span>

<span data-ttu-id="aded1-103">Quand vous déclarez une instance de classe, le mot clé `New` doit être suivi d’un nom de type ou de `With`.</span><span class="sxs-lookup"><span data-stu-id="aded1-103">When you declare an instance of a class, the `New` keyword must be followed by a type name or by `With`.</span></span> <span data-ttu-id="aded1-104">Par exemple, les instructions suivantes déclarent chacune `client` comme étant une instance de la classe `Customer` .</span><span class="sxs-lookup"><span data-stu-id="aded1-104">For example, the following statements each declare `client` to be an instance of the `Customer` class.</span></span> <span data-ttu-id="aded1-105">Le nom de type `Customer` suit `New`.</span><span class="sxs-lookup"><span data-stu-id="aded1-105">The type name `Customer` follows `New`.</span></span>  
  
```vb  
' Dim client As New Customer()  
' The next declaration uses an object initializer.  
Dim client As New Customer() With {.Name = "Litware, Inc."}  
```  
  
 <span data-ttu-id="aded1-106">À partir de Visual Basic 2008, vous pouvez déclarer un objet comme une instance d’un type anonyme, auquel cas vous ne spécifiez pas de type de données.</span><span class="sxs-lookup"><span data-stu-id="aded1-106">Beginning with Visual Basic 2008, you can declare an object to be an instance of an anonymous type, in which case you do not specify a data type.</span></span> <span data-ttu-id="aded1-107">Dans les déclarations de type anonyme, le mot clé `With` suit `New`.</span><span class="sxs-lookup"><span data-stu-id="aded1-107">In anonymous type declarations, the keyword `With` follows `New`.</span></span>  
  
```vb  
Dim person = New With {.Name ="Mike Nash", .Age = 27}  
```
