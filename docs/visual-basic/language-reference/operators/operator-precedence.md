## <a name="associativity"></a><span data-ttu-id="296b1-135">l'associativité</span><span class="sxs-lookup"><span data-stu-id="296b1-135">Associativity</span></span>  
 <span data-ttu-id="296b1-136">Lorsque les opérateurs de même priorité apparaissent ensemble dans une expression, par exemple multiplication et division, le compilateur évalue chaque opération qu’elle rencontre de gauche à droite.</span><span class="sxs-lookup"><span data-stu-id="296b1-136">When operators of equal precedence appear together in an expression, for example multiplication and division, the compiler evaluates each operation as it encounters it from left to right.</span></span> <span data-ttu-id="296b1-137">L'exemple suivant illustre ce comportement.</span><span class="sxs-lookup"><span data-stu-id="296b1-137">The following example illustrates this.</span></span>  
  
```  
Dim n1 As Integer = 96 / 8 / 4  
Dim n2 As Integer = (96 / 8) / 4  
Dim n3 As Integer = 96 / (8 / 4)  
```  
  
 <span data-ttu-id="296b1-138">La première expression évalue la division 96 / 8 (ce qui entraîne à 12), puis la division 12 / 4, ce qui se traduit dans trois.</span><span class="sxs-lookup"><span data-stu-id="296b1-138">The first expression evaluates the division 96 / 8 (which results in 12) and then the division 12 / 4, which results in three.</span></span> <span data-ttu-id="296b1-139">Étant donné que le compilateur évalue les opérations pour `n1` de gauche à droite, la version d’évaluation est le même lorsque cet ordre est indiqué explicitement pour `n2`.</span><span class="sxs-lookup"><span data-stu-id="296b1-139">Because the compiler evaluates the operations for `n1` from left to right, the evaluation is the same when that order is explicitly indicated for `n2`.</span></span> <span data-ttu-id="296b1-140">Les deux `n1` et `n2` ont pour résultat trois.</span><span class="sxs-lookup"><span data-stu-id="296b1-140">Both `n1` and `n2` have a result of three.</span></span> <span data-ttu-id="296b1-141">En revanche, `n3` a un résultat de 48, étant donné que les parenthèses forcent le compilateur à évaluer 8 / 4 première.</span><span class="sxs-lookup"><span data-stu-id="296b1-141">By contrast, `n3` has a result of 48, because the parentheses force the compiler to evaluate 8 / 4 first.</span></span>  
  
 <span data-ttu-id="296b1-142">Étant donné ce comportement, les opérateurs sont dits *associatifs sur leur gauche* en Visual Basic.</span><span class="sxs-lookup"><span data-stu-id="296b1-142">Because of this behavior, operators are said to be *left associative* in Visual Basic.</span></span>  
  
## <a name="overriding-precedence-and-associativity"></a><span data-ttu-id="296b1-143">Substitution de priorité et associativité</span><span class="sxs-lookup"><span data-stu-id="296b1-143">Overriding Precedence and Associativity</span></span>  
 <span data-ttu-id="296b1-144">Vous pouvez utiliser des parenthèses pour forcer certaines parties d’une expression à évaluer avant les autres.</span><span class="sxs-lookup"><span data-stu-id="296b1-144">You can use parentheses to force some parts of an expression to be evaluated before others.</span></span> <span data-ttu-id="296b1-145">Cela peut remplacer l’ordre de priorité et l’associativité de gauche.</span><span class="sxs-lookup"><span data-stu-id="296b1-145">This can override both the order of precedence and the left associativity.</span></span> <span data-ttu-id="296b1-146">Visual Basic exécute toujours les opérations qui sont placées entre parenthèses avant celles en dehors.</span><span class="sxs-lookup"><span data-stu-id="296b1-146">Visual Basic always performs operations that are enclosed in parentheses before those outside.</span></span> <span data-ttu-id="296b1-147">Toutefois, dans les parenthèses, il maintient ordinaire priorité et associativité, sauf si vous utilisez des parenthèses dans les parenthèses.</span><span class="sxs-lookup"><span data-stu-id="296b1-147">However, within parentheses, it maintains ordinary precedence and associativity, unless you use parentheses within the parentheses.</span></span> <span data-ttu-id="296b1-148">L'exemple suivant illustre ce comportement.</span><span class="sxs-lookup"><span data-stu-id="296b1-148">The following example illustrates this.</span></span>  
  
```  
Dim a, b, c, d, e, f, g As Double  
a = 8.0  
b = 3.0  
c = 4.0  
d = 2.0  
e = 1.0  
f = a - b + c / d * e  
' The preceding line sets f to 7.0. Because of natural operator   
' precedence and associativity, it is exactly equivalent to the   
' following line.  
f = (a - b) + ((c / d) * e)  
' The following line overrides the natural operator precedence   
' and left associativity.  
g = (a - (b + c)) / (d * e)  
' The preceding line sets g to 0.5.  
```
