## <a name="cause"></a><span data-ttu-id="84fb7-111">Cause</span><span class="sxs-lookup"><span data-stu-id="84fb7-111">Cause</span></span>

## <a name="rule-description"></a><span data-ttu-id="84fb7-114">Description de la règle</span><span class="sxs-lookup"><span data-stu-id="84fb7-114">Rule description</span></span>

<span data-ttu-id="84fb7-115">Cette règle fractionne le nom de paramètre en jetons en fonction de la Convention de casse mixte et vérifie si chaque jeton est égal à « URI », « Uri », « URN », « URN », « URL » ou « URL ».</span><span class="sxs-lookup"><span data-stu-id="84fb7-115">This rule splits the parameter name into tokens based on the camel casing convention and checks whether each token equals "uri", "Uri", "urn", "Urn", "url", or "Url".</span></span> <span data-ttu-id="84fb7-116">En cas de correspondance, la règle suppose que le paramètre représente un URI (Uniform Resource Identifier).</span><span class="sxs-lookup"><span data-stu-id="84fb7-116">If there is a match, the rule assumes that the parameter represents a uniform resource identifier (URI).</span></span> <span data-ttu-id="84fb7-117">Une représentation sous forme de chaîne d'un URI est sujette aux erreurs d'analyse et d'encodage, et peut entraîner des failles de sécurité.</span><span class="sxs-lookup"><span data-stu-id="84fb7-117">A string representation of a URI is prone to parsing and encoding errors, and can lead to security vulnerabilities.</span></span> <span data-ttu-id="84fb7-118">Si une méthode prend une représentation sous forme de chaîne d’un URI, une surcharge correspondante doit être fournie, qui prend une instance de la <xref:System.Uri> classe, qui fournit ces services de manière sécurisée et sécurisée.</span><span class="sxs-lookup"><span data-stu-id="84fb7-118">If a method takes a string representation of a URI, a corresponding overload should be provided that takes an instance of the <xref:System.Uri> class, which provides these services in a safe and secure manner.</span></span>
