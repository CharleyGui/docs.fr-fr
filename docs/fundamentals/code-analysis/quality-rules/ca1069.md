## <a name="rule-description"></a><span data-ttu-id="95e5e-113">Description de la règle</span><span class="sxs-lookup"><span data-stu-id="95e5e-113">Rule description</span></span>

<span data-ttu-id="95e5e-114">Chaque membre enum doit avoir une valeur de constante unique ou être explicitement assigné avec un membre antérieur dans l’énumération pour indiquer l’intention explicite de la valeur de partage.</span><span class="sxs-lookup"><span data-stu-id="95e5e-114">Every enum member should either have a unique constant value or be explicitly assigned with a prior member in the enum to indicate explicit intent of sharing value.</span></span> <span data-ttu-id="95e5e-115">Par exemple :</span><span class="sxs-lookup"><span data-stu-id="95e5e-115">For example:</span></span>

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = 2,   // CA1069: This is not fine. Either assign a different constant value or 'Field2' to indicate explicit intent of sharing value.
}
```

<span data-ttu-id="95e5e-116">Cette règle permet d’intercepter les bogues fonctionnels introduits dans les scénarios suivants :</span><span class="sxs-lookup"><span data-stu-id="95e5e-116">This rule helps in catching functional bugs introduced from the following scenarios:</span></span>

- <span data-ttu-id="95e5e-118">Copier les erreurs de collage, où l’utilisateur a copié une définition de membre existante, puis a renommé le membre mais a oublié de modifier la valeur.</span><span class="sxs-lookup"><span data-stu-id="95e5e-118">Copy paste mistakes, where the user copied an existing member definition, then renamed the member but forgot to change the value.</span></span>

## <a name="how-to-fix-violations"></a><span data-ttu-id="95e5e-120">Comment corriger les violations</span><span class="sxs-lookup"><span data-stu-id="95e5e-120">How to fix violations</span></span>

<span data-ttu-id="95e5e-121">Pour corriger une violation, assignez une nouvelle valeur de constante unique ou assignez à un membre précédent dans l’énumération pour indiquer l’intention explicite de partager la même valeur.</span><span class="sxs-lookup"><span data-stu-id="95e5e-121">To fix a violation, either assign a new unique constant value or assign with a prior member in the enum to indicate explicit intent of sharing the same value.</span></span> <span data-ttu-id="95e5e-122">Par exemple, l’extrait de code suivant montre une violation de la règle et deux façons de corriger la violation :</span><span class="sxs-lookup"><span data-stu-id="95e5e-122">For example, the following code snippet shows a violation of the rule and couple of ways to fix the violation:</span></span>

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = 2,   // CA1069: This is not fine. Either assign a different constant value or 'Field2' to indicate explicit intent of sharing value.
}
```

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = 3,   // This is now fine
}
```

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = Field2,   // This is also fine
}
```
