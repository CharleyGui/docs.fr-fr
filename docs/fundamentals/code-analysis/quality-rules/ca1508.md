## <a name="rule-description"></a><span data-ttu-id="4f1f0-115">Description de la règle</span><span class="sxs-lookup"><span data-stu-id="4f1f0-115">Rule description</span></span>

<span data-ttu-id="4f1f0-116">Les méthodes peuvent avoir du code conditionnel, comme les instructions If, les expressions binaires ( `==` , `!=` , `<` , `>` ), les vérifications null, etc. Prenons l’exemple du code suivant :</span><span class="sxs-lookup"><span data-stu-id="4f1f0-116">Methods can have conditional code, such as if statements, binary expressions (`==`, `!=`, `<`, `>`), null checks, etc. For example, consider the following code:</span></span>

```csharp
public void M(int i, int j)
{
    if (i != 0)
    {
        return;
    }

    if (j != 0)
    {
        return;
    }

    // Below condition will always evaluate to 'false' as 'i' and 'j' are both '0' here.
    if (i != j)
    {
        // Code in this 'if' branch is dead code.
        // It can either be removed or refactored.
        ...
    }
}
```
