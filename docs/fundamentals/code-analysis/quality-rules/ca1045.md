<span data-ttu-id="95c01-116">Quand un type référence est passé par référence, la méthode envisage d’utiliser le paramètre pour retourner une instance différente de l’objet.</span><span class="sxs-lookup"><span data-stu-id="95c01-116">When a reference type is passed "by reference," the method intends to use the parameter to return a different instance of the object.</span></span> <span data-ttu-id="95c01-117">(Le passage d’un type référence par référence est également appelé utilisation d’un pointeur double, d’un pointeur vers un pointeur ou d’une double indirection.) Using the default calling convention, which is pass "by value," a parameter that takes a reference type already receives a pointer to the object.</span></span> <span data-ttu-id="95c01-118">Le pointeur, pas l’objet vers lequel il pointe, est passé par valeur.</span><span class="sxs-lookup"><span data-stu-id="95c01-118">The pointer, not the object to which it points, is passed by value.</span></span> <span data-ttu-id="95c01-119">Le passage par valeur signifie que la méthode ne peut pas modifier le pointeur pour qu’il pointe vers une nouvelle instance du type référence, mais puisse modifier le contenu de l’objet vers lequel il pointe.</span><span class="sxs-lookup"><span data-stu-id="95c01-119">Passing by value means that the method cannot change the pointer to have it point to a new instance of the reference type, but can change the contents of the object to which it points.</span></span> <span data-ttu-id="95c01-120">Pour la plupart des applications, cela suffit et génère le comportement souhaité.</span><span class="sxs-lookup"><span data-stu-id="95c01-120">For most applications this is sufficient and yields the behavior that you want.</span></span>

<span data-ttu-id="95c01-121">Si une méthode doit retourner une instance différente, utilisez la valeur de retour de la méthode pour y parvenir.</span><span class="sxs-lookup"><span data-stu-id="95c01-121">If a method must return a different instance, use the return value of the method to accomplish this.</span></span> <span data-ttu-id="95c01-122">Consultez la <xref:System.String?displayProperty=fullName> classe pour une variété de méthodes qui opèrent sur des chaînes et retournent une nouvelle instance d’une chaîne.</span><span class="sxs-lookup"><span data-stu-id="95c01-122">See the <xref:System.String?displayProperty=fullName> class for a variety of methods that operate on strings and return a new instance of a string.</span></span> <span data-ttu-id="95c01-123">En utilisant ce modèle, il est laissé à l’appelant de décider si l’objet d’origine est conservé.</span><span class="sxs-lookup"><span data-stu-id="95c01-123">By using this model, it is left to the caller to decide whether the original object is preserved.</span></span>

<span data-ttu-id="95c01-124">Bien que les valeurs de retour soient courantes et largement utilisées, l’application correcte des `out` paramètres et requiert des compétences en matière de `ref` conception et de codage intermédiaires.</span><span class="sxs-lookup"><span data-stu-id="95c01-124">Although return values are commonplace and heavily used, the correct application of `out` and `ref` parameters requires intermediate design and coding skills.</span></span> <span data-ttu-id="95c01-125">Les architectes de bibliothèque qui sont en train de concevoir pour un public général ne doivent pas s’attendre à ce que les utilisateurs maîtrisent le travail avec les `out` `ref` paramètres ou.</span><span class="sxs-lookup"><span data-stu-id="95c01-125">Library architects who design for a general audience should not expect users to master working with `out` or `ref` parameters.</span></span>

> [!NOTE]
