# <a name="ca2302-ensure-binaryformatterbinder-is-set-before-calling-binaryformatterdeserialize"></a>CA2302 : Vérifiez que BinaryFormatter.Binder est défini avant d’appeler BinaryFormatter.Deserialize

| | Valeur |
|-|-|
| **Identificateur de la règle** |CA2302|
| **Catégorie** |[Microsoft.Security](security-warnings.md)|
| **Correction en rupture ou sans rupture** |Sans rupture|

## <a name="cause"></a>Cause

Une <xref:System.Runtime.Serialization.Formatters.Binary.BinaryFormatter?displayProperty=nameWithType> méthode de désérialisation a été appelée ou référencée et la <xref:System.Runtime.Serialization.Formatters.Binary.BinaryFormatter.Binder> propriété peut avoir la valeur null.

Cette règle est similaire à [ca2301](ca2301.md), mais l’analyse ne peut pas déterminer si <xref:System.Runtime.Serialization.Formatters.Binary.BinaryFormatter.Binder> a la valeur null.

Par défaut, cette règle analyse l’ensemble du code base, mais il peut être [configuré](#configure-code-to-analyze).

> [!WARNING]
> La restriction des types avec un SerializationBinder ne peut pas empêcher toutes les attaques. Pour plus d’informations, consultez le [Guide de sécurité BinaryFormatter](../../../standard/serialization/binaryformatter-security-guide.md).

## <a name="rule-description"></a>Description de la règle

[!INCLUDE[insecure-deserializers-description](~/includes/code-analysis/insecure-deserializers-description.md)]

Cette règle recherche <xref:System.Runtime.Serialization.Formatters.Binary.BinaryFormatter?displayProperty=nameWithType> des appels ou des références de méthode de désérialisation lorsque la <xref:System.Runtime.Serialization.Formatters.Binary.BinaryFormatter.Binder> valeur peut être null. Si vous souhaitez interdire toute désérialisation avec, <xref:System.Runtime.Serialization.Formatters.Binary.BinaryFormatter> quelle que soit la <xref:System.Runtime.Serialization.Formatters.Binary.BinaryFormatter.Binder> propriété, désactivez cette règle et [ca2301](ca2301.md), puis activez la règle [ca2300](ca2300.md).

## <a name="pseudo-code-examples"></a>Exemples de pseudo-code

### <a name="violation-1"></a>Violation 1

```csharp
using System;
using System.IO;
using System.Runtime.Serialization;
using System.Runtime.Serialization.Formatters.Binary;

public class BookRecordSerializationBinder : SerializationBinder
{
    public override Type BindToType(string assemblyName, string typeName)
    {
        // One way to discover expected types is through testing deserialization
        // of **valid** data and logging the types used.

        ////Console.WriteLine($"BindToType('{assemblyName}', '{typeName}')");

        if (typeName == "BookRecord" || typeName == "AisleLocation")
        {
            return null;
        }
        else
        {
            throw new ArgumentException("Unexpected type", nameof(typeName));
        }
    }
}

[Serializable]
public class BookRecord
{
    public string Title { get; set; }
    public AisleLocation Location { get; set; }
}

[Serializable]
public class AisleLocation
{
    public char Aisle { get; set; }
    public byte Shelf { get; set; }
}

public class Binders
{
    public static SerializationBinder BookRecord =
        new BookRecordSerializationBinder();
}

public class ExampleClass
{
    public BookRecord DeserializeBookRecord(byte[] bytes)
    {
        BinaryFormatter formatter = new BinaryFormatter();
        formatter.Binder = Binders.BookRecord;
        using (MemoryStream ms = new MemoryStream(bytes))
        {
            return (BookRecord)formatter.Deserialize(ms);    // CA2302 violation
        }
    }
}
```
