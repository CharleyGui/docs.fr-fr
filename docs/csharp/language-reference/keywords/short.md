## <a name="conversions"></a><span data-ttu-id="ffec1-125">Conversions</span><span class="sxs-lookup"><span data-stu-id="ffec1-125">Conversions</span></span>

<span data-ttu-id="ffec1-126">Il existe une conversion implicite prédéfinie de `short` en [int](int.md), [long](long.md), [float](float.md), [double](double.md) ou [decimal](decimal.md).</span><span class="sxs-lookup"><span data-stu-id="ffec1-126">There is a predefined implicit conversion from `short` to [int](int.md), [long](long.md), [float](float.md), [double](double.md), or [decimal](decimal.md).</span></span>

<span data-ttu-id="ffec1-127">Vous ne pouvez pas convertir implicitement des types numériques non littéraux de taille de stockage supérieure à `short` (consultez le [tableau des types intégraux](integral-types-table.md) pour voir les tailles de stockage des types intégraux).</span><span class="sxs-lookup"><span data-stu-id="ffec1-127">You cannot implicitly convert nonliteral numeric types of larger storage size to `short` (see [Integral Types Table](integral-types-table.md) for the storage sizes of integral types).</span></span> <span data-ttu-id="ffec1-128">Prenez l’exemple des deux variables `short``x` et `y` suivantes :</span><span class="sxs-lookup"><span data-stu-id="ffec1-128">Consider, for example, the following two `short` variables `x` and `y`:</span></span>

```csharp
short x = 5, y = 12;
```

<span data-ttu-id="ffec1-129">L’instruction d’assignation suivante entraîne une erreur de compilation, car l’expression arithmétique située à droite de l’opérateur d’assignation donne la valeur [int](int.md) par défaut.</span><span class="sxs-lookup"><span data-stu-id="ffec1-129">The following assignment statement produces a compilation error because the arithmetic expression on the right-hand side of the assignment operator evaluates to [int](int.md) by default.</span></span>

```csharp

```

<span data-ttu-id="ffec1-130">Pour corriger ce problème, utilisez un cast :</span><span class="sxs-lookup"><span data-stu-id="ffec1-130">To fix this problem, use a cast:</span></span>

<span data-ttu-id="ffec1-131">Il est toutefois possible d’utiliser les instructions suivantes, dans lesquelles la variable de destination a une taille de stockage égale ou supérieure :</span><span class="sxs-lookup"><span data-stu-id="ffec1-131">It is also possible to use the following statements, where the destination variable has the same storage size or a larger storage size:</span></span>

```csharp

long n = x + y;
```

<span data-ttu-id="ffec1-132">Il n’existe pas de conversion implicite des types virgule flottante en `short`.</span><span class="sxs-lookup"><span data-stu-id="ffec1-132">There is no implicit conversion from floating-point types to `short`.</span></span> <span data-ttu-id="ffec1-133">Par exemple, l’instruction suivante génère une erreur du compilateur, sauf si vous utilisez un cast explicite :</span><span class="sxs-lookup"><span data-stu-id="ffec1-133">For example, the following statement generates a compiler error unless an explicit cast is used:</span></span>
