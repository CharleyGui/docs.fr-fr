# <a name="lock-statement-c-reference"></a><span data-ttu-id="85689-103">instruction lock (référence C#)</span><span class="sxs-lookup"><span data-stu-id="85689-103">lock statement (C# reference)</span></span>

<span data-ttu-id="85689-104">L’instruction `lock` obtient le verrou d’exclusion mutuelle d’un objet donné, exécute un bloc d’instructions, puis libère le verrou.</span><span class="sxs-lookup"><span data-stu-id="85689-104">The `lock` statement acquires the mutual-exclusion lock for a given object, executes a statement block, and then releases the lock.</span></span> <span data-ttu-id="85689-105">Tant qu’un verrou est maintenu, le thread qui contient le verrou peut à nouveau obtenir et libérer le verrou.</span><span class="sxs-lookup"><span data-stu-id="85689-105">While a lock is held, the thread that holds the lock can again acquire and release the lock.</span></span> <span data-ttu-id="85689-106">Tout autre thread se voit bloquer l’obtention du verrou et attend que ce dernier soit libéré.</span><span class="sxs-lookup"><span data-stu-id="85689-106">Any other thread is blocked from acquiring the lock and waits until the lock is released.</span></span>

<span data-ttu-id="85689-107">L’instruction `lock` se présente sous la forme</span><span class="sxs-lookup"><span data-stu-id="85689-107">The `lock` statement is of the form</span></span>

```csharp
lock (x)
{
    // Your code...
}
```

<span data-ttu-id="85689-108">où `x` est une expression de [type référence](reference-types.md).</span><span class="sxs-lookup"><span data-stu-id="85689-108">where `x` is an expression of a [reference type](reference-types.md).</span></span> <span data-ttu-id="85689-109">Elle équivaut précisément à</span><span class="sxs-lookup"><span data-stu-id="85689-109">It's precisely equivalent to</span></span>

```csharp
object __lockObj = x;
bool __lockWasTaken = false;
try
{
    System.Threading.Monitor.Enter(__lockObj, ref __lockWasTaken);
    // Your code...
}
finally
{
    if (__lockWasTaken) System.Threading.Monitor.Exit(__lockObj);
}
```

<span data-ttu-id="85689-110">Dans la mesure où le code utilise un bloc [try...finally](try-finally.md), le verrou est libéré même si une exception est levée dans le corps d’une instruction `lock`.</span><span class="sxs-lookup"><span data-stu-id="85689-110">Since the code uses a [try...finally](try-finally.md) block, the lock is released even if an exception is thrown within the body of a `lock` statement.</span></span>

<span data-ttu-id="85689-111">Vous ne pouvez pas utiliser l’[opérateur await](../operators/await.md) dans le corps d’une instruction `lock`.</span><span class="sxs-lookup"><span data-stu-id="85689-111">You can't use the [await operator](../operators/await.md) in the body of a `lock` statement.</span></span>
