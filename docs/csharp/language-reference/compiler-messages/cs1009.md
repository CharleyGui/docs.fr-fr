# <a name="compiler-error-cs1009"></a><span data-ttu-id="75be2-103">Erreur du compilateur CS1009</span><span class="sxs-lookup"><span data-stu-id="75be2-103">Compiler Error CS1009</span></span>

<span data-ttu-id="75be2-104">Séquence d’échappement non reconnue</span><span class="sxs-lookup"><span data-stu-id="75be2-104">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="75be2-105">Un caractère inattendu suit une barre oblique inverse (\\) dans un élément [string](../builtin-types/reference-types.md#the-string-type).</span><span class="sxs-lookup"><span data-stu-id="75be2-105">An unexpected character follows a backslash (\\) in a [string](../builtin-types/reference-types.md#the-string-type).</span></span> <span data-ttu-id="75be2-106">Le compilateur attend l’un des caractères d’échappement valides.</span><span class="sxs-lookup"><span data-stu-id="75be2-106">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="75be2-107">Pour plus d’informations, consultez [Caractères d’échappement](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span><span class="sxs-lookup"><span data-stu-id="75be2-107">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="75be2-108">L’exemple suivant génère l’erreur CS1009.</span><span class="sxs-lookup"><span data-stu-id="75be2-108">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="75be2-109">Une cause courante de cette erreur est l’utilisation de la barre oblique inverse dans un nom de fichier, comme dans l’exemple suivant.</span><span class="sxs-lookup"><span data-stu-id="75be2-109">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="75be2-110">Pour résoudre cette erreur, utilisez « \\\\ » ou le littéral de chaîne @-quoted, comme dans l’exemple suivant.</span><span class="sxs-lookup"><span data-stu-id="75be2-110">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
