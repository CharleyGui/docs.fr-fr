 <span data-ttu-id="db81f-106">Vous pouvez surcharger un opérateur afin qu’il prenne en charge les opérandes de certains types.</span><span class="sxs-lookup"><span data-stu-id="db81f-106">You can overload an operator to make it support operands of certain types.</span></span> <span data-ttu-id="db81f-107">Pour plus d’informations, consultez [Surcharge d’opérateur](../../../csharp/language-reference/operators/operator-overloading.md).</span><span class="sxs-lookup"><span data-stu-id="db81f-107">For more information, see [Operator overloading](../../../csharp/language-reference/operators/operator-overloading.md).</span></span>
  
## <a name="example"></a><span data-ttu-id="db81f-108">Exemples</span><span class="sxs-lookup"><span data-stu-id="db81f-108">Example</span></span>

 <span data-ttu-id="db81f-109">Dans l’exemple suivant, l’erreur CS0019 est générée à deux emplacements, car [bool](../../../csharp/language-reference/keywords/bool.md) dans C# n’est pas convertible en [int](../../../csharp/language-reference/builtin-types/integral-numeric-types.md). L’erreur CS0019 est également générée quand l’opérateur de soustraction est appliqué à une chaîne.</span><span class="sxs-lookup"><span data-stu-id="db81f-109">In the following example, CS0019 is generated in two places because [bool](../../../csharp/language-reference/keywords/bool.md) in C# is not convertible to [int](../../../csharp/language-reference/builtin-types/integral-numeric-types.md). CS0019 also is generated when the subtraction operator is applied to a string.</span></span> <span data-ttu-id="db81f-110">L’opérateur d’addition (+) peut être utilisé avec des opérandes de chaîne, car il est surchargé par la classe `String` pour permettre la concaténation de chaînes.</span><span class="sxs-lookup"><span data-stu-id="db81f-110">The addition operator (+) can be used with string operands because that operator is overloaded by the `String` class to perform string concatenation.</span></span>  

```csharp
static void Main()  
{  
    bool result = true;  
    if (result > 0) //CS0019  
    {  
        // Do something.  
    }  
  
    int i = 1;  
    // You cannot compare an integer and a boolean value.  
    if (i == true) //CS0019  
    {  
        //Do something...  
    }  
  
    // The following use of == causes no error. It is the comparison of
    // an integer and a boolean value that causes the error in the
    // previous if statement.  
    if (result == true)  
    {  
        //Do something...  
    }  
  
    string s = "Just try to subtract me.";  
    float f = 100 - s; // CS0019  
}  
```

 <span data-ttu-id="db81f-112">Dans l’exemple suivant, la logique conditionnelle doit être spécifiée en dehors de <xref:System.Diagnostics.ConditionalAttribute>.</span><span class="sxs-lookup"><span data-stu-id="db81f-112">In the following example, conditional logic must be specified outside the <xref:System.Diagnostics.ConditionalAttribute>.</span></span> <span data-ttu-id="db81f-113">Vous ne pouvez passer qu’un symbole prédéfini à <xref:System.Diagnostics.ConditionalAttribute>.</span><span class="sxs-lookup"><span data-stu-id="db81f-113">You can pass only one predefined symbol to the <xref:System.Diagnostics.ConditionalAttribute>.</span></span>  
  
 <span data-ttu-id="db81f-114">L’exemple suivant génère l’erreur CS0019.</span><span class="sxs-lookup"><span data-stu-id="db81f-114">The following sample generates CS0019.</span></span>  

```csharp

public class MyClass  
{  
   [ConditionalAttribute("DEBUG" || "TRACE")]   // CS0019

}  
```
