# <a name="compiler-warning-level-3-cs0675"></a><span data-ttu-id="4b521-103">Avertissement du compilateur (niveau 3) CS0675</span><span class="sxs-lookup"><span data-stu-id="4b521-103">Compiler Warning (level 3) CS0675</span></span>

<span data-ttu-id="4b521-104">Opérateur de bits OR utilisé sur un opérande de signe étendu ; effectuez d’abord un cast en un type non signé plus faible</span><span class="sxs-lookup"><span data-stu-id="4b521-104">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="4b521-105">Le compilateur a explicitement élargi une variable et étendu le signe de celle-ci, puis il a utilisé la valeur résultante dans une opération de bits OR.</span><span class="sxs-lookup"><span data-stu-id="4b521-105">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="4b521-106">Cela peut entraîner un comportement inattendu.</span><span class="sxs-lookup"><span data-stu-id="4b521-106">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="4b521-107">L’exemple suivant génère l’avertissement CS0675 :</span><span class="sxs-lookup"><span data-stu-id="4b521-107">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
