# <a name="compiler-error-cs0201"></a>Erreur du compilateur CS0201

Seuls une assignation, un appel, un incrément, un décrément et des expressions d’objet new peuvent être utilisés comme instruction  
  
 Le compilateur génère une erreur quand il rencontre une instruction non valide. Une déclaration invalide est toute ligne ou série de lignes se[=](../operators/assignment-operator.md)terminant par un point-virgule qui ne représente pas une affectation ( ), méthode d’appel [()](../operators/member-access-operators.md#invocation-operator-), [nouveau](../operators/new-operator.md), [--](../operators/arithmetic-operators.md#decrement-operator---) ou [++](../operators/arithmetic-operators.md#increment-operator-) opération. Pour plus d’informations, consultez [Instructions, expressions et opérateurs](../../programming-guide/statements-expressions-operators/index.md).  
  
## <a name="example"></a> Exemple

 L’exemple suivant génère l’erreur CS0201, car 2 * 3 est une expression, et non une instruction. Pour que le code se compile, essayez d’assigner la valeur de l’expression à une variable.  

```csharp
// CS0201.cs  
public class MainClass  
{  
   public static void Main()  
   {  
      2 * 3;   // CS0201  
      // Try the following line instead.  
      //   int i = 2 * 3;  
   }  
}  
```

## <a name="example"></a> Exemple

 L’exemple suivant génère l’erreur CS0201, car l’expression checked n’est pas en soi une instruction, bien qu’elle soit paramétrable par une opération d’incrémentation.  

```csharp
// CS0201_b.cs  
// compile with: /target:library  
public class MyList<T>
{  
   public void Add(T x)  
   {  
      int i = 0;  
      if ( (object)x == null)  
      {  
         checked(i++);   // CS0201  
  
         // OK  
         checked {  
            i++;
         }  
      }  
   }  
}  
```
