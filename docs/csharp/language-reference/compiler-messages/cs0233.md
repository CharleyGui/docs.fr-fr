# <a name="compiler-error-cs0233"></a><span data-ttu-id="ba768-102">Erreur du compilateur CS0233</span><span class="sxs-lookup"><span data-stu-id="ba768-102">Compiler Error CS0233</span></span>
<span data-ttu-id="ba768-103">« identifier » n’a pas de taille prédéfinie ; c’est pourquoi sizeof ne peut être utilisé que dans un contexte unsafe</span><span class="sxs-lookup"><span data-stu-id="ba768-103">'identifier' does not have a predefined size, therefore sizeof can only be used in an unsafe context</span></span>
  
 <span data-ttu-id="ba768-104">Sans contexte [unsafe](../keywords/unsafe.md), l’opérateur [sizeof](../operators/sizeof.md) ne peut être utilisé que pour les types dont la taille est une constante au moment de la compilation.</span><span class="sxs-lookup"><span data-stu-id="ba768-104">Without [unsafe](../keywords/unsafe.md) context, the [sizeof](../operators/sizeof.md) operator can only be used for types which size is a compile-time constant.</span></span> <span data-ttu-id="ba768-105">Si vous obtenez cette erreur, utilisez un contexte unsafe.</span><span class="sxs-lookup"><span data-stu-id="ba768-105">If you are getting this error, use an unsafe context.</span></span>
  
<span data-ttu-id="ba768-106">L’exemple suivant génère l’erreur CS0233 :</span><span class="sxs-lookup"><span data-stu-id="ba768-106">The following example generates CS0233:</span></span>
  
```csharp  
// CS0233.cs  
using System;  
using System.Runtime.InteropServices;  
  
[StructLayout(LayoutKind.Sequential)]  
public struct S  
{  
    public int a;  
}  
  
public class MyClass  
{  
    public static void Main()  
    {  
        S myS = new S();  
        Console.WriteLine(sizeof(S));   // CS0233  
        // Try the following instead:  
        // unsafe
        // {
        //     Console.WriteLine(sizeof(S));
        // }
   }  
}  
```
