# <a name="compiler-warning-level-1-cs1690"></a><span data-ttu-id="81feb-102">Avertissement du compilateur (niveau 1) CS1690</span><span class="sxs-lookup"><span data-stu-id="81feb-102">Compiler Warning (level 1) CS1690</span></span>
<span data-ttu-id="81feb-103">L’accès à un membre sur 'member' peut provoquer une exception runtime, car il s’agit d’un champ d’une classe marshalée par référence</span><span class="sxs-lookup"><span data-stu-id="81feb-103">Accessing a member on 'member' may cause a runtime exception because it is a field of a marshal-by-reference class</span></span>  
  
 <span data-ttu-id="81feb-104">Cet avertissement se produit quand vous essayez d’appeler une méthode, une propriété ou un indexeur sur un membre d’une classe qui dérive de <xref:System.MarshalByRefObject> et que le membre est un type valeur.</span><span class="sxs-lookup"><span data-stu-id="81feb-104">This warning occurs when you try to call a method, property, or indexer on a member of a class that derives from <xref:System.MarshalByRefObject>, and the member is a value type.</span></span> <span data-ttu-id="81feb-105">Les objets qui héritent de `MarshalByRefObject` doivent généralement être marshalés par référence dans un domaine d’application.</span><span class="sxs-lookup"><span data-stu-id="81feb-105">Objects that inherit from `MarshalByRefObject` are typically intended to be marshaled by reference across an application domain.</span></span> <span data-ttu-id="81feb-106">Si un code tente d’accéder directement au membre de type valeur d’un tel objet sur un domaine d’application, un temps d’exécution <xref:System.InvalidOperationException> se produira.</span><span class="sxs-lookup"><span data-stu-id="81feb-106">If any code ever attempts to directly access the value-type member of such an object across an application domain, a runtime <xref:System.InvalidOperationException> will occur.</span></span> <span data-ttu-id="81feb-107">Pour résoudre l’avertissement, commencez par copier le membre dans une variable locale et appelez la méthode sur cette variable.</span><span class="sxs-lookup"><span data-stu-id="81feb-107">To resolve the warning, first copy the member into a local variable and call the method on that variable.</span></span>
  
 <span data-ttu-id="81feb-108">L’exemple suivant génère l’avertissement CS1690 :</span><span class="sxs-lookup"><span data-stu-id="81feb-108">The following sample generates CS1690:</span></span>  
  
```csharp  
// CS1690.cs  
using System;

class WarningCS1690 : MarshalByRefObject
{
    int i = 5;

    public static void Main()
    {
        AppDomain domain = AppDomain.CreateDomain("MyDomain");
        Type t = typeof(WarningCS1690);
        WarningCS1690 e = (WarningCS1690)domain.CreateInstanceAndUnwrap(t.Assembly.FullName,t.FullName);

        e.i.ToString(); // CS1690

        // OK  
        int i = e.i;
        i.ToString();
        e.i = i;
    }
}
```
