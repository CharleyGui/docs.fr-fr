## <a name="example"></a><span data-ttu-id="e66af-111">Exemple</span><span class="sxs-lookup"><span data-stu-id="e66af-111">Example</span></span>  

 <span data-ttu-id="e66af-112">L’exemple suivant génère l’erreur CS0686 :</span><span class="sxs-lookup"><span data-stu-id="e66af-112">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="e66af-113">Exemple</span><span class="sxs-lookup"><span data-stu-id="e66af-113">Example</span></span>  

 <span data-ttu-id="e66af-114">Cette erreur peut également se produire quand vous déclarez des événements.</span><span class="sxs-lookup"><span data-stu-id="e66af-114">This error can also occur when declaring events.</span></span>  <span data-ttu-id="e66af-115">La construction d’événements génère automatiquement les méthodes `add_event` et `remove_event`, qui sont susceptibles d’être en conflit avec les méthodes portant le même nom dans une interface, comme dans l’exemple suivant :</span><span class="sxs-lookup"><span data-stu-id="e66af-115">The event construct automatically generates the `add_event` and `remove_event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }  
}  
  
// Correct (using explicit interface implementation):  
class D : I  
{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```
