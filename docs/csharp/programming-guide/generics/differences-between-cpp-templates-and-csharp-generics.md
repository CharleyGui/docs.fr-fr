<span data-ttu-id="10c86-103">Les génériques C# et les modèles C++ sont des fonctionnalités linguistiques qui prennent en charge les types paramétrables.</span><span class="sxs-lookup"><span data-stu-id="10c86-103">C# Generics and C++ templates are both language features that provide support for parameterized types.</span></span> <span data-ttu-id="10c86-104">Toutefois, il existe de nombreuses différences entre les deux.</span><span class="sxs-lookup"><span data-stu-id="10c86-104">However, there are many differences between the two.</span></span> <span data-ttu-id="10c86-105">Au niveau de la syntaxe, les génériques C# constituent une approche plus simple des types paramétrables, sans la complexité des modèles C++.</span><span class="sxs-lookup"><span data-stu-id="10c86-105">At the syntax level, C# generics are a simpler approach to parameterized types without the complexity of C++ templates.</span></span> <span data-ttu-id="10c86-106">De plus, C# n’essaie pas de fournir toutes les fonctionnalités fournies par les modèles C++.</span><span class="sxs-lookup"><span data-stu-id="10c86-106">In addition, C# does not attempt to provide all of the functionality that C++ templates provide.</span></span> <span data-ttu-id="10c86-107">Au niveau de l’implémentation, la principale différence est que les substitutions de type générique C# sont effectuées au moment de l’exécution. Ainsi, les informations de type générique sont conservées pour les objets instanciés.</span><span class="sxs-lookup"><span data-stu-id="10c86-107">At the implementation level, the primary difference is that C# generic type substitutions are performed at runtime and generic type information is thereby preserved for instantiated objects.</span></span> <span data-ttu-id="10c86-108">Pour plus d’informations, consultez [Génériques dans le runtime](./generics-in-the-run-time.md).</span><span class="sxs-lookup"><span data-stu-id="10c86-108">For more information, see [Generics in the Run Time](./generics-in-the-run-time.md).</span></span>  
  
 <span data-ttu-id="10c86-109">Voici les principales différences entre les modèles C++ et les génériques C# :</span><span class="sxs-lookup"><span data-stu-id="10c86-109">The following are the key differences between C# Generics and C++ templates:</span></span>  
  
- <span data-ttu-id="10c86-110">Les génériques C# n’offrent pas le même niveau de flexibilité que les modèles C++.</span><span class="sxs-lookup"><span data-stu-id="10c86-110">C# generics do not provide the same amount of flexibility as C++ templates.</span></span> <span data-ttu-id="10c86-111">Par exemple, il n’est pas possible d’appeler des opérateurs arithmétiques dans une classe générique C#, bien qu’il soit possible d’appeler des opérateurs définis par l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="10c86-111">For example, it is not possible to call arithmetic operators in a C# generic class, although it is possible to call user defined operators.</span></span>  
  
- <span data-ttu-id="10c86-112">C# n’autorise pas les paramètres de modèle sans type, tels que `template C<int i> {}`.</span><span class="sxs-lookup"><span data-stu-id="10c86-112">C# does not allow non-type template parameters, such as `template C<int i> {}`.</span></span>  
  
- <span data-ttu-id="10c86-113">C# ne prend pas en charge la spécialisation explicite (une implémentation personnalisée d’un modèle pour un type spécifique).</span><span class="sxs-lookup"><span data-stu-id="10c86-113">C# does not support explicit specialization; that is, a custom implementation of a template for a specific type.</span></span>  
  
- <span data-ttu-id="10c86-114">C# ne prend pas en charge la spécialisation partielle (une implémentation personnalisée pour un sous-ensemble des arguments de type).</span><span class="sxs-lookup"><span data-stu-id="10c86-114">C# does not support partial specialization: a custom implementation for a subset of the type arguments.</span></span>  
  
- <span data-ttu-id="10c86-115">C# n’autorise pas l’utilisation du paramètre de type comme classe de base pour le type générique.</span><span class="sxs-lookup"><span data-stu-id="10c86-115">C# does not allow the type parameter to be used as the base class for the generic type.</span></span>  
  
- <span data-ttu-id="10c86-116">C# n’autorise pas les paramètres de type à avoir des types par défaut.</span><span class="sxs-lookup"><span data-stu-id="10c86-116">C# does not allow type parameters to have default types.</span></span>  
  
- <span data-ttu-id="10c86-117">En C#, un paramètre de type générique ne peut pas être lui-même un générique, bien que les types construits puissent être utilisés comme des génériques.</span><span class="sxs-lookup"><span data-stu-id="10c86-117">In C#, a generic type parameter cannot itself be a generic, although constructed types can be used as generics.</span></span> <span data-ttu-id="10c86-118">C++ autorise les paramètres de modèle.</span><span class="sxs-lookup"><span data-stu-id="10c86-118">C++ does allow template parameters.</span></span>  
  
- <span data-ttu-id="10c86-119">C++ autorise le code qui peut ne pas être valide pour tous les paramètres de type dans le modèle, qui est ensuite vérifié pour identifier le type spécifique utilisé comme paramètre de type.</span><span class="sxs-lookup"><span data-stu-id="10c86-119">C++ allows code that might not be valid for all type parameters in the template, which is then checked for the specific type used as the type parameter.</span></span> <span data-ttu-id="10c86-120">C# exige que le code dans une classe soit écrit de telle sorte qu’il fonctionne avec tout type qui satisfait aux contraintes.</span><span class="sxs-lookup"><span data-stu-id="10c86-120">C# requires code in a class to be written in such a way that it will work with any type that satisfies the constraints.</span></span> <span data-ttu-id="10c86-121">Par exemple, en C++ il est possible d’écrire une fonction qui utilise les opérateurs arithmétiques `+` et `-` sur des objets du paramètre de type, ce qui générera une erreur au moment de l’instanciation du modèle avec un type qui ne prend pas en charge ces opérateurs.</span><span class="sxs-lookup"><span data-stu-id="10c86-121">For example, in C++ it is possible to write a function that uses the arithmetic operators `+` and `-` on objects of the type parameter, which will produce an error at the time of instantiation of the template with a type that does not support these operators.</span></span> <span data-ttu-id="10c86-122">Ceci n’est pas autorisé par C#. Les seules constructions de langage autorisées sont celles qui peuvent être déduites des contraintes.</span><span class="sxs-lookup"><span data-stu-id="10c86-122">C# disallows this; the only language constructs allowed are those that can be deduced from the constraints.</span></span>
