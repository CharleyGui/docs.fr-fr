<span data-ttu-id="b669e-103">Une classe [statique](../../language-reference/keywords/static.md) est fondamentalement identique à une classe non statique, à une différence près : une classe statique ne peut pas être instanciée.</span><span class="sxs-lookup"><span data-stu-id="b669e-103">A [static](../../language-reference/keywords/static.md) class is basically the same as a non-static class, but there is one difference: a static class cannot be instantiated.</span></span> <span data-ttu-id="b669e-104">En d’autres termes, vous ne pouvez pas utiliser l’opérateur [new](../../language-reference/operators/new-operator.md) pour créer une variable du type classe.</span><span class="sxs-lookup"><span data-stu-id="b669e-104">In other words, you cannot use the [new](../../language-reference/operators/new-operator.md) operator to create a variable of the class type.</span></span> <span data-ttu-id="b669e-105">Étant donné qu’il n’y a aucune variable d’instance, vous accédez aux membres d’une classe statique en utilisant le nom de classe lui-même.</span><span class="sxs-lookup"><span data-stu-id="b669e-105">Because there is no instance variable, you access the members of a static class by using the class name itself.</span></span> <span data-ttu-id="b669e-106">Par exemple, si vous avez une classe statique nommée `UtilityClass` qui a une méthode statique publique nommée `MethodA`, vous appelez la méthode comme illustré dans l’exemple suivant :</span><span class="sxs-lookup"><span data-stu-id="b669e-106">For example, if you have a static class that is named `UtilityClass` that has a public static method named `MethodA`, you call the method as shown in the following example:</span></span>

```csharp  
UtilityClass.MethodA();  
```  
  
 <span data-ttu-id="b669e-107">Une classe statique peut servir de conteneur pratique pour des ensembles de méthodes qui opèrent simplement sur des paramètres d’entrée et n’ont pas à obtenir ou définir de champs d’instance internes.</span><span class="sxs-lookup"><span data-stu-id="b669e-107">A static class can be used as a convenient container for sets of methods that just operate on input parameters and do not have to get or set any internal instance fields.</span></span> <span data-ttu-id="b669e-108">Par exemple, dans la bibliothèque de classes .NET, la <xref:System.Math?displayProperty=nameWithType> classe statique contient des méthodes qui effectuent des opérations mathématiques, sans qu’il soit nécessaire de stocker ou de récupérer des données qui sont uniques à une instance particulière de la <xref:System.Math> classe.</span><span class="sxs-lookup"><span data-stu-id="b669e-108">For example, in the .NET Class Library, the static <xref:System.Math?displayProperty=nameWithType> class contains methods that perform mathematical operations, without any requirement to store or retrieve data that is unique to a particular instance of the <xref:System.Math> class.</span></span> <span data-ttu-id="b669e-109">Autrement dit, vous appliquez les membres de la classe en spécifiant le nom de la classe et le nom de la méthode, comme illustré dans l’exemple suivant.</span><span class="sxs-lookup"><span data-stu-id="b669e-109">That is, you apply the members of the class by specifying the class name and the method name, as shown in the following example.</span></span>  
  
```csharp  
double dub = -3.14;  
Console.WriteLine(Math.Abs(dub));  
Console.WriteLine(Math.Floor(dub));  
Console.WriteLine(Math.Round(Math.Abs(dub)));  
  
// Output:  
// 3.14  
// -4  
// 3  
```  
  
 <span data-ttu-id="b669e-110">Comme c’est le cas avec tous les types de classe, les informations de type pour une classe statique sont chargées par le Runtime .NET lorsque le programme qui référence la classe est chargé.</span><span class="sxs-lookup"><span data-stu-id="b669e-110">As is the case with all class types, the type information for a static class is loaded by the .NET runtime when the program that references the class is loaded.</span></span> <span data-ttu-id="b669e-111">Le programme ne peut pas spécifier exactement quand la classe est chargée.</span><span class="sxs-lookup"><span data-stu-id="b669e-111">The program cannot specify exactly when the class is loaded.</span></span> <span data-ttu-id="b669e-112">Toutefois, il est garanti qu’elle sera chargée, que ses champs seront initialisés et que son constructeur statique sera appelé avant que la classe soit référencée pour la première fois dans votre programme.</span><span class="sxs-lookup"><span data-stu-id="b669e-112">However, it is guaranteed to be loaded and to have its fields initialized and its static constructor called before the class is referenced for the first time in your program.</span></span> <span data-ttu-id="b669e-113">Un constructeur statique est appelé une seule fois et une classe statique reste en mémoire pendant la durée de vie du domaine d’application dans lequel votre programme réside.</span><span class="sxs-lookup"><span data-stu-id="b669e-113">A static constructor is only called one time, and a static class remains in memory for the lifetime of the application domain in which your program resides.</span></span>

 <span data-ttu-id="b669e-115">La liste suivante fournit les fonctionnalités principales d’une classe statique :</span><span class="sxs-lookup"><span data-stu-id="b669e-115">The following list provides the main features of a static class:</span></span>  
  
- <span data-ttu-id="b669e-116">Elle contient uniquement des membres statiques.</span><span class="sxs-lookup"><span data-stu-id="b669e-116">Contains only static members.</span></span>  
  
- <span data-ttu-id="b669e-117">Elle ne peut pas être instanciée.</span><span class="sxs-lookup"><span data-stu-id="b669e-117">Cannot be instantiated.</span></span>  
  
- <span data-ttu-id="b669e-118">Elle est verrouillée (sealed).</span><span class="sxs-lookup"><span data-stu-id="b669e-118">Is sealed.</span></span>

 <span data-ttu-id="b669e-120">La création d’une classe statique est par conséquent très semblable à la création d’une classe contenant uniquement des membres statiques et un constructeur privé.</span><span class="sxs-lookup"><span data-stu-id="b669e-120">Creating a static class is therefore basically the same as creating a class that contains only static members and a private constructor.</span></span> <span data-ttu-id="b669e-121">Un constructeur privé empêche la classe d’être instanciée.</span><span class="sxs-lookup"><span data-stu-id="b669e-121">A private constructor prevents the class from being instantiated.</span></span> <span data-ttu-id="b669e-122">L’avantage de l’utilisation d’une classe statique est que le compilateur peut vérifier qu’aucun membre d’instance n’a été ajouté par erreur.</span><span class="sxs-lookup"><span data-stu-id="b669e-122">The advantage of using a static class is that the compiler can check to make sure that no instance members are accidentally added.</span></span> <span data-ttu-id="b669e-123">Le compilateur garantit que les instances de cette classe ne peuvent pas être créées.</span><span class="sxs-lookup"><span data-stu-id="b669e-123">The compiler will guarantee that instances of this class cannot be created.</span></span>  
  
 <span data-ttu-id="b669e-124">Les classes statiques sont scellées (sealed) et ne peuvent par conséquent pas être héritées.</span><span class="sxs-lookup"><span data-stu-id="b669e-124">Static classes are sealed and therefore cannot be inherited.</span></span> <span data-ttu-id="b669e-125">Elles ne peuvent hériter d’aucune classe à part <xref:System.Object>.</span><span class="sxs-lookup"><span data-stu-id="b669e-125">They cannot inherit from any class except <xref:System.Object>.</span></span> <span data-ttu-id="b669e-126">Les classes static ne peuvent pas contenir un constructeur d’instance.</span><span class="sxs-lookup"><span data-stu-id="b669e-126">Static classes cannot contain an instance constructor.</span></span> <span data-ttu-id="b669e-127">Toutefois, ils peuvent contenir un constructeur statique.</span><span class="sxs-lookup"><span data-stu-id="b669e-127">However, they can contain a static constructor.</span></span> <span data-ttu-id="b669e-128">Les classes non statiques doivent également définir un constructeur statique si la classe contient des membres statiques qui requièrent une initialisation non triviale.</span><span class="sxs-lookup"><span data-stu-id="b669e-128">Non-static classes should also define a static constructor if the class contains static members that require non-trivial initialization.</span></span> <span data-ttu-id="b669e-129">Pour plus d’informations, consultez [Constructeurs statiques](./static-constructors.md).</span><span class="sxs-lookup"><span data-stu-id="b669e-129">For more information, see [Static Constructors](./static-constructors.md).</span></span>

 <span data-ttu-id="b669e-133">Une classe non statique peut contenir des méthodes, des champs, des propriétés ou des événements statiques.</span><span class="sxs-lookup"><span data-stu-id="b669e-133">A non-static class can contain static methods, fields, properties, or events.</span></span> <span data-ttu-id="b669e-134">Le membre statique peut être appelé sur une classe même quand aucune instance de la classe n’a été créée.</span><span class="sxs-lookup"><span data-stu-id="b669e-134">The static member is callable on a class even when no instance of the class has been created.</span></span> <span data-ttu-id="b669e-135">Le membre statique est toujours accessible par le nom de la classe, et non par le nom de l’instance.</span><span class="sxs-lookup"><span data-stu-id="b669e-135">The static member is always accessed by the class name, not the instance name.</span></span> <span data-ttu-id="b669e-136">Une seule copie d’un membre statique existe, quel que soit le nombre d’instances de la classe qui ont été créées.</span><span class="sxs-lookup"><span data-stu-id="b669e-136">Only one copy of a static member exists, regardless of how many instances of the class are created.</span></span> <span data-ttu-id="b669e-137">Les méthodes et les propriétés statiques ne peuvent pas accéder à des champs et des événements non statiques dans leur type conteneur, et elles ne peuvent pas accéder à une variable d’instance de n’importe quel objet, sauf si elle est explicitement passée dans un paramètre de méthode.</span><span class="sxs-lookup"><span data-stu-id="b669e-137">Static methods and properties cannot access non-static fields and events in their containing type, and they cannot access an instance variable of any object unless it's explicitly passed in a method parameter.</span></span>
