- <span data-ttu-id="5bd15-109">Création de méthodes d’opérateur de requête standard telles que <xref:System.Linq.Enumerable.Where%2A> et <xref:System.Linq.Enumerable.Select%2A> qui étendent un type, pour activer l’interrogation LINQ personnalisée de ce type.</span><span class="sxs-lookup"><span data-stu-id="5bd15-109">Creating standard query operator methods such as <xref:System.Linq.Enumerable.Where%2A> and <xref:System.Linq.Enumerable.Select%2A> that extend a type, to enable custom LINQ querying of that type.</span></span>  
  
- <span data-ttu-id="5bd15-110">Création d'un fournisseur pour votre source de données qui implémente l'interface <xref:System.Linq.IQueryable%601>.</span><span class="sxs-lookup"><span data-stu-id="5bd15-110">Creating a provider for your data source that implements the <xref:System.Linq.IQueryable%601> interface.</span></span> <span data-ttu-id="5bd15-111">Un fournisseur qui implémente cette interface reçoit des requêtes LINQ sous la forme d’arborescences d’expression, qu’il peut exécuter de façon personnalisée, par exemple à distance.</span><span class="sxs-lookup"><span data-stu-id="5bd15-111">A provider that implements this interface receives LINQ queries in the form of expression trees, which it can execute in a custom way, for example remotely.</span></span>

## <a name="iqueryable-linq-providers"></a><span data-ttu-id="5bd15-124">Fournisseurs LINQ IQueryable</span><span class="sxs-lookup"><span data-stu-id="5bd15-124">IQueryable LINQ Providers</span></span>

 <span data-ttu-id="5bd15-127">Un fournisseur `IQueryable` moins complexe peut interagir avec une méthode unique d'un service Web.</span><span class="sxs-lookup"><span data-stu-id="5bd15-127">A less complex `IQueryable` provider might interface with a single method of a Web service.</span></span> <span data-ttu-id="5bd15-128">Ce type de fournisseur est très particulier, car il attend des informations spécifiques dans les requêtes qu'il gère.</span><span class="sxs-lookup"><span data-stu-id="5bd15-128">This type of provider is very specific because it expects specific information in the queries that it handles.</span></span> <span data-ttu-id="5bd15-129">Il se caractérise par un système de type fermé, exposant peut-être un type de résultat unique.</span><span class="sxs-lookup"><span data-stu-id="5bd15-129">It has a closed type system, perhaps exposing a single result type.</span></span> <span data-ttu-id="5bd15-130">La plus grande part de l'exécution de la requête est effectuée localement, par exemple à l'aide des implémentations <xref:System.Linq.Enumerable> des opérateurs de requête standard.</span><span class="sxs-lookup"><span data-stu-id="5bd15-130">Most of the execution of the query occurs locally, for example by using the <xref:System.Linq.Enumerable> implementations of the standard query operators.</span></span> <span data-ttu-id="5bd15-131">Un fournisseur moins complexe peut examiner une seule expression d'appel de méthode dans l'arborescence d'expression qui représente la requête, et la logique restante de la requête doit être gérée ailleurs.</span><span class="sxs-lookup"><span data-stu-id="5bd15-131">A less complex provider might examine only one method call expression in the expression tree that represents the query, and let the remaining logic of the query be handled elsewhere.</span></span>
