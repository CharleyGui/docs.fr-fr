# <a name="compiler-warning-level-2-cs0440"></a><span data-ttu-id="dc766-103">Avertissement du compilateur (niveau 2) CS0440</span><span class="sxs-lookup"><span data-stu-id="dc766-103">Compiler Warning (level 2) CS0440</span></span>
<span data-ttu-id="dc766-104">La définition d'un alias nommé 'global' n'est pas très judicieuse dans la mesure où 'global::' fait toujours référence à l'espace de noms global et non à un alias</span><span class="sxs-lookup"><span data-stu-id="dc766-104">Defining an alias named 'global' is ill-advised since 'global::' always references the global namespace and not an alias</span></span>  
  
 <span data-ttu-id="dc766-105">Cet avertissement est émis quand vous définissez un alias nommé global.</span><span class="sxs-lookup"><span data-stu-id="dc766-105">This warning is issued when you define an alias named global.</span></span>  
  
## <a name="example"></a><span data-ttu-id="dc766-106">Exemple</span><span class="sxs-lookup"><span data-stu-id="dc766-106">Example</span></span>  
 <span data-ttu-id="dc766-107">L’exemple suivant génère l’avertissement CS0440 :</span><span class="sxs-lookup"><span data-stu-id="dc766-107">The following example generates CS0440:</span></span>  
  
```csharp  
// CS0440.cs  
// Compile with: /W:1  
  
using global = MyClass;   // CS0440  
class MyClass  
{  
    static void Main()  
    {  
        // Note how global refers to the global namespace  
        // even though it is redefined above.  
        global::System.Console.WriteLine();  
    }  
}  
```
