# <a name="compiler-warning-level-4-cs0649"></a><span data-ttu-id="92397-103">Avertissement du compilateur (niveau 4) CS0649</span><span class="sxs-lookup"><span data-stu-id="92397-103">Compiler Warning (level 4) CS0649</span></span>
<span data-ttu-id="92397-104">Le champ 'field' n’est jamais assigné et aura toujours sa valeur par défaut 'value'</span><span class="sxs-lookup"><span data-stu-id="92397-104">Field 'field' is never assigned to, and will always have its default value 'value'</span></span>  
  
 <span data-ttu-id="92397-105">Le compilateur a détecté une déclaration de champ privé ou interne non initialisée à laquelle une valeur n’est jamais assignée.</span><span class="sxs-lookup"><span data-stu-id="92397-105">The compiler detected an uninitialized private or internal field declaration that is never assigned a value.</span></span>  
  
 <span data-ttu-id="92397-106">L’exemple suivant génère l’avertissement CS0649 :</span><span class="sxs-lookup"><span data-stu-id="92397-106">The following sample generates CS0649:</span></span>  
  
```csharp  
// CS0649.cs  
// compile with: /W:4  
using System.Collections;  
  
class MyClass  
{  
   Hashtable table;  // CS0649  
   // You may have intended to initialize the variable to null  
   // Hashtable table = null;  
  
   // Or you may have meant to create an object here  
   // Hashtable table = new Hashtable();  
  
   public void Func(object o, string p)  
   {  
      // Or here  
      // table = new Hashtable();  
      table[p] = o;  
   }  
  
   public static void Main()  
   {  
   }  
}  
```
