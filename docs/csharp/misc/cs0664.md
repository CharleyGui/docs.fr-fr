# <a name="compiler-error-cs0664"></a>Erreur du compilateur CS0664

Impossible de convertir implicitement un littéral de type double en type 'type' ; utilisez un suffixe 'suffixe' pour créer un littéral de ce type  
  
 Une affectation n’a pas pu être effectuée ; utilisez un suffixe pour corriger l’instruction. La documentation de chaque type identifie le suffixe correspondant au type. Pour plus d’informations sur les conversions, consultez [Cast et conversions de types](../programming-guide/types/casting-and-type-conversions.md).  
  
 L’exemple suivant génère l’erreur CS0664 :  
  
```csharp  
// CS0664.cs  
class Example  
{  
    static void Main()  
    {  
        decimal d1 = 1.0;   // CS0664, because 1.0 is interpreted  
                            // as a double.  
  
        // Try the following line instead.  
        decimal d2 = 1.0M;  // The M tells the compiler that 1.0 is a  
                            // decimal.  
        Console.WriteLine(d2);  
    }  
}  
```
