# <a name="compiler-error-cs0505"></a><span data-ttu-id="48c68-103">Erreur du compilateur CS0505</span><span class="sxs-lookup"><span data-stu-id="48c68-103">Compiler Error CS0505</span></span>
<span data-ttu-id="48c68-104">'membre1' : substitution impossible, car 'membre2' n’est pas une fonction</span><span class="sxs-lookup"><span data-stu-id="48c68-104">'member1': cannot override because 'member2' is not a function</span></span>  
  
 <span data-ttu-id="48c68-105">Une déclaration de classe a tenté de substituer un élément qui n’est pas une méthode dans une classe de base.</span><span class="sxs-lookup"><span data-stu-id="48c68-105">A class declaration attempted to override a non-method in a base class.</span></span> <span data-ttu-id="48c68-106">Les substitutions doivent correspondre au type de membre.</span><span class="sxs-lookup"><span data-stu-id="48c68-106">Overrides must match the member type.</span></span> <span data-ttu-id="48c68-107">Si la présence d’une méthode de même nom qu’une méthode contenue dans une classe de base est souhaitée, utilisez [new](../language-reference/keywords/new-modifier.md) (et non [override](../language-reference/keywords/override.md)) dans la déclaration de méthode de la classe de base.</span><span class="sxs-lookup"><span data-stu-id="48c68-107">If a method with the same name as a method in a base class is desired, use [new](../language-reference/keywords/new-modifier.md) (and not [override](../language-reference/keywords/override.md)) on the method declaration in the base class.</span></span>  
  
 <span data-ttu-id="48c68-108">L’exemple suivant génère l’erreur CS0505 :</span><span class="sxs-lookup"><span data-stu-id="48c68-108">The following sample generates CS0505:</span></span>  
  
```csharp  
// CS0505.cs  
// compile with: /target:library  
public class clx  
{  
   public int i;  
}  
  
public class cly : clx  
{  
   public override int i() { return 0; }   // CS0505  
}  
```
