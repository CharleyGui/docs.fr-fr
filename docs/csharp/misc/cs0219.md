# <a name="compiler-warning-level-3-cs0219"></a><span data-ttu-id="e2a2e-102">Avertissement du compilateur (niveau 3) CS0219</span><span class="sxs-lookup"><span data-stu-id="e2a2e-102">Compiler Warning (level 3) CS0219</span></span>
<span data-ttu-id="e2a2e-103">La variable ’variable’ est assignée, mais sa valeur n’est jamais utilisée</span><span class="sxs-lookup"><span data-stu-id="e2a2e-103">The variable 'variable' is assigned but its value is never used</span></span>

 <span data-ttu-id="e2a2e-104">Le compilateur émet un avertissement de niveau trois lorsque vous déclarez et assignez une variable, mais ne l’utilisez pas.</span><span class="sxs-lookup"><span data-stu-id="e2a2e-104">The compiler issues a level-three warning, when you declare and assign a variable, but do not use it.</span></span>

 > [!NOTE]
 > <span data-ttu-id="e2a2e-105">Le compilateur génère cet avertissement uniquement lorsque la valeur de la variable est une constante au moment de la compilation.</span><span class="sxs-lookup"><span data-stu-id="e2a2e-105">The compiler generates this warning only when the variable value is a compile-time constant.</span></span> <span data-ttu-id="e2a2e-106">L’assignation d’un résultat d’expression ou de méthode non constante à une variable locale facilite l’observation de ces expressions dans le débogueur.</span><span class="sxs-lookup"><span data-stu-id="e2a2e-106">Assigning a non-constant expression or method result to a local variable makes it easier to observe those expressions in the debugger.</span></span> <span data-ttu-id="e2a2e-107">Elle rend également le résultat accessible, en empêchant garbage collection alors que cette variable est accessible.</span><span class="sxs-lookup"><span data-stu-id="e2a2e-107">It also makes the result reachable, preventing garbage collection while that variable is reachable.</span></span>
 
 <span data-ttu-id="e2a2e-108">L’exemple suivant montre les cas où et quand l’avertissement n’est pas généré :</span><span class="sxs-lookup"><span data-stu-id="e2a2e-108">The following sample shows the cases when and when not the warning is generated:</span></span>

```csharp
// CS0219.cs
// compile with: /W:3
public class MyClass
{
   public static void Main()
   {
      var interpolated = "Interpolated";
      var a = 0; // CS0219
      int b = GetZero(); // Doesn't generate a warning.
      var c = "Regular string"; // CS0219
      var d = $"Constant interpolated string"; // Doesn't generate a warning.
      var e = $"{interpolated} string"; // Doesn't generate a warning.
   }
   
   private static int GetZero()
   {
       return 0;
   }
}  
```
