# <a name="compiler-error-cs0104"></a><span data-ttu-id="794de-102">Erreur du compilateur CS0104</span><span class="sxs-lookup"><span data-stu-id="794de-102">Compiler Error CS0104</span></span>
<span data-ttu-id="794de-103">'référence' est une référence ambiguë entre 'identificateur' et 'identificateur’</span><span class="sxs-lookup"><span data-stu-id="794de-103">'reference' is an ambiguous reference between 'identifier' and 'identifier'</span></span>  
  
 <span data-ttu-id="794de-104">Votre programme contient des directives [using](../language-reference/keywords/using.md) pour deux espaces de noms et votre code fait référence à un nom qui apparaît dans les deux espaces de noms.</span><span class="sxs-lookup"><span data-stu-id="794de-104">Your program contains [using](../language-reference/keywords/using.md) directives for two namespaces and your code references a name that appears in both namespaces.</span></span>  
  
 <span data-ttu-id="794de-105">L’exemple suivant génère l’erreur CS0104 :</span><span class="sxs-lookup"><span data-stu-id="794de-105">The following sample generates CS0104:</span></span>  
  
```csharp  
// CS0104.cs  
using x;  
using y;  
  
namespace x  
{  
   public class Test  
   {  
   }  
}  
  
namespace y  
{  
   public class Test  
   {  
   }  
}  
  
public class a  
{  
   public static void Main()  
   {  
      Test test = new Test();   // CS0104, is Test in x or y namespace?  
      // try the following line instead  
      // y.Test test = new y.Test();  
   }  
}  
```
