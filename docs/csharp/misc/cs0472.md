# <a name="compiler-warning-level-2-cs0472"></a><span data-ttu-id="4b07c-103">Avertissement du compilateur (niveau 2) CS0472</span><span class="sxs-lookup"><span data-stu-id="4b07c-103">Compiler Warning (level 2) CS0472</span></span>
<span data-ttu-id="4b07c-104">Le résultat de l’expression est toujours 'valeur1', car une valeur de type 'valeur2' n’est jamais égale à 'null' du type 'valeur3'</span><span class="sxs-lookup"><span data-stu-id="4b07c-104">The result of the expression is always 'value1' since a value of type 'value2' is never equal to 'null' of type 'value3'</span></span>

## <a name="example"></a><span data-ttu-id="4b07c-106">Exemple</span><span class="sxs-lookup"><span data-stu-id="4b07c-106">Example</span></span>  
 <span data-ttu-id="4b07c-107">L’exemple suivant génère l’erreur CS0472 :</span><span class="sxs-lookup"><span data-stu-id="4b07c-107">The following sample generates CS0472.</span></span>  
  
```csharp  
public class Test  
{  
    public static int Main()  
    {  
        int i = 5;  
        int counter = 0;  
  
        // Comparison:  
        if (i == null)  // CS0472  
        // To resolve, use a valid value for i.  
            counter++;
        return counter;  
    }  
}  
```
