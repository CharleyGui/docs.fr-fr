# <a name="compiler-error-cs0426"></a><span data-ttu-id="dc588-103">Erreur du compilateur CS0426</span><span class="sxs-lookup"><span data-stu-id="dc588-103">Compiler Error CS0426</span></span>
<span data-ttu-id="dc588-104">Le nom de type 'identificateur' n’existe pas dans le type 'type'</span><span class="sxs-lookup"><span data-stu-id="dc588-104">The type name 'identifier' does not exist in the type 'type'</span></span>  
  
 <span data-ttu-id="dc588-105">Cette erreur se produit lorsque vous faites référence à un type imbriqué dans un autre type, mais que ce type imbriqué n’existe pas.</span><span class="sxs-lookup"><span data-stu-id="dc588-105">This error occurs when you reference a type nested within another type, but no such nested type exists.</span></span> <span data-ttu-id="dc588-106">Cela peut se produire si vous orthographiez mal le nom du type imbriqué.</span><span class="sxs-lookup"><span data-stu-id="dc588-106">This can occur if you mistype the name of the nested type.</span></span> <span data-ttu-id="dc588-107">Vérifiez l’orthographe des noms utilisés et vérifiez que le type englobant comporte le membre attendu.</span><span class="sxs-lookup"><span data-stu-id="dc588-107">Check the spelling of the names used, and verify that the enclosing type has the expected member.</span></span>  
  
 <span data-ttu-id="dc588-108">L’exemple suivant génère l’erreur CS0426 car la classe C n’a aucun type imbriqué A :</span><span class="sxs-lookup"><span data-stu-id="dc588-108">The following sample generates CS0426 because class C has no nested type A:</span></span>  
  
```csharp  
// CS0426.cs  
  
class C  
{  
    // No nested types are declared.
}  
  
class D  
{  
   public static void Main()  
   {  
       C c = new C();  
       // Attempt to reference a nested type A:  
       C.A a; // CS0426 because no such type C.A  
   }  
}  
```
