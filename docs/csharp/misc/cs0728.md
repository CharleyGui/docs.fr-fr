 <span data-ttu-id="77e4c-106">Il existe plusieurs scénarios où les blocs `using` ou `lock` provoquent une fuite temporaire de ressources.</span><span class="sxs-lookup"><span data-stu-id="77e4c-106">There are several scenarios where `using` or `lock` blocks will result in a temporary leak of resources.</span></span> <span data-ttu-id="77e4c-107">En voici un exemple :</span><span class="sxs-lookup"><span data-stu-id="77e4c-107">Here is one example:</span></span>  
  
 `thisType f = null;`

 `f = new thisType();`  
  
 `...`  
  
 `}`  
  
 <span data-ttu-id="77e4c-108">Dans ce cas, la valeur d’origine (null) de la variable `thisType` est supprimée à la fin de l’exécution du bloc `using` , mais l’objet `thisType` créé dans le bloc ne l’est pas, même s’il fait finalement l’objet d’une récupération de mémoire.</span><span class="sxs-lookup"><span data-stu-id="77e4c-108">In this case, the original value, such as null, of the variable `thisType` will be disposed of when the `using` block finishes executing, but the `thisType` object created inside the block will not be, although it will eventually get garbage collected.</span></span>

## <a name="example"></a><span data-ttu-id="77e4c-111">Exemple</span><span class="sxs-lookup"><span data-stu-id="77e4c-111">Example</span></span>  

 <span data-ttu-id="77e4c-112">Le code suivant génère l’erreur CS0728.</span><span class="sxs-lookup"><span data-stu-id="77e4c-112">The following code will generate warning CS0728.</span></span>

```csharp
{  
    public void Dispose() {  }  
}

{  
    public static void dummy()  
    {  
        ValidBase vb = null;

}  
```
