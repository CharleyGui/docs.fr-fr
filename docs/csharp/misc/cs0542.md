# <a name="compiler-error-cs0542"></a><span data-ttu-id="7c766-103">Erreur du compilateur CS0542</span><span class="sxs-lookup"><span data-stu-id="7c766-103">Compiler Error CS0542</span></span>

<span data-ttu-id="7c766-104">'user-defined type' : les noms de membres doivent être différents de leur type englobant</span><span class="sxs-lookup"><span data-stu-id="7c766-104">'user-defined type' : member names cannot be the same as their enclosing type</span></span>  
  
 <span data-ttu-id="7c766-105">Les membres d’une classe ou d’un struct ne peuvent pas porter le même nom que la classe ou le struct, sauf s’ils sont des constructeurs.</span><span class="sxs-lookup"><span data-stu-id="7c766-105">The members of a class or struct cannot have the same name as the class or struct, unless the member is a constructor.</span></span>  
  
 <span data-ttu-id="7c766-106">L’exemple suivant génère l’erreur CS0542 :</span><span class="sxs-lookup"><span data-stu-id="7c766-106">The following sample generates CS0542:</span></span>  
  
```csharp  
// CS0542.cs  
class C  
{  
    public int C;  
}  
```  
  
 <span data-ttu-id="7c766-107">Cette erreur peut se produire si vous placez par inadvertance un type de retour sur un constructeur, ce qui en fait une méthode ordinaire.</span><span class="sxs-lookup"><span data-stu-id="7c766-107">This error might be caused if you inadvertently put a return type on a constructor, which in effect makes it into an ordinary method.</span></span> <span data-ttu-id="7c766-108">L’exemple suivant génère l’erreur CS0542 parce que `F` est une méthode, et non pas un constructeur, puisqu’elle a un type de retour :</span><span class="sxs-lookup"><span data-stu-id="7c766-108">The following example generates CS0542 because `F` is a method, not a constructor, because it has a return type:</span></span>  
  
```csharp  
// CS0542.cs  
class F  
{  
   // Remove void from F() to resolve the problem.  
   void F()   // CS0542, same name as the class  
   {  
   }  
}  
  
class MyClass  
{  
   public static void Main()  
   {  
   }  
}  
```  
  
 <span data-ttu-id="7c766-109">Si votre classe est nommée 'Item' et a un indexeur déclaré en tant que `this`, cette erreur peut se produire.</span><span class="sxs-lookup"><span data-stu-id="7c766-109">If your class is named 'Item' and has an indexer declared as `this`, you may get this error.</span></span> <span data-ttu-id="7c766-110">Un indexeur par défaut porte le nom 'Item' dans le code émis, ce qui crée le conflit.</span><span class="sxs-lookup"><span data-stu-id="7c766-110">A default indexer is given the name 'Item' in the emitted code, creating the conflict.</span></span>  
  
```csharp  
// CS0542b.cs  
class Item  
{  
   public int this[int i]  // CS0542  
   {  
      get  
      {  
         return 0;  
      }  
   }  
}  
  
class CMain  
{  
   public static void Main()  
   {  
   }  
}  
```
