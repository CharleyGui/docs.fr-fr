# <a name="weak-event-patterns"></a><span data-ttu-id="77a34-103">Modèles d'événement faible</span><span class="sxs-lookup"><span data-stu-id="77a34-103">Weak Event Patterns</span></span>
<span data-ttu-id="77a34-104">Dans les applications, il est possible que les gestionnaires qui sont attachés à des sources d’événements ne soient pas détruits avec l’objet d’écouteur qui a attaché le gestionnaire à la source.</span><span class="sxs-lookup"><span data-stu-id="77a34-104">In applications, it is possible that handlers that are attached to event sources will not be destroyed in coordination with the listener object that attached the handler to the source.</span></span> <span data-ttu-id="77a34-105">Cette situation peut entraîner des fuites de mémoire.</span><span class="sxs-lookup"><span data-stu-id="77a34-105">This situation can lead to memory leaks.</span></span> [!INCLUDE[TLA#tla_winclient](../../../../includes/tlasharptla-winclient-md.md)]<span data-ttu-id="77a34-106">introduit un modèle de conception qui peut être utilisé pour résoudre ce problème, en fournissant une classe de gestionnaire dédiée pour des événements particuliers et en implémentant une interface sur les écouteurs de cet événement.</span><span class="sxs-lookup"><span data-stu-id="77a34-106">introduces a design pattern that can be used to address this issue, by providing a dedicated manager class for particular events and implementing an interface on listeners for that event.</span></span> <span data-ttu-id="77a34-107">Ce modèle de conception est connu sous le nom de *modèle d’événement faible*.</span><span class="sxs-lookup"><span data-stu-id="77a34-107">This design pattern is known as the *weak event pattern*.</span></span>

 <span data-ttu-id="77a34-109">L’écoute des événements peut entraîner des fuites de mémoire.</span><span class="sxs-lookup"><span data-stu-id="77a34-109">Listening for events can lead to memory leaks.</span></span> <span data-ttu-id="77a34-110">La technique classique pour écouter un événement consiste à utiliser la syntaxe spécifique au langage qui attache un gestionnaire à un événement sur une source.</span><span class="sxs-lookup"><span data-stu-id="77a34-110">The typical technique for listening to an event is to use the language-specific syntax that attaches a handler to an event on a source.</span></span> <span data-ttu-id="77a34-111">Par exemple, en C#, cette syntaxe est la suivante : `source.SomeEvent += new SomeEventHandler(MyEventHandler)` .</span><span class="sxs-lookup"><span data-stu-id="77a34-111">For example, in C#, that syntax is: `source.SomeEvent += new SomeEventHandler(MyEventHandler)`.</span></span>
