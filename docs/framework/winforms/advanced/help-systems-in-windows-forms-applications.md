# <a name="help-systems-in-windows-forms-applications"></a>Systèmes d'aide dans les applications Windows Forms
L’un des courtesies les plus importants, en tant que développeur d’applications, peut fournir à vos utilisateurs un système d’aide compétent. C’est là qu’elles s’activent lorsqu’elles deviennent confuses ou détournées. Il est facile de fournir un système d’aide dans une application Windows à l’aide du [composant HelpProvider](../controls/helpprovider-component-windows-forms.md).  
  
## <a name="different-types-of-help"></a>Différents types d’aide  
 Le composant Windows Forms <xref:System.Windows.Forms.HelpProvider> sert à associer un fichier d'aide HTML Help 1.x (fichier .chm généré par HTML Help Workshop ou fichier .htm) à votre application Windows. Le composant <xref:System.Windows.Forms.HelpProvider> peut être utilisé pour fournir une aide contextuelle pour les contrôles sur des Windows Forms ou des contrôles spécifiques. En outre, le composant <xref:System.Windows.Forms.HelpProvider> peut ouvrir un fichier d’aide dans des zones spécifiques, telles que la page principale d’une table des matières, un index ou une fonction de recherche. Pour obtenir des informations générales sur le composant <xref:System.Windows.Forms.HelpProvider>, consultez [vue d’ensemble du composant HelpProvider](../controls/helpprovider-component-overview-windows-forms.md). Pour plus d’informations sur l’utilisation du composant <xref:System.Windows.Forms.HelpProvider> pour afficher l’aide contextuelle sur Windows Forms, consultez [Comment : afficher l’aide contextuelle](how-to-display-pop-up-help.md). Pour plus d’informations sur l’utilisation du composant <xref:System.Windows.Forms.ToolTip> pour afficher l’aide spécifique aux contrôles, consultez l’aide du [contrôle à l’aide d’info-bulles](control-help-using-tooltips.md).  
  
 Vous pouvez générer des fichiers HTML Help 1. x avec HTML Help Workshop. Pour plus d’informations sur l’aide HTML, consultez « HTML Help Workshop » ou les autres rubriques « HTML Help » dans MSDN.
