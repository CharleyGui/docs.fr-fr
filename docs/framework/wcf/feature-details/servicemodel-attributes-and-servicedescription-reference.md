# <a name="servicemodel-attributes-and-servicedescription-reference"></a><span data-ttu-id="89147-102">Attributs ServiceModel et référence ServiceDescription</span><span class="sxs-lookup"><span data-stu-id="89147-102">ServiceModel Attributes and ServiceDescription Reference</span></span>
<span data-ttu-id="89147-103">L' *arborescence de description* est la hiérarchie de types (à partir de la <xref:System.ServiceModel.Description.ServiceDescription?displayProperty=nameWithType> classe) qui décrivent ensemble chaque aspect d’un service.</span><span class="sxs-lookup"><span data-stu-id="89147-103">The *description tree* is the hierarchy of types (starting with the <xref:System.ServiceModel.Description.ServiceDescription?displayProperty=nameWithType> class) that together describe every aspect of a service.</span></span> <span data-ttu-id="89147-104">Windows Communication Foundation (WCF) utilise une arborescence de description pour créer un service d’exécution de service valide, pour publier Web Services Description Language (WSDL), le langage XSD (XML Schema Definition) et les assertions de stratégie (métadonnées) sur le service que les clients peuvent utiliser pour se connecter au service et pour générer différentes représentations du code et des fichiers de configuration des valeurs de l’arborescence de description.</span><span class="sxs-lookup"><span data-stu-id="89147-104">Windows Communication Foundation (WCF) uses a description tree to build a valid service runtime, to publish Web Services Description Language (WSDL), XML Schema definition language (XSD), and policy assertions (metadata) about the service that clients can use to connect to and use the service, and to generate various code and configuration file representations of the description tree values.</span></span>  
  
 <span data-ttu-id="89147-105">Cette rubrique décrit comment obtenir les propriétés relatives au contrat à partir du contrat de service, et la manière dont elles sont implémentées et ajoutées à l’arborescence de description.</span><span class="sxs-lookup"><span data-stu-id="89147-105">This topic describes how contract-related properties are obtained from the service contract, and how they are implemented and added to the description tree.</span></span> <span data-ttu-id="89147-106">Dans certains cas, les valeurs d’attribut sont converties en propriétés de comportement et le comportement est ensuite inséré dans l’arborescence de description.</span><span class="sxs-lookup"><span data-stu-id="89147-106">In some cases, attribute values are converted into behavior properties and behavior is then inserted into the description tree.</span></span> <span data-ttu-id="89147-107">Pour plus d’informations sur la conversion des valeurs de l’arborescence de description en métadonnées, consultez [ServiceDescription et référence WSDL](servicedescription-and-wsdl-reference.md).</span><span class="sxs-lookup"><span data-stu-id="89147-107">For more information about how the description tree values are converted into metadata, see [ServiceDescription and WSDL Reference](servicedescription-and-wsdl-reference.md).</span></span>  
  
## <a name="mapping-operations-to-the-description-tree"></a><span data-ttu-id="89147-108">Mappage des opérations à l’arborescence de description</span><span class="sxs-lookup"><span data-stu-id="89147-108">Mapping Operations to the Description Tree</span></span>  
 <span data-ttu-id="89147-109">Dans les applications WCF, les contrats de service sont modélisés par des interfaces (ou classes) qui utilisent des attributs pour marquer l’interface ou la classe et ses méthodes comme un regroupement d’opérations.</span><span class="sxs-lookup"><span data-stu-id="89147-109">In WCF applications, service contracts are modeled by interfaces (or classes) that use attributes to mark the interface or class and its methods as a grouping of operations.</span></span> <span data-ttu-id="89147-110">Lorsqu’une classe <xref:System.ServiceModel.ServiceHost> est ouverte, les implémentations et les contrats de service sont pris en compte dans, et fusionnées avec, les informations de configuration dans une arborescence de description.</span><span class="sxs-lookup"><span data-stu-id="89147-110">When a <xref:System.ServiceModel.ServiceHost> class is opened, any service contracts and implementations are reflected over and merged with configuration information into a description tree.</span></span>
