## <a name="state-management-and-process-recycling"></a><span data-ttu-id="3a88f-115">Gestion des états et recyclage de processus</span><span class="sxs-lookup"><span data-stu-id="3a88f-115">State Management and Process Recycling</span></span>

## <a name="optimizing-performance-in-middle-tier-scenarios"></a><span data-ttu-id="3a88f-122">Optimisation des performances dans les scénarios de couche intermédiaire</span><span class="sxs-lookup"><span data-stu-id="3a88f-122">Optimizing Performance in Middle-Tier Scenarios</span></span>  
 <span data-ttu-id="3a88f-123">Pour des performances optimales dans un *scénario de couche intermédiaire*— un service qui appelle d’autres services en réponse aux messages entrants, instanciez le client du service WCF au service distant qu’une seule fois et la réutiliser dans plusieurs entrant requêtes.</span><span class="sxs-lookup"><span data-stu-id="3a88f-123">For optimal performance in a *middle-tier scenario*—a service that calls out to other services in response to incoming messages—instantiate the WCF service client to the remote service once and reuse it across multiple incoming requests.</span></span> <span data-ttu-id="3a88f-124">Instanciation des clients de service WCF est une opération coûteuse par rapport à un service appel sur une instance cliente préexistante, et les scénarios de couche intermédiaire produisent des gains de performances distincts en mettant en cache des clients distants sur des requêtes.</span><span class="sxs-lookup"><span data-stu-id="3a88f-124">Instantiating WCF service clients is an expensive operation relative to making a service call on a pre-existing client instance, and middle-tier scenarios produce distinct performance gains by caching remote clients across requests.</span></span> <span data-ttu-id="3a88f-125">Les clients de service WCF sont thread-safe, donc il n’est pas nécessaire de synchroniser l’accès à un client sur plusieurs threads.</span><span class="sxs-lookup"><span data-stu-id="3a88f-125">WCF service clients are thread-safe, so it is not necessary to synchronize access to a client across multiple threads.</span></span>

 <span data-ttu-id="3a88f-131">Pour vous assurer que le nom d’hôte correct apparaît dans les métadonnées du service que WCF génère, configurez l’identité par défaut pour le site Web IIS qui héberge les services WCF pour utiliser un nom d’hôte explicite.</span><span class="sxs-lookup"><span data-stu-id="3a88f-131">To ensure that the correct hostname appears in the service metadata WCF generates, configure the default identity for the IIS Web site that hosts WCF services to use an explicit hostname.</span></span> <span data-ttu-id="3a88f-132">Par exemple, les ordinateurs qui se trouvent à l’intérieur de la `www.contoso.com` batterie de serveurs doit utiliser une liaison de site IIS de \*:80:www.contoso.com pour HTTP et \*: 443:www.contoso.com pour le protocole HTTPS.</span><span class="sxs-lookup"><span data-stu-id="3a88f-132">For example, computers that reside inside of the `www.contoso.com` farm should use an IIS site binding of \*:80:www.contoso.com for HTTP and \*:443:www.contoso.com for HTTPS.</span></span>
