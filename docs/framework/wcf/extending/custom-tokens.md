# <a name="custom-tokens"></a>Jetons personnalisés
Bien que Windows Communication Foundation (WCF) prenne en charge en mode natif les certificats X. 509, les jetons de contexte de sécurité, les tickets Kerberos et les jetons de nom d’utilisateur en tant qu’informations d’identification, il est suffisamment flexible pour vous permettre d’utiliser vos propres informations d’identification personnalisées.  
  
## <a name="in-this-section"></a>Dans cette section  
 [Guide pratique pour Créer un jeton personnalisé](how-to-create-a-custom-token.md)  
 Décrit comment créer un jeton de sécurité personnalisé à l'aide de la classe <xref:System.IdentityModel.Tokens.SecurityToken> et comment l'intégrer à un fournisseur et un authentificateur de jetons de sécurité personnalisés.
