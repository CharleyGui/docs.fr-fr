#### <a name="setting-a-clientcredentials-value-in-configuration"></a><span data-ttu-id="abf1a-182">Définition d’une \<clientCredentials> valeur dans la configuration</span><span class="sxs-lookup"><span data-stu-id="abf1a-182">Setting a \<clientCredentials> Value in Configuration</span></span>  

 <span data-ttu-id="abf1a-183">Les valeurs d’informations d’identification sont spécifiées à l’aide d’un comportement de point de terminaison en tant qu’éléments enfants de l' [\<clientCredentials>](../configure-apps/file-schema/wcf/clientcredentials.md) élément.</span><span class="sxs-lookup"><span data-stu-id="abf1a-183">Credential values are specified by using an endpoint behavior as child elements of the [\<clientCredentials>](../configure-apps/file-schema/wcf/clientcredentials.md) element.</span></span> <span data-ttu-id="abf1a-184">L'élément utilisé dépend du type d'information d'identification du client.</span><span class="sxs-lookup"><span data-stu-id="abf1a-184">The element used depends on the client credential type.</span></span> <span data-ttu-id="abf1a-185">Par exemple, l’exemple suivant illustre la configuration pour définir un certificat X. 509 à l’aide de l' <[\<clientCertificate>](../configure-apps/file-schema/wcf/clientcertificate-of-clientcredentials-element.md) .</span><span class="sxs-lookup"><span data-stu-id="abf1a-185">For example, the following example shows the configuration to set an X.509 certificate using the <[\<clientCertificate>](../configure-apps/file-schema/wcf/clientcertificate-of-clientcredentials-element.md).</span></span>

<configuration>  
  <system.serviceModel>  
    <behaviors>  
      <endpointBehaviors>
        <behavior name="myEndpointBehavior">

 <span data-ttu-id="abf1a-186">Pour définir les informations d’identification du client dans la configuration, ajoutez un [\<endpointBehaviors>](../configure-apps/file-schema/wcf/endpointbehaviors.md) élément au fichier de configuration.</span><span class="sxs-lookup"><span data-stu-id="abf1a-186">To set the client credential in configuration, add an [\<endpointBehaviors>](../configure-apps/file-schema/wcf/endpointbehaviors.md) element to the configuration file.</span></span> <span data-ttu-id="abf1a-187">En outre, l’élément de comportement ajouté doit être lié au point de terminaison du service à l’aide `behaviorConfiguration` de l’attribut du [ \<endpoint> \<client> d'](../configure-apps/file-schema/wcf/endpoint-of-client.md) élément, comme indiqué dans l’exemple suivant.</span><span class="sxs-lookup"><span data-stu-id="abf1a-187">Additionally, the added behavior element must be linked to the service's endpoint using the `behaviorConfiguration` attribute of the [\<endpoint> of \<client>](../configure-apps/file-schema/wcf/endpoint-of-client.md) element as shown in the following example.</span></span> <span data-ttu-id="abf1a-188">La valeur de l'attribut `behaviorConfiguration` doit correspondre à la valeur de l'attribut de comportement `name`.</span><span class="sxs-lookup"><span data-stu-id="abf1a-188">The value of the `behaviorConfiguration` attribute must match the value of the behavior `name` attribute.</span></span>
