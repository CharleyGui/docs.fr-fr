### <a name="schema-owners-and-permissions"></a><span data-ttu-id="f63b3-110">Propriétaires et autorisations de schéma</span><span class="sxs-lookup"><span data-stu-id="f63b3-110">Schema Owners and Permissions</span></span>  
 <span data-ttu-id="f63b3-111">Les schémas peuvent appartenir à n'importe quelle principal de sécurité de base de données, tandis qu'une principal de sécurité unique peut posséder plusieurs schémas.</span><span class="sxs-lookup"><span data-stu-id="f63b3-111">Schemas can be owned by any database principal, and a single principal can own multiple schemas.</span></span> <span data-ttu-id="f63b3-112">Vous pouvez appliquer des règles de sécurité à un schéma, dont hériteront tous les objets du schéma.</span><span class="sxs-lookup"><span data-stu-id="f63b3-112">You can apply security rules to a schema, which are inherited by all objects in the schema.</span></span> <span data-ttu-id="f63b3-113">Une fois que vous avez défini des autorisations d'accès pour un schéma, elles sont automatiquement appliquées lors de l'ajout de nouveaux objets au schéma.</span><span class="sxs-lookup"><span data-stu-id="f63b3-113">Once you set up access permissions for a schema, those permissions are automatically applied as new objects are added to the schema.</span></span> <span data-ttu-id="f63b3-114">Un schéma par défaut peut être affecté aux utilisateurs, et plusieurs utilisateurs de base de données peuvent partager le même schéma.</span><span class="sxs-lookup"><span data-stu-id="f63b3-114">Users can be assigned a default schema, and multiple database users can share the same schema.</span></span>
