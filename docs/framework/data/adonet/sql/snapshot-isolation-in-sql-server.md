## <a name="how-snapshot-isolation-and-row-versioning-work"></a><span data-ttu-id="35a00-156">Fonctionnement de l'isolation d'instantané et de la gestion des versions de ligne</span><span class="sxs-lookup"><span data-stu-id="35a00-156">How Snapshot Isolation and Row Versioning Work</span></span>  
 <span data-ttu-id="35a00-157">Lorsque le niveau d’isolement d’instantané est activé, chaque fois qu’une ligne est mise à jour, le SQL Server Moteur de base de données stocke une copie de la ligne d’origine dans **tempdb**et ajoute un numéro de séquence de transaction à la ligne.</span><span class="sxs-lookup"><span data-stu-id="35a00-157">When the SNAPSHOT isolation level is enabled, each time a row is updated, the SQL Server Database Engine stores a copy of the original row in **tempdb**, and adds a transaction sequence number to the row.</span></span> <span data-ttu-id="35a00-158">La section suivante présente la séquence des événements qui se produit :</span><span class="sxs-lookup"><span data-stu-id="35a00-158">The following is the sequence of events that occurs:</span></span>  
  
- <span data-ttu-id="35a00-159">Une nouvelle transaction est initiée, qui reçoit un numéro de séquence de transaction.</span><span class="sxs-lookup"><span data-stu-id="35a00-159">A new transaction is initiated, and it is assigned a transaction sequence number.</span></span>  
  
- <span data-ttu-id="35a00-160">Le Moteur de base de données lit une ligne dans la transaction et récupère la version de ligne de **tempdb** dont le numéro de séquence est le plus proche et le plus petit que le numéro de séquence de la transaction.</span><span class="sxs-lookup"><span data-stu-id="35a00-160">The Database Engine reads a row within the transaction and retrieves the row version from **tempdb** whose sequence number is closest to, and lower than, the transaction sequence number.</span></span>  
  
- <span data-ttu-id="35a00-161">Le moteur de base de données vérifie si le numéro de séquence de la transaction ne figure pas dans la liste des numéros de séquence de transaction des transactions non validées actives au démarrage de la transaction d'instantané.</span><span class="sxs-lookup"><span data-stu-id="35a00-161">The Database Engine checks to see if the transaction sequence number is not in the list of transaction sequence numbers of the uncommitted transactions active when the snapshot transaction started.</span></span>
