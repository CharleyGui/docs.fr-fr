# <a name="entity-data-model-inheritance"></a><span data-ttu-id="f6fea-102">Entity Data Model : Héritage</span><span class="sxs-lookup"><span data-stu-id="f6fea-102">Entity Data Model: Inheritance</span></span>

<span data-ttu-id="f6fea-103">Le Entity Data Model (EDM) prend en charge l’héritage pour les [types d’entités](entity-type.md).</span><span class="sxs-lookup"><span data-stu-id="f6fea-103">The Entity Data Model (EDM) supports inheritance for [entity types](entity-type.md).</span></span> <span data-ttu-id="f6fea-104">L'héritage dans le modèle EDM est semblable à l'héritage pour les classes dans les langages de programmation orientés objet.</span><span class="sxs-lookup"><span data-stu-id="f6fea-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="f6fea-105">Comme pour les classes dans les langages orientés objet, dans un modèle conceptuel, vous pouvez définir un type d’entité ( *type dérivé*) qui hérite d’un autre type d’entité ( *type de base*).</span><span class="sxs-lookup"><span data-stu-id="f6fea-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="f6fea-106">Toutefois, contrairement aux classes dans la programmation orientée objet, dans un modèle conceptuel, le type dérivé hérite toujours de toutes les [Propriétés](property.md) et [Propriétés de navigation](navigation-property.md) du type de base.</span><span class="sxs-lookup"><span data-stu-id="f6fea-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](property.md) and [navigation properties](navigation-property.md) of the base type.</span></span> <span data-ttu-id="f6fea-107">Vous ne pouvez pas remplacer les propriétés héritées dans un type dérivé.</span><span class="sxs-lookup"><span data-stu-id="f6fea-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="f6fea-108">Dans un modèle conceptuel, vous pouvez générer des hiérarchies d'héritage dans lesquelles un type dérivé hérite d'un autre type dérivé.</span><span class="sxs-lookup"><span data-stu-id="f6fea-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="f6fea-109">Le type situé en haut de la hiérarchie (celui de la hiérarchie qui n’est pas un type dérivé) est appelé *type racine*.</span><span class="sxs-lookup"><span data-stu-id="f6fea-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="f6fea-110">Dans une hiérarchie d’héritage, la [clé d’entité](entity-key.md) doit être définie sur le type racine.</span><span class="sxs-lookup"><span data-stu-id="f6fea-110">In an inheritance hierarchy, the [entity key](entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="f6fea-111">Vous ne pouvez pas générer de hiérarchies d'héritage dans lesquelles un type dérivé hérite de plusieurs types.</span><span class="sxs-lookup"><span data-stu-id="f6fea-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="f6fea-112">Par exemple, dans un modèle conceptuel avec un type d'entité `Book`, vous pouvez définir les types dérivés `FictionBook` et `NonFictionBook` qui héritent de `Book`.</span><span class="sxs-lookup"><span data-stu-id="f6fea-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="f6fea-113">Toutefois, vous ne pouvez pas ensuite définir un type qui hérite à la fois du type `FictionBook` et du type `NonFictionBook`.</span><span class="sxs-lookup"><span data-stu-id="f6fea-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
