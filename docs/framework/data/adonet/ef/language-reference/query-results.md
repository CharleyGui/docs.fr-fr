# <a name="query-results"></a><span data-ttu-id="46d09-102">Résultats de requête</span><span class="sxs-lookup"><span data-stu-id="46d09-102">Query Results</span></span>
<span data-ttu-id="46d09-103">Une fois qu’une requête de LINQ to Entities est convertie en arborescences de commandes et exécutée, les résultats de la requête sont généralement retournés comme suit :</span><span class="sxs-lookup"><span data-stu-id="46d09-103">After a LINQ to Entities query is converted to command trees and executed, the query results are usually returned as one of the following:</span></span>  
  
- <span data-ttu-id="46d09-104">une collection de zéro, un ou plusieurs objets entité typés ou une projection de types complexes dans le modèle conceptuel ;</span><span class="sxs-lookup"><span data-stu-id="46d09-104">A collection of zero or more typed entity objects or a projection of complex types in the conceptual model.</span></span>  
  
- <span data-ttu-id="46d09-105">des types CLR pris en charge par le modèle conceptuel ;</span><span class="sxs-lookup"><span data-stu-id="46d09-105">CLR types supported by the conceptual model.</span></span>  
  
- <span data-ttu-id="46d09-106">Collections inline.</span><span class="sxs-lookup"><span data-stu-id="46d09-106">Inline collections.</span></span>  
  
- <span data-ttu-id="46d09-107">Types anonymes.</span><span class="sxs-lookup"><span data-stu-id="46d09-107">Anonymous types.</span></span>  
  
 <span data-ttu-id="46d09-108">Lorsque la requête a été exécutée sur la source de données, les résultats sont matérialisés en types CLR et retournés au client.</span><span class="sxs-lookup"><span data-stu-id="46d09-108">When the query has executed against the data source, the results are materialized into CLR types and returned to the client.</span></span> <span data-ttu-id="46d09-109">Toute matérialisation d’objets est effectuée par Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="46d09-109">All object materialization is performed by the Entity Framework.</span></span> <span data-ttu-id="46d09-110">Toute erreur qui résulte d’une impossibilité d’effectuer un mappage entre Entity Framework et le CLR lève une exception à lever pendant la matérialisation d’objets.</span><span class="sxs-lookup"><span data-stu-id="46d09-110">Any errors that result from an inability to map between the Entity Framework and the CLR will cause exceptions to be thrown during object materialization.</span></span>
  
 <span data-ttu-id="46d09-111">Si l’exécution de la requête retourne des types de modèle conceptuel primitifs, les résultats se composent de types CLR autonomes et déconnectés de l’Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="46d09-111">If the query execution returns primitive conceptual model types, the results consist of CLR types that are stand-alone and disconnected from the Entity Framework.</span></span> <span data-ttu-id="46d09-112">Toutefois, si la requête retourne une collection d’objets entité typés, représentée par <xref:System.Data.Objects.ObjectQuery%601>, ces types sont suivis par le contexte d’objet.</span><span class="sxs-lookup"><span data-stu-id="46d09-112">However, if the query returns a collection of typed entity objects, represented by <xref:System.Data.Objects.ObjectQuery%601>, those types are tracked by the object context.</span></span> <span data-ttu-id="46d09-113">Tous les comportements d’objet (tels que les collections enfants/parents, le suivi des modifications, le polymorphisme, etc.) sont définis dans la Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="46d09-113">All object behavior (such as child/parent collections, change tracking, polymorphism, and so on) are as defined in the Entity Framework.</span></span> <span data-ttu-id="46d09-114">Cette fonctionnalité peut être utilisée dans sa capacité, comme défini dans la Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="46d09-114">This functionality can be used in its capacity, as defined in the Entity Framework.</span></span> <span data-ttu-id="46d09-115">Pour plus d’informations, consultez [utilisation des objets](../working-with-objects.md).</span><span class="sxs-lookup"><span data-stu-id="46d09-115">For more information, see [Working with Objects](../working-with-objects.md).</span></span>
  
 <span data-ttu-id="46d09-116">Les types Struct retournés par les requêtes (tels que les types anonymes et les types complexes Nullable) peuvent être de valeur `null`.</span><span class="sxs-lookup"><span data-stu-id="46d09-116">Struct types returned from queries (such as anonymous types and nullable complex types) can be of `null` value.</span></span> <span data-ttu-id="46d09-117">Une propriété <xref:System.Data.Objects.DataClasses.EntityCollection%601> d'une entité retournée peut également être de valeur `null`.</span><span class="sxs-lookup"><span data-stu-id="46d09-117">An <xref:System.Data.Objects.DataClasses.EntityCollection%601> property of a returned entity can also be of `null` value.</span></span> <span data-ttu-id="46d09-118">Cela peut être le résultat de la projection de la propriété de collection d'une entité de valeur `null`, par exemple, en cas d'appel de <xref:System.Linq.Queryable.FirstOrDefault%2A> sur un objet <xref:System.Data.Objects.ObjectQuery%601> qui n'a pas d'éléments.</span><span class="sxs-lookup"><span data-stu-id="46d09-118">This can result from projecting the collection property of an entity that is of `null` value, such as calling <xref:System.Linq.Queryable.FirstOrDefault%2A> on an <xref:System.Data.Objects.ObjectQuery%601> that has no elements.</span></span>  
  
 <span data-ttu-id="46d09-119">Dans certains cas, une requête peut sembler générer un résultat matérialisé pendant son exécution, mais il s'avère qu'elle est exécutée sur le serveur et que l'objet entité n'est jamais matérialisé dans le CLR.</span><span class="sxs-lookup"><span data-stu-id="46d09-119">In certain situations, a query might appear to generate a materialized result during its execution, but the query will be executed on the server and the entity object will never be materialized in the CLR.</span></span> <span data-ttu-id="46d09-120">Cela peut être la cause de problèmes si vous êtes dépendant des effets secondaires de la matérialisation d'objets.</span><span class="sxs-lookup"><span data-stu-id="46d09-120">This can cause problems if you are depending on side effects of object materialization.</span></span>  
  
 <span data-ttu-id="46d09-121">L'exemple suivant contient une classe personnalisée, `MyContact`, avec une propriété `LastName`.</span><span class="sxs-lookup"><span data-stu-id="46d09-121">The following example contains a custom class, `MyContact`, with a `LastName` property.</span></span> <span data-ttu-id="46d09-122">Lorsque la propriété `LastName` est définie, une variable `count` est incrémentée.</span><span class="sxs-lookup"><span data-stu-id="46d09-122">When the `LastName` property is set, a `count` variable is incremented.</span></span> <span data-ttu-id="46d09-123">Si vous exécutez les deux requêtes suivantes, la première incrémente `count` mais pas la deuxième.</span><span class="sxs-lookup"><span data-stu-id="46d09-123">If you execute the two following queries, the first query will increment `count` while the second query will not.</span></span> <span data-ttu-id="46d09-124">Cela est dû au fait que dans la deuxième requête, la propriété `LastName` est projetée à partir des résultats et que la classe `MyContact` n'est jamais créée, car elle n'est pas utile à l'exécution de la requête sur le magasin.</span><span class="sxs-lookup"><span data-stu-id="46d09-124">This is because in the second query the `LastName` property is projected from the results and the `MyContact` class is never created, because it is not required to execute the query on the store.</span></span>
