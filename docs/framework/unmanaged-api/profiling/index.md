# <a name="profiling-unmanaged-api-reference"></a>Profilage (Informations de référence sur les API non managées)

L’API de profilage permet à un profileur de surveiller l’exécution d’un programme par le CLR (Common Language Runtime).

## <a name="in-this-section"></a>Dans cette section

 [Vue d’ensemble du profilage](profiling-overview.md) Décrit les services et les interfaces fournis par le CLR pour prendre en charge le profilage dans l’environnement .NET Framework.

 [Interfaces de profilage](profiling-interfaces.md) Décrit les interfaces non managées utilisées par l’API de profilage.

 [Configuration d’un environnement de profilage](setting-up-a-profiling-environment.md) Décrit les étapes à suivre pour profiler une application .NET Framework.

 [Profileurs CLR et applications Windows Store](clr-profilers-and-windows-store-apps.md) Explique comment déplacer des outils de diagnostic qui utilisent l’API de profilage CLR pour qu’ils fonctionnent correctement avec les applications Windows Store.

 [CORPROF_E_UNSUPPORTED_CALL_SEQUENCE HRESULT](corprof-e-unsupported-call-sequence-hresult.md) Documente les conditions dans lesquelles un appel de méthode retourne le HRESULT `CORPROF_E_UNSUPPORTED_CALL_SEQUENCE`.

 [Fonctions statiques globales de profilage](profiling-global-static-functions.md) Décrit les fonctions statiques globales non managées utilisées par l’API de profilage.

 [Énumérations de profilage](profiling-enumerations.md) Décrit les énumérations non managées utilisées par l’API de profilage.

 [Structures de profilage](profiling-structures.md) Décrit les structures non managées utilisées par l’API de profilage.
