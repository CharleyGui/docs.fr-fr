# <a name="peer-names-and-pnrp-ids"></a><span data-ttu-id="5a7ac-102">Noms d’homologues et ID PNRP</span><span class="sxs-lookup"><span data-stu-id="5a7ac-102">Peer Names and PNRP IDs</span></span>
<span data-ttu-id="5a7ac-103">Un nom de pair représente un point de terminaison pour la communication. ## <a name="peer-names"></a><span data-ttu-id="5a7ac-105">Noms de pairs</span><span class="sxs-lookup"><span data-stu-id="5a7ac-105">Peer Names</span></span>  
 <span data-ttu-id="5a7ac-106">Les noms de pairs peuvent être inscrits comme étant sécurisés ou non sécurisés.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-106">Peer names can be registered as unsecured or secured.</span></span> <span data-ttu-id="5a7ac-107">Les noms non sécurisés sont de simples chaînes de texte qui peuvent faire l’objet d’une usurpation d’identité, puisque n’importe qui peut inscrire une duplication d’un nom non sécurisé.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-107">Unsecured names are just text strings that are subject to spoofing, as anyone can register a duplicate unsecured name.</span></span> <span data-ttu-id="5a7ac-108">Il est préférable d’utiliser les noms non sécurisés au sein de réseaux privés ou protégés.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-108">Unsecured names are best used in private or otherwise protected networks.</span></span> <span data-ttu-id="5a7ac-109">Les noms sécurisés sont protégés par un certificat et une signature numérique.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-109">Secured names are protected with a certificate and a digital signature.</span></span> <span data-ttu-id="5a7ac-110">Seul l’éditeur d’origine est en mesure de prouver sa propriété d’un nom sécurisé.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-110">Only the original publisher will be able to prove ownership of a secured name.</span></span>  
  
 <span data-ttu-id="5a7ac-111">La combinaison d’un cloud et d’une étendue fournit un environnement relativement sécurisé pour les pairs qui participent à des activités PNRP.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-111">The combination of cloud and scope provides a reasonably secure environment for peers that participate in PNRP activity.</span></span> <span data-ttu-id="5a7ac-112">Toutefois, l’utilisation d’un nom de pair sécurisé ne garantit pas la sécurité de l’ensemble de l’application réseau.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-112">However, using a secured peer name does not ensure the overall security of the networking application.</span></span> <span data-ttu-id="5a7ac-113">La sécurité d’une application dépend de son implémentation.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-113">Security of the application is implementation-dependent.</span></span>  
  
 <span data-ttu-id="5a7ac-114">Les noms de pairs sécurisés sont inscrits uniquement par leur propriétaire et sont protégés par un chiffrement à clé publique.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-114">Secured peer names are only registered by their owner and are protected with public key cryptography.</span></span> <span data-ttu-id="5a7ac-115">Un nom de pair sécurisé est considéré comme appartenant à l’entité pair qui contient la clé privée correspondante.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-115">A secured peer name is considered owned by the peer entity having the corresponding private key.</span></span> <span data-ttu-id="5a7ac-116">La propriété peut être prouvée via l’adresse pair certifiée, qui est signée à l’aide de la clé privée.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-116">Ownership can be proved via the certified peer address (CPA), which is signed using the private key.</span></span> <span data-ttu-id="5a7ac-117">Un utilisateur malveillant ne peut pas feindre la propriété d’un nom de pair sans disposer de la clé privée correspondante.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-117">A malicious user cannot forge ownership of a peer name without the corresponding private key.</span></span>  
  
## <a name="pnrp-ids"></a><span data-ttu-id="5a7ac-118">ID PNRP</span><span class="sxs-lookup"><span data-stu-id="5a7ac-118">PNRP IDs</span></span>  
 <span data-ttu-id="5a7ac-119">![ID PNRP](./media/fdc9e8a0-4a1c-488d-a019-bc3a1973220c.gif "fdc9e8a0-4a1c-488d-a019-bc3a1973220c")</span><span class="sxs-lookup"><span data-stu-id="5a7ac-119">![PNRP ID](./media/fdc9e8a0-4a1c-488d-a019-bc3a1973220c.gif "fdc9e8a0-4a1c-488d-a019-bc3a1973220c")</span></span>  
  
 <span data-ttu-id="5a7ac-120">Les ID PNRP sont formés des composants suivants :</span><span class="sxs-lookup"><span data-stu-id="5a7ac-120">PNRP IDs are composed of the following:</span></span>

- <span data-ttu-id="5a7ac-127">Les 128 bits d’ordre bas sont utilisés pour l’emplacement du service, qui est un nombre généré qui identifie les différentes instances du même ID P2P dans un même cloud.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-127">The low-order 128 bits are used for the Service Location, which is a generated number that identifies different instances of the same P2P ID in the same cloud.</span></span>  
  
 <span data-ttu-id="5a7ac-128">Cette combinaison d’ID P2P et de l’emplacement du service permet à plusieurs ID PNRP d’être inscrits à partir d’un même ordinateur.</span><span class="sxs-lookup"><span data-stu-id="5a7ac-128">This combination of P2P ID and Service Location allows multiple PNRP IDs to be registered from a single computer.</span></span>
