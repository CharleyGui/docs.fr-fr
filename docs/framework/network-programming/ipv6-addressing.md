# <a name="ipv6-addressing"></a><span data-ttu-id="937bb-102">Adressage IPv6</span><span class="sxs-lookup"><span data-stu-id="937bb-102">IPv6 Addressing</span></span>

- <span data-ttu-id="937bb-109">**Format hexadécimal/deux-points**.</span><span class="sxs-lookup"><span data-stu-id="937bb-109">**Colon-hexadecimal form**.</span></span> <span data-ttu-id="937bb-110">Il s’agit du format recommandé : n:n:n:n:n:n:n:n.</span><span class="sxs-lookup"><span data-stu-id="937bb-110">This is the preferred form n:n:n:n:n:n:n:n.</span></span> <span data-ttu-id="937bb-111">Chaque n représente la valeur hexadécimale de l’un des huit éléments 16 bits de l’adresse.</span><span class="sxs-lookup"><span data-stu-id="937bb-111">Each n represents the hexadecimal value of one of the eight 16-bit elements of the address.</span></span> <span data-ttu-id="937bb-112">Par exemple : `3FFE:FFFF:7654:FEDA:1245:BA98:3210:4562`.</span><span class="sxs-lookup"><span data-stu-id="937bb-112">For example: `3FFE:FFFF:7654:FEDA:1245:BA98:3210:4562`.</span></span>

- <span data-ttu-id="937bb-113">**Format compressé**.</span><span class="sxs-lookup"><span data-stu-id="937bb-113">**Compressed form**.</span></span> <span data-ttu-id="937bb-114">En raison de la ## <a name="address-types"></a><span data-ttu-id="937bb-124">Types d’adresses</span><span class="sxs-lookup"><span data-stu-id="937bb-124">Address Types</span></span>

<span data-ttu-id="937bb-125">Les bits de début de l’adresse déterminent le type de l’adresse IPv6.</span><span class="sxs-lookup"><span data-stu-id="937bb-125">The leading bits in the address define the specific IPv6 address type.</span></span> <span data-ttu-id="937bb-126">Le champ de longueur variable qui contient ces bits de début est appelé « préfixe de format ».</span><span class="sxs-lookup"><span data-stu-id="937bb-126">The variable-length field containing these leading bits is called a Format Prefix (FP).</span></span>

<span data-ttu-id="937bb-127">Une adresse de monodiffusion IPv6 est constituée de deux parties.</span><span class="sxs-lookup"><span data-stu-id="937bb-127">An IPv6 unicast address is divided into two parts.</span></span> <span data-ttu-id="937bb-128">La première partie contient le préfixe de l’adresse et la seconde contient l’identificateur d’interface.</span><span class="sxs-lookup"><span data-stu-id="937bb-128">The first part contains the address prefix, and the second part contains the interface identifier.</span></span> <span data-ttu-id="937bb-129">Voici comment écrire de manière concise la combinaison préfixe-adresse IPv6 : ipv6-address/prefix-length.</span><span class="sxs-lookup"><span data-stu-id="937bb-129">A concise way to express an IPv6 address/prefix combination is as follows: ipv6-address/prefix-length.</span></span>

<span data-ttu-id="937bb-130">Voici un exemple d’adresse avec un préfixe 64 bits.</span><span class="sxs-lookup"><span data-stu-id="937bb-130">The following is an example of an address with a 64-bit prefix.</span></span>

<span data-ttu-id="937bb-132">Dans cet exemple, le préfixe est `3FFE:FFFF:0:CD30`.</span><span class="sxs-lookup"><span data-stu-id="937bb-132">The prefix in this example is `3FFE:FFFF:0:CD30`.</span></span> <span data-ttu-id="937bb-133">L’adresse peut également être écrite au format compressé, de la manière suivante : `3FFE:FFFF:0:CD30::/64`.</span><span class="sxs-lookup"><span data-stu-id="937bb-133">The address can also be written in a compressed form, as `3FFE:FFFF:0:CD30::/64`.</span></span>

<span data-ttu-id="937bb-134">IPv6 définit les types d’adresses suivants :</span><span class="sxs-lookup"><span data-stu-id="937bb-134">IPv6 defines the following address types:</span></span>

- <span data-ttu-id="937bb-135">**Adresses de monodiffusion**.</span><span class="sxs-lookup"><span data-stu-id="937bb-135">**Unicast address**.</span></span> <span data-ttu-id="937bb-136">Identificateur d’une interface unique.</span><span class="sxs-lookup"><span data-stu-id="937bb-136">An identifier for a single interface.</span></span> <span data-ttu-id="937bb-137">Un paquet envoyé à cette adresse est remis à l’interface identifiée.</span><span class="sxs-lookup"><span data-stu-id="937bb-137">A packet sent to this address is delivered to the identified interface.</span></span> <span data-ttu-id="937bb-138">Les adresses de monodiffusion se distinguent des adresses de multidiffusion par la valeur de l’octet d’ordre haut.</span><span class="sxs-lookup"><span data-stu-id="937bb-138">The unicast addresses are distinguished from the multicast addresses by the value of the high-order octet.</span></span> <span data-ttu-id="937bb-139">L’octet d’ordre haut des adresses de multidiffusion a une valeur hexadécimale de FF.</span><span class="sxs-lookup"><span data-stu-id="937bb-139">The multicast addresses' high-order octet has the hexadecimal value of FF.</span></span> <span data-ttu-id="937bb-140">Pour cet octet, toute autre valeur correspond à une adresse de monodiffusion.</span><span class="sxs-lookup"><span data-stu-id="937bb-140">Any other value for this octet identifies a unicast address.</span></span> <span data-ttu-id="937bb-141">Voici les différents types d’adresses de monodiffusion :</span><span class="sxs-lookup"><span data-stu-id="937bb-141">The following are different types of unicast addresses:</span></span>

- <span data-ttu-id="937bb-155">**Adresses anycast**.</span><span class="sxs-lookup"><span data-stu-id="937bb-155">**Anycast address**.</span></span> <span data-ttu-id="937bb-156">Identificateur d’un ensemble d’interfaces (appartenant généralement à des nœuds différents).</span><span class="sxs-lookup"><span data-stu-id="937bb-156">An identifier for a set of interfaces (typically belonging to different nodes).</span></span> <span data-ttu-id="937bb-157">Un paquet envoyé à cette adresse est remis à seulement une interface identifiée par cette adresse.</span><span class="sxs-lookup"><span data-stu-id="937bb-157">A packet sent to this address is delivered to only one interface identified by the address.</span></span> <span data-ttu-id="937bb-158">Il s’agit de l’interface la plus proche, d’après les métriques de routage.</span><span class="sxs-lookup"><span data-stu-id="937bb-158">This is the nearest interface as identified by routing metrics.</span></span> <span data-ttu-id="937bb-159">Les adresses anycast proviennent de l’espace d’adressage de monodiffusion et ne peuvent pas être distinguées par leur syntaxe.</span><span class="sxs-lookup"><span data-stu-id="937bb-159">Anycast addresses are taken from the unicast address space and are not syntactically distinguishable.</span></span> <span data-ttu-id="937bb-160">L’interface adressée effectue la distinction entre les adresses de monodiffusion et les adresses anycast, dans le cadre de sa configuration.</span><span class="sxs-lookup"><span data-stu-id="937bb-160">The addressed interface performs the distinction between unicast and anycast addresses as a function of its configuration.</span></span>
