class="sxs-lookup"><span data-stu-id="69a28-104">The purpose of the TPL is to make developers more productive by simplifying the process of adding parallelism and concurrency to applications.</span></span> <span data-ttu-id="69a28-105">La bibliothèque parallèle de tâches met à l'échelle dynamiquement le degré d'accès concurrentiel pour utiliser plus efficacement tous les processeurs disponibles.</span><span class="sxs-lookup"><span data-stu-id="69a28-105">The TPL scales the degree of concurrency dynamically to most efficiently use all the processors that are available.</span></span> <span data-ttu-id="69a28-106">De plus, la bibliothèque parallèle de tâches gère le partitionnement du travail, la planification de threads sur le <span data-ttu-id="69a28-107">L'utilisation de la bibliothèque parallèle de tâches vous permet de maximiser les performances de votre code tout en vous concentrant sur le travail que votre programme doit accomplir.</span><span class="sxs-lookup"><span data-stu-id="69a28-107">By using TPL, you can maximize the performance of your code while focusing on the work that your program is designed to accomplish.</span></span>  
  
 <span data-ttu-id="69a28-108">À compter de .NET Framework 4, la bibliothèque parallèle de tâches est la meilleure méthode pour écrire le code multithread et parallèle.</span><span class="sxs-lookup"><span data-stu-id="69a28-108">Starting with the .NET Framework 4, the TPL is the preferred way to write multithreaded and parallel code.</span></span> <span data-ttu-id="69a28-109">Toutefois, tout le code est pas approprié pour la parallélisation ; par exemple, si une boucle exécute uniquement une petite quantité de travail sur chaque itération ou ne s'exécute que pour un nombre limité d'itérations, la charge mémoire de la parallélisation peut ralentir l'exécution du code.</span><span class="sxs-lookup"><span data-stu-id="69a28-109">However, not all code is suitable for parallelization; for example, if a loop performs only a small amount of work on each iteration, or it doesn't run for many iterations, then the overhead of parallelization can cause the code to run more slowly.</span></span> <span data-ttu-id="69a28-110">En outre, comme tout code multithread, la parallélisation rend l'exécution du programme plus complexe.</span><span class="sxs-lookup"><span data-stu-id="69a28-110">Furthermore, parallelization like any multithreaded code adds complexity to your program execution.</span></span> <span data-ttu-id="69a28-111">Même si la bibliothèque parallèle de tâches simplifie les scénarios multithread, il est recommandé de connaître les notions fondamentales des concepts de threading, tels que les verrous, les interblocages et les conditions de concurrence critique, afin de pouvoir utiliser efficacement la bibliothèque parallèle de tâches.</span><span class="sxs-lookup"><span data-stu-id="69a28-111">Although the TPL simplifies multithreaded scenarios, we recommend that you have a basic understanding of threading concepts, for example, locks, deadlocks, and race conditions, so that you can use the TPL effectively.</span></span>
