## <a name="parallelmergeoptions"></a><span data-ttu-id="9d603-117">ParallelMergeOptions</span><span class="sxs-lookup"><span data-stu-id="9d603-117">ParallelMergeOptions</span></span>  
 <span data-ttu-id="9d603-118">L’énumération <xref:System.Linq.ParallelMergeOptions> inclut les options suivantes qui spécifient, pour les formes de requête prises en charge, la manière dont la sortie finale de la requête est transmise quand les résultats sont utilisés sur un thread :</span><span class="sxs-lookup"><span data-stu-id="9d603-118">The <xref:System.Linq.ParallelMergeOptions> enumeration includes the following options that specify, for supported query shapes, how the final output of the query is yielded when the results are consumed on one thread:</span></span>  
  
- `Not Buffered`  
  
     <span data-ttu-id="9d603-119">Avec l’option <xref:System.Linq.ParallelMergeOptions.NotBuffered>, chaque élément traité est retourné à partir de chaque thread dès qu’il est généré.</span><span class="sxs-lookup"><span data-stu-id="9d603-119">The <xref:System.Linq.ParallelMergeOptions.NotBuffered> option causes each processed element to be returned from each thread as soon as it is produced.</span></span> <span data-ttu-id="9d603-120">Ce comportement revient à « diffuser en continu » la sortie.</span><span class="sxs-lookup"><span data-stu-id="9d603-120">This behavior is analogous to "streaming" the output.</span></span> <span data-ttu-id="9d603-121">Si l’opérateur <xref:System.Linq.ParallelEnumerable.AsOrdered%2A> est présent dans la requête, `NotBuffered` conserve l’ordre des éléments sources.</span><span class="sxs-lookup"><span data-stu-id="9d603-121">If the <xref:System.Linq.ParallelEnumerable.AsOrdered%2A> operator is present in the query, `NotBuffered` preserves the order of the source elements.</span></span> <span data-ttu-id="9d603-122">Bien que `NotBuffered` commence à transmettre les résultats dès qu’ils sont disponibles, la durée totale nécessaire pour générer tous les résultats peut toujours être supérieure à celles des autres options de fusion.</span><span class="sxs-lookup"><span data-stu-id="9d603-122">Although `NotBuffered` starts yielding results as soon as they're available,, the total time to produce all the results might still be longer than using one of the other merge options.</span></span>  
  
- `Auto Buffered`  
  
     <span data-ttu-id="9d603-123">Avec l’option <xref:System.Linq.ParallelMergeOptions.AutoBuffered>, la requête regroupe des éléments dans une mémoire tampon, puis transmet régulièrement tout le contenu de cette mémoire simultanément au thread utilisateur.</span><span class="sxs-lookup"><span data-stu-id="9d603-123">The <xref:System.Linq.ParallelMergeOptions.AutoBuffered> option causes the query to collect elements into a buffer and then periodically yield the buffer contents all at once to the consuming thread.</span></span> <span data-ttu-id="9d603-124">Cette option revient à transmettre les données sources dans des « blocs » au lieu d’utiliser le comportement de « diffusion en continu » de `NotBuffered`.</span><span class="sxs-lookup"><span data-stu-id="9d603-124">This is analogous to yielding the source data in "chunks" instead of using the "streaming" behavior of `NotBuffered`.</span></span> <span data-ttu-id="9d603-125">`AutoBuffered` peut nécessiter plus de temps que `NotBuffered` pour rendre le premier élément disponible sur le thread utilisateur.</span><span class="sxs-lookup"><span data-stu-id="9d603-125">`AutoBuffered` may take longer than `NotBuffered` to make the first element available on the consuming thread.</span></span> <span data-ttu-id="9d603-126">La taille de la mémoire tampon et le comportement exact de transmission ne sont pas configurables et peuvent varier en fonction de différents facteurs liés à la requête.</span><span class="sxs-lookup"><span data-stu-id="9d603-126">The size of the buffer and the exact yielding behavior are not configurable and may vary, depending on various factors that relate to the query.</span></span>  
  
- `FullyBuffered`  
  
     <span data-ttu-id="9d603-127">Avec l’option <xref:System.Linq.ParallelMergeOptions.FullyBuffered>, la sortie de la requête entière est mise en mémoire tampon avant que l’un des éléments ne soit transmis.</span><span class="sxs-lookup"><span data-stu-id="9d603-127">The <xref:System.Linq.ParallelMergeOptions.FullyBuffered> option causes the output of the whole query to be buffered before any of the elements are yielded.</span></span> <span data-ttu-id="9d603-128">Cette option peut nécessiter plus de temps pour que le premier élément soit disponible sur le thread utilisateur, mais les résultats complets peuvent toujours être générés plus rapidement qu’avec les autres options.</span><span class="sxs-lookup"><span data-stu-id="9d603-128">When you use this option, it can take longer before the first element is available on the consuming thread, but the complete results might still be produced faster than by using the other options.</span></span>
