# <a name="choosing-between-class-and-struct"></a><span data-ttu-id="10853-102">Choix entre classe et structure</span><span class="sxs-lookup"><span data-stu-id="10853-102">Choosing Between Class and Struct</span></span>
<span data-ttu-id="10853-103">L’une des décisions de conception de base de chaque concepteur d’infrastructure est de concevoir un type en tant que classe (type référence) ou en tant que struct (un type valeur).</span><span class="sxs-lookup"><span data-stu-id="10853-103">One of the basic design decisions every framework designer faces is whether to design a type as a class (a reference type) or as a struct (a value type).</span></span> <span data-ttu-id="10853-104">Il est essentiel de bien comprendre les différences de comportement des types de référence et des types de valeur pour faire ce choix.</span><span class="sxs-lookup"><span data-stu-id="10853-104">Good understanding of the differences in the behavior of reference types and value types is crucial in making this choice.</span></span>

 <span data-ttu-id="10853-105">La première différence entre les types référence et les types valeur est que nous envisageons que les types référence sont alloués sur le tas et récupérés par le garbage collector, tandis que les types valeur sont alloués sur la pile ou inline dans les types contenants et désalloués lorsque la pile se déroule ou lorsque leur type conteneur est libéré.</span><span class="sxs-lookup"><span data-stu-id="10853-105">The first difference between reference types and value types we will consider is that reference types are allocated on the heap and garbage-collected, whereas value types are allocated either on the stack or inline in containing types and deallocated when the stack unwinds or when their containing type gets deallocated.</span></span> <span data-ttu-id="10853-106">Par conséquent, les allocations et les désallocations de types valeur sont en général moins coûteuses que les allocations et les désallocations de types référence.</span><span class="sxs-lookup"><span data-stu-id="10853-106">Therefore, allocations and deallocations of value types are in general cheaper than allocations and deallocations of reference types.</span></span>

 <span data-ttu-id="10853-107">Ensuite, les tableaux de types référence sont alloués hors ligne, ce qui signifie que les éléments de tableau sont simplement des références aux instances du type référence résidant sur le tas.</span><span class="sxs-lookup"><span data-stu-id="10853-107">Next, arrays of reference types are allocated out-of-line, meaning the array elements are just references to instances of the reference type residing on the heap.</span></span> <span data-ttu-id="10853-108">Les tableaux de types valeur sont alloués inline, ce qui signifie que les éléments de tableau sont les instances réelles du type valeur.</span><span class="sxs-lookup"><span data-stu-id="10853-108">Value type arrays are allocated inline, meaning that the array elements are the actual instances of the value type.</span></span> <span data-ttu-id="10853-109">Par conséquent, les allocations et les désallocations des tableaux de types valeur sont très coûteuses que les allocations et les désallocations de tableaux de types référence.</span><span class="sxs-lookup"><span data-stu-id="10853-109">Therefore, allocations and deallocations of value type arrays are much cheaper than allocations and deallocations of reference type arrays.</span></span> <span data-ttu-id="10853-110">En outre, dans la majorité des cas, les tableaux de types valeur présentent une plus grande localité de référence.</span><span class="sxs-lookup"><span data-stu-id="10853-110">In addition, in a majority of cases value type arrays exhibit much better locality of reference.</span></span>

 <span data-ttu-id="10853-111">La différence suivante est liée à l’utilisation de la mémoire.</span><span class="sxs-lookup"><span data-stu-id="10853-111">The next difference is related to memory usage.</span></span> <span data-ttu-id="10853-112">Les types valeur sont boxed lorsqu’ils sont castés en un type référence ou l’une des interfaces qu’ils implémentent.</span><span class="sxs-lookup"><span data-stu-id="10853-112">Value types get boxed when cast to a reference type or one of the interfaces they implement.</span></span> <span data-ttu-id="10853-113">Ils obtiennent unboxed lorsqu’ils sont reconvertis en type valeur.</span><span class="sxs-lookup"><span data-stu-id="10853-113">They get unboxed when cast back to the value type.</span></span> <span data-ttu-id="10853-114">Étant donné que les zones sont des objets alloués sur le tas et qui sont récupérés par le garbage collector, un trop grand nombre de conversions boxing et unboxing peuvent avoir un impact négatif sur le tas, le garbage collector et enfin sur les performances de l’application.</span><span class="sxs-lookup"><span data-stu-id="10853-114">Because boxes are objects that are allocated on the heap and are garbage-collected, too much boxing and unboxing can have a negative impact on the heap, the garbage collector, and ultimately the performance of the application.</span></span>  <span data-ttu-id="10853-115">En revanche, aucune conversion boxing de ce type ne se produit, car les types référence sont convertis.</span><span class="sxs-lookup"><span data-stu-id="10853-115">In contrast, no such boxing occurs as reference types are cast.</span></span> <span data-ttu-id="10853-116">(Pour plus d’informations, consultez [conversion boxing et unboxing](../../csharp/programming-guide/types/boxing-and-unboxing.md)).</span><span class="sxs-lookup"><span data-stu-id="10853-116">(For more information, see [Boxing and Unboxing](../../csharp/programming-guide/types/boxing-and-unboxing.md)).</span></span>

 <span data-ttu-id="10853-117">Ensuite, les assignations de type référence copient la référence, tandis que les assignations de type valeur copient la valeur entière.</span><span class="sxs-lookup"><span data-stu-id="10853-117">Next, reference type assignments copy the reference, whereas value type assignments copy the entire value.</span></span> <span data-ttu-id="10853-118">Par conséquent, les assignations de types de référence volumineux sont moins coûteuses que les assignations de types de valeur élevée.</span><span class="sxs-lookup"><span data-stu-id="10853-118">Therefore, assignments of large reference types are cheaper than assignments of large value types.</span></span>

 <span data-ttu-id="10853-119">Enfin, les types référence sont passés par référence, tandis que les types valeur sont passés par valeur.</span><span class="sxs-lookup"><span data-stu-id="10853-119">Finally, reference types are passed by reference, whereas value types are passed by value.</span></span> <span data-ttu-id="10853-120">Les modifications apportées à une instance d’un type référence affectent toutes les références pointant vers l’instance.</span><span class="sxs-lookup"><span data-stu-id="10853-120">Changes to an instance of a reference type affect all references pointing to the instance.</span></span> <span data-ttu-id="10853-121">Les instances de type valeur sont copiées lorsqu’elles sont passées par valeur.</span><span class="sxs-lookup"><span data-stu-id="10853-121">Value type instances are copied when they are passed by value.</span></span> <span data-ttu-id="10853-122">Quand une instance d’un type valeur est modifiée, elle n’affecte pas l’une de ses copies.</span><span class="sxs-lookup"><span data-stu-id="10853-122">When an instance of a value type is changed, it of course does not affect any of its copies.</span></span> <span data-ttu-id="10853-123">Étant donné que les copies ne sont pas créées explicitement par l’utilisateur mais sont implicitement créées lorsque des arguments sont passés ou que des valeurs de retour sont retournées, les types valeur qui peuvent être modifiés peuvent être déroutants pour de nombreux utilisateurs.</span><span class="sxs-lookup"><span data-stu-id="10853-123">Because the copies are not created explicitly by the user but are implicitly created when arguments are passed or return values are returned, value types that can be changed can be confusing to many users.</span></span> <span data-ttu-id="10853-124">Par conséquent, les types valeur doivent être immuables.</span><span class="sxs-lookup"><span data-stu-id="10853-124">Therefore, value types should be immutable.</span></span>

 <span data-ttu-id="10853-125">En règle générale, la majorité des types dans un Framework doivent être des classes.</span><span class="sxs-lookup"><span data-stu-id="10853-125">As a rule of thumb, the majority of types in a framework should be classes.</span></span> <span data-ttu-id="10853-126">Toutefois, il existe certaines situations dans lesquelles les caractéristiques d’un type de valeur rendent l’utilisation de structs plus appropriée.</span><span class="sxs-lookup"><span data-stu-id="10853-126">There are, however, some situations in which the characteristics of a value type make it more appropriate to use structs.</span></span>

 <span data-ttu-id="10853-127">✔️ ENVISAGER de définir un struct au lieu d’une classe si les instances du type sont petites et généralement éphémères ou sont généralement incorporées dans d’autres objets.</span><span class="sxs-lookup"><span data-stu-id="10853-127">✔️ CONSIDER defining a struct instead of a class if instances of the type are small and commonly short-lived or are commonly embedded in other objects.</span></span>

 <span data-ttu-id="10853-128">❌Évitez de définir un struct à moins que le type ait toutes les caractéristiques suivantes :</span><span class="sxs-lookup"><span data-stu-id="10853-128">❌ AVOID defining a struct unless the type has all of the following characteristics:</span></span>

- <span data-ttu-id="10853-129">Il représente logiquement une valeur unique, similaire aux types primitifs ( `int` , `double` , etc.).</span><span class="sxs-lookup"><span data-stu-id="10853-129">It logically represents a single value, similar to primitive types (`int`, `double`, etc.).</span></span>

- <span data-ttu-id="10853-130">Elle a une taille d’instance inférieure à 16 octets.</span><span class="sxs-lookup"><span data-stu-id="10853-130">It has an instance size under 16 bytes.</span></span>

- <span data-ttu-id="10853-131">Elle est immuable.</span><span class="sxs-lookup"><span data-stu-id="10853-131">It is immutable.</span></span>

- <span data-ttu-id="10853-132">Il n’est pas nécessaire d’effectuer fréquemment des Boxing.</span><span class="sxs-lookup"><span data-stu-id="10853-132">It will not have to be boxed frequently.</span></span>

 <span data-ttu-id="10853-133">Dans tous les autres cas, vous devez définir vos types en tant que classes.</span><span class="sxs-lookup"><span data-stu-id="10853-133">In all other cases, you should define your types as classes.</span></span>
