# <a name="extension-methods"></a><span data-ttu-id="82b7f-102">Méthodes d'extension</span><span class="sxs-lookup"><span data-stu-id="82b7f-102">Extension Methods</span></span>
<span data-ttu-id="82b7f-103">Les méthodes d’extension sont une fonctionnalité de langage qui permet d’appeler des méthodes statiques à l’aide de la syntaxe d’appel de méthode d’instance.</span><span class="sxs-lookup"><span data-stu-id="82b7f-103">Extension methods are a language feature that allows static methods to be called using instance method call syntax.</span></span> <span data-ttu-id="82b7f-104">Ces méthodes doivent accepter au moins un paramètre, qui représente l’instance sur laquelle la méthode doit fonctionner.</span><span class="sxs-lookup"><span data-stu-id="82b7f-104">These methods must take at least one parameter, which represents the instance the method is to operate on.</span></span>  
  
 <span data-ttu-id="82b7f-105">La classe qui définit ces méthodes d’extension est appelée classe « sponsor » et doit être déclarée comme static.</span><span class="sxs-lookup"><span data-stu-id="82b7f-105">The class that defines such extension methods is referred to as the "sponsor" class, and it must be declared as static.</span></span> <span data-ttu-id="82b7f-106">Pour utiliser des méthodes d’extension, vous devez importer l’espace de noms définissant la classe sponsor.</span><span class="sxs-lookup"><span data-stu-id="82b7f-106">To use extension methods, one must import the namespace defining the sponsor class.</span></span>  
  
 <span data-ttu-id="82b7f-107">**X AVOID** leurs intentions soient amicales définissant les méthodes d’extension, en particulier sur les types que vous ne possédez pas.</span><span class="sxs-lookup"><span data-stu-id="82b7f-107">**X AVOID** frivolously defining extension methods, especially on types you don’t own.</span></span>  
  
 <span data-ttu-id="82b7f-108">Si vous disposez d’un code source de type, envisagez plutôt d’utiliser des méthodes d’instance normales.</span><span class="sxs-lookup"><span data-stu-id="82b7f-108">If you do own source code of a type, consider using regular instance methods instead.</span></span> <span data-ttu-id="82b7f-109">Si vous ne possédez pas et que vous souhaitez ajouter une méthode, soyez très vigilant.</span><span class="sxs-lookup"><span data-stu-id="82b7f-109">If you don’t own, and you want to add a method, be very careful.</span></span> <span data-ttu-id="82b7f-110">La libéralisation de l’utilisation des méthodes d’extension risque d’encombrer les API de types qui n’ont pas été conçues pour avoir ces méthodes.</span><span class="sxs-lookup"><span data-stu-id="82b7f-110">Liberal use of extension methods has the potential of cluttering APIs of types that were not designed to have these methods.</span></span>  
  
 <span data-ttu-id="82b7f-111">**✓ CONSIDER** à l’aide des méthodes d’extension dans un des scénarios suivants :</span><span class="sxs-lookup"><span data-stu-id="82b7f-111">**✓ CONSIDER** using extension methods in any of the following scenarios:</span></span>  
  
- <span data-ttu-id="82b7f-112">Pour fournir des fonctionnalités d’assistance relatives à chaque implémentation d’une interface, si cette fonctionnalité peut être écrite en termes d’interface principale.</span><span class="sxs-lookup"><span data-stu-id="82b7f-112">To provide helper functionality relevant to every implementation of an interface, if said functionality can be written in terms of the core interface.</span></span> <span data-ttu-id="82b7f-113">Cela est dû au fait que les implémentations concrètes ne peuvent pas être assignées à des interfaces.</span><span class="sxs-lookup"><span data-stu-id="82b7f-113">This is because concrete implementations cannot otherwise be assigned to interfaces.</span></span> <span data-ttu-id="82b7f-114">Par exemple, les opérateurs de `LINQ to Objects` sont implémentés en tant que méthodes d’extension pour tous les types de <xref:System.Collections.Generic.IEnumerable%601>.</span><span class="sxs-lookup"><span data-stu-id="82b7f-114">For example, the `LINQ to Objects` operators are implemented as extension methods for all <xref:System.Collections.Generic.IEnumerable%601> types.</span></span> <span data-ttu-id="82b7f-115">Ainsi, toute implémentation de `IEnumerable<>` est automatiquement compatible LINQ.</span><span class="sxs-lookup"><span data-stu-id="82b7f-115">Thus, any `IEnumerable<>` implementation is automatically LINQ-enabled.</span></span>  
  
- <span data-ttu-id="82b7f-116">Quand une méthode d’instance introduit une dépendance sur un certain type, mais une telle dépendance rompt les règles de gestion des dépendances.</span><span class="sxs-lookup"><span data-stu-id="82b7f-116">When an instance method would introduce a dependency on some type, but such a dependency would break dependency management rules.</span></span> <span data-ttu-id="82b7f-117">Par exemple, une dépendance de <xref:System.String> à <xref:System.Uri?displayProperty=nameWithType> n’est probablement pas souhaitable, donc `String.ToUri()` méthode d’instance qui retourne `System.Uri` serait la mauvaise conception d’une perspective de gestion des dépendances.</span><span class="sxs-lookup"><span data-stu-id="82b7f-117">For example, a dependency from <xref:System.String> to <xref:System.Uri?displayProperty=nameWithType> is probably not desirable, and so `String.ToUri()` instance method returning `System.Uri` would be the wrong design from a dependency management perspective.</span></span> <span data-ttu-id="82b7f-118">Une méthode d’extension statique `Uri.ToUri(this string str)` retourner `System.Uri` serait une conception bien meilleure.</span><span class="sxs-lookup"><span data-stu-id="82b7f-118">A static extension method `Uri.ToUri(this string str)` returning `System.Uri` would be a much better design.</span></span>  
  
 <span data-ttu-id="82b7f-119">**X AVOID** définition des méthodes d’extension sur <xref:System.Object?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="82b7f-119">**X AVOID** defining extension methods on <xref:System.Object?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="82b7f-120">Visual Basic les utilisateurs ne seront pas en mesure d’appeler ces méthodes sur des références d’objet à l’aide de la syntaxe de méthode d’extension.</span><span class="sxs-lookup"><span data-stu-id="82b7f-120">Visual Basic users will not be able to call such methods on object references using the extension method syntax.</span></span> <span data-ttu-id="82b7f-121">Visual Basic ne prend pas en charge l’appel de telles méthodes, car, dans Visual Basic, la déclaration d’une référence en tant qu’objet force la liaison tardive de tous les appels de méthode sur celui-ci (le membre réel appelé est déterminé au moment de l’exécution), tandis que les liaisons aux méthodes d’extension sont déterminées à au moment de la compilation (liaison anticipée).</span><span class="sxs-lookup"><span data-stu-id="82b7f-121">Visual Basic does not support calling such methods because, in Visual Basic, declaring a reference as Object forces all method invocations on it to be late bound (actual member called is determined at runtime), while bindings to extension methods are determined at compile-time (early bound).</span></span>  
  
 <span data-ttu-id="82b7f-122">Notez que l’instruction s’applique à d’autres langages où le même comportement de liaison est présent, ou où les méthodes d’extension ne sont pas prises en charge.</span><span class="sxs-lookup"><span data-stu-id="82b7f-122">Note that the guideline applies to other languages where the same binding behavior is present, or where extension methods are not supported.</span></span>  
  
 <span data-ttu-id="82b7f-123">**X DO NOT** méthodes d’extension dans le même espace de noms en tant que le type étendu est utilisée pour ajouter des méthodes aux interfaces ou pour la gestion des dépendances.</span><span class="sxs-lookup"><span data-stu-id="82b7f-123">**X DO NOT** put extension methods in the same namespace as the extended type unless it is for adding methods to interfaces or for dependency management.</span></span>  
  
 <span data-ttu-id="82b7f-124">**X AVOID** définissant deux ou plusieurs méthodes d’extension avec la même signature, même s’ils résident dans différents espaces de noms.</span><span class="sxs-lookup"><span data-stu-id="82b7f-124">**X AVOID** defining two or more extension methods with the same signature, even if they reside in different namespaces.</span></span>  
  
 <span data-ttu-id="82b7f-125">**✓ CONSIDER** définition des méthodes d’extension dans le même espace de noms en tant que le type étendu si le type est une interface et si les méthodes d’extension sont destinés à être utilisés dans la plupart des cas.</span><span class="sxs-lookup"><span data-stu-id="82b7f-125">**✓ CONSIDER** defining extension methods in the same namespace as the extended type if the type is an interface and if the extension methods are meant to be used in most or all cases.</span></span>  
  
 <span data-ttu-id="82b7f-126">**X DO NOT** définir des méthodes d’extension mise en œuvre d’une fonctionnalité dans les espaces de noms normalement avec d’autres fonctionnalités.</span><span class="sxs-lookup"><span data-stu-id="82b7f-126">**X DO NOT** define extension methods implementing a feature in namespaces normally associated with other features.</span></span> <span data-ttu-id="82b7f-127">Au lieu de cela, définissez-les dans l’espace de noms associé à la fonctionnalité à laquelle ils appartiennent.</span><span class="sxs-lookup"><span data-stu-id="82b7f-127">Instead, define them in the namespace associated with the feature they belong to.</span></span>  
  
 <span data-ttu-id="82b7f-128">**X AVOID** générique d’affectation de noms d’espaces de noms dédié aux méthodes d’extension (par exemple, « Extensions »).</span><span class="sxs-lookup"><span data-stu-id="82b7f-128">**X AVOID** generic naming of namespaces dedicated to extension methods (e.g., "Extensions").</span></span> <span data-ttu-id="82b7f-129">Utilisez un nom descriptif (par exemple, « routage ») à la place.</span><span class="sxs-lookup"><span data-stu-id="82b7f-129">Use a descriptive name (e.g., "Routing") instead.</span></span>
