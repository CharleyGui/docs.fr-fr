# <a name="general-naming-conventions"></a><span data-ttu-id="47436-102">Conventions générales d'affectation de noms</span><span class="sxs-lookup"><span data-stu-id="47436-102">General Naming Conventions</span></span>
<span data-ttu-id="47436-103">Cette section décrit les conventions d’affectation de noms générales relatives au choix de mots, des instructions sur l’utilisation des abréviations et des acronymes, ainsi que des recommandations sur la façon d’éviter d’utiliser des noms spécifiques à une langue.</span><span class="sxs-lookup"><span data-stu-id="47436-103">This section describes general naming conventions that relate to word choice, guidelines on using abbreviations and acronyms, and recommendations on how to avoid using language-specific names.</span></span>

## <a name="word-choice"></a><span data-ttu-id="47436-104">Choix de mots</span><span class="sxs-lookup"><span data-stu-id="47436-104">Word Choice</span></span>
 <span data-ttu-id="47436-105">✔️ Choisissez des noms d’identificateur facilement lisibles.</span><span class="sxs-lookup"><span data-stu-id="47436-105">✔️ DO choose easily readable identifier names.</span></span>

 <span data-ttu-id="47436-106">Par exemple, une propriété nommée `HorizontalAlignment` est plus lisible que `AlignmentHorizontal`.</span><span class="sxs-lookup"><span data-stu-id="47436-106">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>

 <span data-ttu-id="47436-107">✔️ préférez la lisibilité par rapport à la concision.</span><span class="sxs-lookup"><span data-stu-id="47436-107">✔️ DO favor readability over brevity.</span></span>

 <span data-ttu-id="47436-108">Le nom de la propriété `CanScrollHorizontally` est mieux que `ScrollableX` (une référence obscurcie à l’axe X).</span><span class="sxs-lookup"><span data-stu-id="47436-108">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>

 <span data-ttu-id="47436-109">❌ n’utilisez pas de traits de soulignement, de traits d’Union ou tout autre caractère non alphanumérique.</span><span class="sxs-lookup"><span data-stu-id="47436-109">❌ DO NOT use underscores, hyphens, or any other nonalphanumeric characters.</span></span>

 <span data-ttu-id="47436-110">❌ n’utilisez pas la notation hongroise.</span><span class="sxs-lookup"><span data-stu-id="47436-110">❌ DO NOT use Hungarian notation.</span></span>

 <span data-ttu-id="47436-111">❌ éviter d’utiliser des identificateurs qui sont en conflit avec des mots clés de langages de programmation largement utilisés.</span><span class="sxs-lookup"><span data-stu-id="47436-111">❌ AVOID using identifiers that conflict with keywords of widely used programming languages.</span></span>

 <span data-ttu-id="47436-112">Conformément à la règle 4 du Common Language Specification (CLS), tous les langages conformes doivent fournir un mécanisme qui permet d’accéder à des éléments nommés qui utilisent un mot clé de ce langage comme identificateur.</span><span class="sxs-lookup"><span data-stu-id="47436-112">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="47436-113">C#, par exemple, utilise le signe @ comme mécanisme d’échappement dans ce cas.</span><span class="sxs-lookup"><span data-stu-id="47436-113">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="47436-114">Toutefois, il est toujours judicieux d’éviter les mots clés courants, car il est bien plus difficile d’utiliser une méthode avec la séquence d’échappement que l’autre sans lui.</span><span class="sxs-lookup"><span data-stu-id="47436-114">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>

## <a name="using-abbreviations-and-acronyms"></a><span data-ttu-id="47436-115">Utilisation des abréviations et des acronymes</span><span class="sxs-lookup"><span data-stu-id="47436-115">Using Abbreviations and Acronyms</span></span>
 <span data-ttu-id="47436-116">❌ n’utilisez pas d’abréviations ou de contractions dans le cadre des noms d’identificateur.</span><span class="sxs-lookup"><span data-stu-id="47436-116">❌ DO NOT use abbreviations or contractions as part of identifier names.</span></span>

 <span data-ttu-id="47436-117">Par exemple, utilisez `GetWindow` plutôt que `GetWin`.</span><span class="sxs-lookup"><span data-stu-id="47436-117">For example, use `GetWindow` rather than `GetWin`.</span></span>

 <span data-ttu-id="47436-118">❌ n’utilisez pas les acronymes qui ne sont pas largement acceptés, et même s’ils le sont, uniquement lorsque cela est nécessaire.</span><span class="sxs-lookup"><span data-stu-id="47436-118">❌ DO NOT use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>

 <span data-ttu-id="47436-121">Par exemple, `GetLength` est un nom plus approprié que `GetInt`.</span><span class="sxs-lookup"><span data-stu-id="47436-121">For example, `GetLength` is a better name than `GetInt`.</span></span>

 <span data-ttu-id="47436-122">✔️ Utilisez un nom de type CLR générique, plutôt qu’un nom propre au langage, dans les rares cas où un identificateur n’a aucune signification sémantique au-delà de son type.</span><span class="sxs-lookup"><span data-stu-id="47436-122">✔️ DO use a generic CLR type name, rather than a language-specific name, in the rare cases when an identifier has no semantic meaning beyond its type.</span></span>
