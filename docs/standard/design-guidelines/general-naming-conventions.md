# <a name="general-naming-conventions"></a><span data-ttu-id="06b1b-102">Conventions générales d'affectation de noms</span><span class="sxs-lookup"><span data-stu-id="06b1b-102">General Naming Conventions</span></span>
<span data-ttu-id="06b1b-103">Cette section décrit les conventions d’affectation de noms générales relatives au choix de mots, des instructions sur l’utilisation des abréviations et des acronymes, ainsi que des recommandations sur la façon d’éviter d’utiliser des noms spécifiques à une langue.</span><span class="sxs-lookup"><span data-stu-id="06b1b-103">This section describes general naming conventions that relate to word choice, guidelines on using abbreviations and acronyms, and recommendations on how to avoid using language-specific names.</span></span>  
  
## <a name="word-choice"></a><span data-ttu-id="06b1b-104">Choix de mots</span><span class="sxs-lookup"><span data-stu-id="06b1b-104">Word Choice</span></span>  
 <span data-ttu-id="06b1b-105">**✓ DO** choisir des noms d’identificateurs lisibles.</span><span class="sxs-lookup"><span data-stu-id="06b1b-105">**✓ DO** choose easily readable identifier names.</span></span>  
  
 <span data-ttu-id="06b1b-106">Par exemple, une propriété nommée `HorizontalAlignment` est plus lisible que `AlignmentHorizontal`.</span><span class="sxs-lookup"><span data-stu-id="06b1b-106">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>  
  
 <span data-ttu-id="06b1b-107">**✓ DO** privilégier la lisibilité des raisons de concision.</span><span class="sxs-lookup"><span data-stu-id="06b1b-107">**✓ DO** favor readability over brevity.</span></span>  
  
 <span data-ttu-id="06b1b-108">Le nom de la propriété `CanScrollHorizontally` est mieux que `ScrollableX` (une référence obscurcie à l’axe X).</span><span class="sxs-lookup"><span data-stu-id="06b1b-108">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>  
  
 <span data-ttu-id="06b1b-109">**X DO NOT** utiliser d’autres caractères non alphanumériques, des traits d’union ou des traits de soulignement.</span><span class="sxs-lookup"><span data-stu-id="06b1b-109">**X DO NOT** use underscores, hyphens, or any other nonalphanumeric characters.</span></span>  
  
 <span data-ttu-id="06b1b-110">**X DO NOT** utilisent une notation hongroise.</span><span class="sxs-lookup"><span data-stu-id="06b1b-110">**X DO NOT** use Hungarian notation.</span></span>  
  
 <span data-ttu-id="06b1b-111">**X AVOID** à l’aide d’identificateurs qui entrent en conflit avec les mots clés de largement utilisé des langages de programmation.</span><span class="sxs-lookup"><span data-stu-id="06b1b-111">**X AVOID** using identifiers that conflict with keywords of widely used programming languages.</span></span>  
  
 <span data-ttu-id="06b1b-112">Conformément à la règle 4 du Common Language Specification (CLS), tous les langages conformes doivent fournir un mécanisme qui permet d’accéder à des éléments nommés qui utilisent un mot clé de ce langage comme identificateur.</span><span class="sxs-lookup"><span data-stu-id="06b1b-112">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="06b1b-113">C#, par exemple, utilise le signe @ comme mécanisme d’échappement dans ce cas.</span><span class="sxs-lookup"><span data-stu-id="06b1b-113">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="06b1b-114">Toutefois, il est toujours judicieux d’éviter les mots clés courants, car il est bien plus difficile d’utiliser une méthode avec la séquence d’échappement que l’autre sans lui.</span><span class="sxs-lookup"><span data-stu-id="06b1b-114">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>  
  
## <a name="using-abbreviations-and-acronyms"></a><span data-ttu-id="06b1b-115">Utilisation des abréviations et des acronymes</span><span class="sxs-lookup"><span data-stu-id="06b1b-115">Using Abbreviations and Acronyms</span></span>  
 <span data-ttu-id="06b1b-116">**X DO NOT** utiliser des abréviations ou contractions comme faisant partie des noms d’identificateur.</span><span class="sxs-lookup"><span data-stu-id="06b1b-116">**X DO NOT** use abbreviations or contractions as part of identifier names.</span></span>  
  
 <span data-ttu-id="06b1b-117">Par exemple, utilisez `GetWindow` plutôt que `GetWin`.</span><span class="sxs-lookup"><span data-stu-id="06b1b-117">For example, use `GetWindow` rather than `GetWin`.</span></span>  
  
 <span data-ttu-id="06b1b-118">**X DO NOT** utiliser des acronymes ne sont pas largement reconnue et même si elles sont uniquement lorsque cela est nécessaire.</span><span class="sxs-lookup"><span data-stu-id="06b1b-118">**X DO NOT** use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>

 <span data-ttu-id="06b1b-120">**✓ DO** utiliser des noms sémantiquement intéressants plutôt que des mots clés spécifiques à la langue pour les noms de type.</span><span class="sxs-lookup"><span data-stu-id="06b1b-120">**✓ DO** use semantically interesting names rather than language-specific keywords for type names.</span></span>  
  
 <span data-ttu-id="06b1b-121">Par exemple, `GetLength` est un nom plus approprié que `GetInt`.</span><span class="sxs-lookup"><span data-stu-id="06b1b-121">For example, `GetLength` is a better name than `GetInt`.</span></span>  
  
 <span data-ttu-id="06b1b-122">**✓ DO** utiliser un nom de type CLR générique, plutôt que d’un nom spécifique au langage, dans les rares cas où un identificateur n’a aucune signification sémantique au-delà de son type.</span><span class="sxs-lookup"><span data-stu-id="06b1b-122">**✓ DO** use a generic CLR type name, rather than a language-specific name, in the rare cases when an identifier has no semantic meaning beyond its type.</span></span>

 <span data-ttu-id="06b1b-189">**✓ DO** préférez l’ajout d’un suffixe au lieu d’un préfixe pour indiquer une nouvelle version d’une API existante.</span><span class="sxs-lookup"><span data-stu-id="06b1b-189">**✓ DO** prefer adding a suffix rather than a prefix to indicate a new version of an existing API.</span></span>
