 <span data-ttu-id="bc837-105">✔️ NE choisissez facilement lisible noms d’identification.</span><span class="sxs-lookup"><span data-stu-id="bc837-105">✔️ DO choose easily readable identifier names.</span></span>

 <span data-ttu-id="bc837-106">Par exemple, une `HorizontalAlignment` propriété nommée est `AlignmentHorizontal`plus lisible en anglais que .</span><span class="sxs-lookup"><span data-stu-id="bc837-106">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>

 <span data-ttu-id="bc837-107">✔️ ne favorisent la lisibilité sur la brièveté.</span><span class="sxs-lookup"><span data-stu-id="bc837-107">✔️ DO favor readability over brevity.</span></span>

 <span data-ttu-id="bc837-108">Le nom `CanScrollHorizontally` de `ScrollableX` la propriété est meilleur que (une référence obscure à l’axe X).</span><span class="sxs-lookup"><span data-stu-id="bc837-108">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>

 <span data-ttu-id="bc837-109">❌NE PAS utiliser les soulignements, les traits d’union, ou tout autre caractère non anphanumérique.</span><span class="sxs-lookup"><span data-stu-id="bc837-109">❌ DO NOT use underscores, hyphens, or any other nonalphanumeric characters.</span></span>

 <span data-ttu-id="bc837-110">❌NE PAS utiliser la notation hongroise.</span><span class="sxs-lookup"><span data-stu-id="bc837-110">❌ DO NOT use Hungarian notation.</span></span>

 <span data-ttu-id="bc837-111">❌AVOID à l’aide d’identifiants qui entrent en conflit avec les mots clés des langages de programmation largement utilisés.</span><span class="sxs-lookup"><span data-stu-id="bc837-111">❌ AVOID using identifiers that conflict with keywords of widely used programming languages.</span></span>

 <span data-ttu-id="bc837-112">Conformément à la règle 4 de la spécification linguistique commune (CLS), toutes les langues conformes doivent fournir un mécanisme qui permet l’accès à des éléments nommés qui utilisent un mot clé de cette langue comme identifiant.</span><span class="sxs-lookup"><span data-stu-id="bc837-112">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="bc837-113">C, par exemple, utilise le signe de l’inscription comme mécanisme d’évacuation dans ce cas.</span><span class="sxs-lookup"><span data-stu-id="bc837-113">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="bc837-114">Cependant, il est toujours une bonne idée d’éviter les mots clés communs, car il est beaucoup plus difficile d’utiliser une méthode avec la séquence d’évacuation que l’un sans elle.</span><span class="sxs-lookup"><span data-stu-id="bc837-114">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>

 <span data-ttu-id="bc837-117">Par exemple, `GetWindow` utiliser `GetWin`plutôt que .</span><span class="sxs-lookup"><span data-stu-id="bc837-117">For example, use `GetWindow` rather than `GetWin`.</span></span>

 <span data-ttu-id="bc837-118">❌NE PAS utiliser les acronymes qui ne sont pas largement acceptés, et même si elles sont, seulement si nécessaire.</span><span class="sxs-lookup"><span data-stu-id="bc837-118">❌ DO NOT use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>
