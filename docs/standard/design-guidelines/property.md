# <a name="property-design"></a><span data-ttu-id="02807-102">Conception des propriétés</span><span class="sxs-lookup"><span data-stu-id="02807-102">Property Design</span></span>
<span data-ttu-id="02807-103">Bien que les propriétés soient techniquement très similaires aux méthodes, elles sont très différentes en termes de scénarios d’utilisation.</span><span class="sxs-lookup"><span data-stu-id="02807-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="02807-104">Elles doivent être considérées comme des champs intelligents.</span><span class="sxs-lookup"><span data-stu-id="02807-104">They should be seen as smart fields.</span></span> <span data-ttu-id="02807-105">Ils ont la syntaxe d’appel des champs et la flexibilité des méthodes.</span><span class="sxs-lookup"><span data-stu-id="02807-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>  
  
 <span data-ttu-id="02807-106">**✓ DO** créer des propriétés get-only si l’appelant ne doit pas être en mesure de modifier la valeur de la propriété.</span><span class="sxs-lookup"><span data-stu-id="02807-106">**✓ DO** create get-only properties if the caller should not be able to change the value of the property.</span></span>  
  
 <span data-ttu-id="02807-107">Gardez à l’esprit que si le type de la propriété est un type référence mutable, la valeur de la propriété peut être modifiée même si la propriété est en extraction seule.</span><span class="sxs-lookup"><span data-stu-id="02807-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>  
  
 <span data-ttu-id="02807-108">**X DO NOT** fournissent des propriétés ou propriétés à définir uniquement avec la méthode setter ayant le plus large d’accessibilité de l’accesseur Get.</span><span class="sxs-lookup"><span data-stu-id="02807-108">**X DO NOT** provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>  
  
 <span data-ttu-id="02807-109">Par exemple, n’utilisez pas de propriétés avec un accesseur Set public et un accesseur Get protégé.</span><span class="sxs-lookup"><span data-stu-id="02807-109">For example, do not use properties with a public setter and a protected getter.</span></span>  
  
 <span data-ttu-id="02807-110">Si l’accesseur Get de la propriété ne peut pas être fourni, implémentez la fonctionnalité comme une méthode à la place.</span><span class="sxs-lookup"><span data-stu-id="02807-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="02807-111">Envisagez de démarrer le nom de la méthode avec `Set` et suivez ce que vous auriez nommé la propriété.</span><span class="sxs-lookup"><span data-stu-id="02807-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="02807-112">Par exemple, <xref:System.AppDomain> a une méthode appelée `SetCachePath` au lieu d’avoir une propriété définie uniquement appelée `CachePath`.</span><span class="sxs-lookup"><span data-stu-id="02807-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>  
  
 <span data-ttu-id="02807-113">**✓ DO** fournissent des valeurs par défaut raisonnables pour toutes les propriétés, garantissant que les valeurs par défaut n’entraînent pas une faille de sécurité ou du code très inefficace.</span><span class="sxs-lookup"><span data-stu-id="02807-113">**✓ DO** provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>  
  
 <span data-ttu-id="02807-114">**✓ DO** permettent d’être définie dans n’importe quel ordre, même si cela aboutit à un état temporaire non valide de l’objet.</span><span class="sxs-lookup"><span data-stu-id="02807-114">**✓ DO** allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>  
  
 <span data-ttu-id="02807-115">Il est courant que deux propriétés ou plus soient associées à un point où certaines valeurs d’une propriété peuvent ne pas être valides étant donné les valeurs d’autres propriétés sur le même objet.</span><span class="sxs-lookup"><span data-stu-id="02807-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="02807-116">Dans ce cas, les exceptions résultant de l’état non valide doivent être reportées jusqu’à ce que les propriétés interdépendantes soient utilisées ensemble par l’objet.</span><span class="sxs-lookup"><span data-stu-id="02807-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>  
  
 <span data-ttu-id="02807-117">**✓ DO** conserver la valeur précédente si un accesseur Set de propriété lève une exception.</span><span class="sxs-lookup"><span data-stu-id="02807-117">**✓ DO** preserve the previous value if a property setter throws an exception.</span></span>  
  
 <span data-ttu-id="02807-118">**X AVOID** lever des exceptions à partir des accesseurs Get de propriété.</span><span class="sxs-lookup"><span data-stu-id="02807-118">**X AVOID** throwing exceptions from property getters.</span></span>  
  
 <span data-ttu-id="02807-119">Les accesseurs get de propriété doivent être des opérations simples et ne doivent pas avoir de conditions préalables.</span><span class="sxs-lookup"><span data-stu-id="02807-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="02807-120">Si un accesseur Get peut lever une exception, il doit probablement être remanié pour être une méthode.</span><span class="sxs-lookup"><span data-stu-id="02807-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="02807-121">Notez que cette règle ne s’applique pas aux indexeurs, où nous attendons des exceptions en raison de la validation des arguments.</span><span class="sxs-lookup"><span data-stu-id="02807-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>  
  
### <a name="indexed-property-design"></a><span data-ttu-id="02807-122">Conception des propriétés indexées</span><span class="sxs-lookup"><span data-stu-id="02807-122">Indexed Property Design</span></span>  
 <span data-ttu-id="02807-123">Une propriété indexée est une propriété spéciale qui peut avoir des paramètres et qui peut être appelée avec une syntaxe spéciale similaire à l’indexation de tableau.</span><span class="sxs-lookup"><span data-stu-id="02807-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>  
  
 <span data-ttu-id="02807-124">Les propriétés indexées sont communément appelées indexeurs.</span><span class="sxs-lookup"><span data-stu-id="02807-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="02807-125">Les indexeurs ne doivent être utilisés que dans les API qui permettent d’accéder aux éléments d’une collection logique.</span><span class="sxs-lookup"><span data-stu-id="02807-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="02807-126">Par exemple, une chaîne est une collection de caractères, et l’indexeur sur <xref:System.String?displayProperty=nameWithType> a été ajouté pour accéder à ses caractères.</span><span class="sxs-lookup"><span data-stu-id="02807-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>  
  
 <span data-ttu-id="02807-127">**✓ CONSIDER** des indexeurs pour fournir l’accès aux données stockées dans un tableau interne.</span><span class="sxs-lookup"><span data-stu-id="02807-127">**✓ CONSIDER** using indexers to provide access to data stored in an internal array.</span></span>  
  
 <span data-ttu-id="02807-128">**✓ CONSIDER** fournir des indexeurs sur les types de collections d’éléments.</span><span class="sxs-lookup"><span data-stu-id="02807-128">**✓ CONSIDER** providing indexers on types representing collections of items.</span></span>  
  
 <span data-ttu-id="02807-129">**X AVOID** à l’aide des propriétés avec plusieurs paramètres indexées.</span><span class="sxs-lookup"><span data-stu-id="02807-129">**X AVOID** using indexed properties with more than one parameter.</span></span>  
  
 <span data-ttu-id="02807-130">Si la conception requiert plusieurs paramètres, reconsidérez si la propriété représente vraiment un accesseur à une collection logique.</span><span class="sxs-lookup"><span data-stu-id="02807-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="02807-131">Si ce n’est pas le cas, utilisez plutôt des méthodes.</span><span class="sxs-lookup"><span data-stu-id="02807-131">If it does not, use methods instead.</span></span> <span data-ttu-id="02807-132">Envisagez de démarrer le nom de la méthode avec `Get` ou `Set`.</span><span class="sxs-lookup"><span data-stu-id="02807-132">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="02807-133">**X AVOID** indexeurs avec les types de paramètres autres que <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, ou d’un enum.</span><span class="sxs-lookup"><span data-stu-id="02807-133">**X AVOID** indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>  
  
 <span data-ttu-id="02807-134">Si la conception requiert d’autres types de paramètres, réévaluez fortement si l’API représente vraiment un accesseur à une collection logique.</span><span class="sxs-lookup"><span data-stu-id="02807-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="02807-135">Si ce n’est pas le cas, utilisez une méthode.</span><span class="sxs-lookup"><span data-stu-id="02807-135">If it does not, use a method.</span></span> <span data-ttu-id="02807-136">Envisagez de démarrer le nom de la méthode avec `Get` ou `Set`.</span><span class="sxs-lookup"><span data-stu-id="02807-136">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="02807-137">**✓ DO** utiliser le nom `Item` pour des propriétés indexées, sauf s’il existe un meilleur nom (par exemple, consultez la <xref:System.String.Chars%2A> propriété sur `System.String`).</span><span class="sxs-lookup"><span data-stu-id="02807-137">**✓ DO** use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>  
  
 <span data-ttu-id="02807-138">Dans C#, les indexeurs sont par défaut un élément nommé.</span><span class="sxs-lookup"><span data-stu-id="02807-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="02807-139">Le <xref:System.Runtime.CompilerServices.IndexerNameAttribute> peut être utilisé pour personnaliser ce nom.</span><span class="sxs-lookup"><span data-stu-id="02807-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>  
  
 <span data-ttu-id="02807-140">**X DO NOT** fournissent à la fois un indexeur et des méthodes qui sont sémantiquement équivalents.</span><span class="sxs-lookup"><span data-stu-id="02807-140">**X DO NOT** provide both an indexer and methods that are semantically equivalent.</span></span>  
  
 <span data-ttu-id="02807-141">**X DO NOT** fournissent plusieurs familles d’indexeurs surchargés dans un type.</span><span class="sxs-lookup"><span data-stu-id="02807-141">**X DO NOT** provide more than one family of overloaded indexers in one type.</span></span>  
  
 <span data-ttu-id="02807-142">Cela est appliqué par le C# compilateur.</span><span class="sxs-lookup"><span data-stu-id="02807-142">This is enforced by the C# compiler.</span></span>  
  
 <span data-ttu-id="02807-143">**X DO NOT** différent de l’utilisation des propriétés indexées.</span><span class="sxs-lookup"><span data-stu-id="02807-143">**X DO NOT** use nondefault indexed properties.</span></span>  
  
 <span data-ttu-id="02807-144">Cela est appliqué par le C# compilateur.</span><span class="sxs-lookup"><span data-stu-id="02807-144">This is enforced by the C# compiler.</span></span>  
  
### <a name="property-change-notification-events"></a><span data-ttu-id="02807-145">Événements de notification de modification de propriété</span><span class="sxs-lookup"><span data-stu-id="02807-145">Property Change Notification Events</span></span>  
 <span data-ttu-id="02807-146">Parfois, il est utile de fournir un événement pour notifier l’utilisateur des modifications apportées à une valeur de propriété.</span><span class="sxs-lookup"><span data-stu-id="02807-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="02807-147">Par exemple, `System.Windows.Forms.Control` déclenche un événement `TextChanged` après la modification de la valeur de sa propriété `Text`.</span><span class="sxs-lookup"><span data-stu-id="02807-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>  
  
 <span data-ttu-id="02807-148">**✓ CONSIDER** modifier les déclenchement d’événements de notification lorsque les valeurs de propriété dans les API de niveau supérieur (composants de concepteur généralement) sont modifiés.</span><span class="sxs-lookup"><span data-stu-id="02807-148">**✓ CONSIDER** raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>  
  
 <span data-ttu-id="02807-149">S’il existe un bon scénario pour qu’un utilisateur sache quand une propriété d’un objet change, l’objet doit déclencher un événement de notification de modification pour la propriété.</span><span class="sxs-lookup"><span data-stu-id="02807-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>  
  
 <span data-ttu-id="02807-150">Toutefois, il est peu probable qu’il soit nécessaire de déclencher de tels événements pour les API de bas niveau, telles que les types de base ou les collections.</span><span class="sxs-lookup"><span data-stu-id="02807-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="02807-151">Par exemple, <xref:System.Collections.Generic.List%601> ne déclencherait pas de tels événements lorsqu’un nouvel élément est ajouté à la liste et que la propriété `Count` change.</span><span class="sxs-lookup"><span data-stu-id="02807-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>  
  
 <span data-ttu-id="02807-152">**✓ CONSIDER** déclenchement modifier les événements de notification lorsque la valeur d’une propriété modifiée via la force externe.</span><span class="sxs-lookup"><span data-stu-id="02807-152">**✓ CONSIDER** raising change notification events when the value of a property changes via external forces.</span></span>  
  
 <span data-ttu-id="02807-153">Si une valeur de propriété change via une force externe (d’une manière autre que en appelant des méthodes sur l’objet), les événements Raise indiquent au développeur que la valeur change et a changé.</span><span class="sxs-lookup"><span data-stu-id="02807-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="02807-154">Le `Text` propriété d’un contrôle zone de texte en est un bon exemple.</span><span class="sxs-lookup"><span data-stu-id="02807-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="02807-155">Lorsque l’utilisateur tape du texte dans un `TextBox`, la valeur de la propriété change automatiquement.</span><span class="sxs-lookup"><span data-stu-id="02807-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
