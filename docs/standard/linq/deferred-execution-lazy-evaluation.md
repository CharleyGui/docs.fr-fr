# <a name="deferred-execution-and-lazy-evaluation-linq-to-xml"></a><span data-ttu-id="1010a-103">Exécution différée et évaluation différée (LINQ to XML)</span><span class="sxs-lookup"><span data-stu-id="1010a-103">Deferred execution and lazy evaluation (LINQ to XML)</span></span>

<span data-ttu-id="1010a-104">Les opérations de requête et d'axe sont souvent implémentées pour utiliser l'exécution différée.</span><span class="sxs-lookup"><span data-stu-id="1010a-104">Query and axis operations are often implemented to use deferred execution.</span></span> <span data-ttu-id="1010a-105">Cet article explique les conditions requises et les avantages de l’exécution différée, ainsi que certaines considérations relatives à l’implémentation.</span><span class="sxs-lookup"><span data-stu-id="1010a-105">This article explains the requirements and advantages of deferred execution, and some implementation considerations.</span></span>

## <a name="deferred-execution"></a><span data-ttu-id="1010a-106">Exécution différée</span><span class="sxs-lookup"><span data-stu-id="1010a-106">Deferred execution</span></span>

<span data-ttu-id="1010a-107">Le terme "exécution différée" signifie que l’évaluation d’une expression est retardée jusqu’à ce que sa valeur *réalisée* soit réellement nécessaire.</span><span class="sxs-lookup"><span data-stu-id="1010a-107">Deferred execution means that the evaluation of an expression is delayed until its *realized* value is actually required.</span></span> <span data-ttu-id="1010a-108">L'exécution différée peut améliorer sensiblement les performances lorsque vous devez manipuler de grandes collectes de données, en particulier dans les programmes qui contiennent une série de manipulations ou de requêtes chaînées.</span><span class="sxs-lookup"><span data-stu-id="1010a-108">Deferred execution can greatly improve performance when you have to manipulate large data collections, especially in programs that contain a series of chained queries or manipulations.</span></span> <span data-ttu-id="1010a-109">Dans le meilleur cas, l'exécution différée autorise une seule itération dans la collection source.</span><span class="sxs-lookup"><span data-stu-id="1010a-109">In the best case, deferred execution enables only a single iteration through the source collection.</span></span>

<span data-ttu-id="1010a-110">Les technologies LINQ utilisent beaucoup l'exécution différée dans les membres des classes <xref:System.Linq?displayProperty=nameWithType> principales et dans les méthodes d'extension dans les différents espaces de noms LINQ, tels que <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="1010a-110">The LINQ technologies make extensive use of deferred execution in both the members of core <xref:System.Linq?displayProperty=nameWithType> classes and in the extension methods in the various LINQ namespaces, such as <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span></span>

<span data-ttu-id="1010a-111">L’exécution différée est prise en charge directement dans le langage C# par le mot clé [yield (référence C#)](../../csharp/language-reference/keywords/yield.md) (sous la forme de l' `yield-return` instruction) lorsqu’elle est utilisée dans un bloc itérateur.</span><span class="sxs-lookup"><span data-stu-id="1010a-111">Deferred execution is supported directly in the C# language by the [yield (C# Reference)](../../csharp/language-reference/keywords/yield.md) keyword (in the form of the `yield-return` statement) when used within an iterator block.</span></span> <span data-ttu-id="1010a-112">Un tel itérateur doit renvoyer une collection de type <xref:System.Collections.IEnumerator> ou <xref:System.Collections.Generic.IEnumerator%601> (ou un type dérivé).</span><span class="sxs-lookup"><span data-stu-id="1010a-112">Such an iterator must return a collection of type <xref:System.Collections.IEnumerator> or <xref:System.Collections.Generic.IEnumerator%601> (or a derived type).</span></span>

## <a name="eager-vs-lazy-evaluation"></a><span data-ttu-id="1010a-113">Évaluation des versions hâtif et tardive</span><span class="sxs-lookup"><span data-stu-id="1010a-113">Eager vs. lazy evaluation</span></span>

<span data-ttu-id="1010a-114">Lorsque vous écrivez une méthode qui implémente l'exécution différée, vous devez également décider s'il faut implémenter la méthode à l'aide de l'évaluation différée ou de l'évaluation stricte.</span><span class="sxs-lookup"><span data-stu-id="1010a-114">When you write a method that implements deferred execution, you also have to decide whether to implement the method using lazy evaluation or eager evaluation.</span></span>

- <span data-ttu-id="1010a-115">Avec l’*évaluation différée*, un seul élément de la collection source est traité pendant chaque appel à l’itérateur.</span><span class="sxs-lookup"><span data-stu-id="1010a-115">In *lazy evaluation*, a single element of the source collection is processed during each call to the iterator.</span></span> <span data-ttu-id="1010a-116">Il s'agit du mode d'implémentation par défaut des itérateurs.</span><span class="sxs-lookup"><span data-stu-id="1010a-116">This is the typical way in which iterators are implemented.</span></span>
- <span data-ttu-id="1010a-117">Avec l’*évaluation stricte*, le premier appel à l’itérateur entraîne le traitement de l’ensemble de la collection.</span><span class="sxs-lookup"><span data-stu-id="1010a-117">In *eager evaluation*, the first call to the iterator will result in the entire collection being processed.</span></span> <span data-ttu-id="1010a-118">Une copie temporaire de la collection source peut également être nécessaire.</span><span class="sxs-lookup"><span data-stu-id="1010a-118">A temporary copy of the source collection might also be required.</span></span> <span data-ttu-id="1010a-119">Par exemple, la méthode <xref:System.Linq.Enumerable.OrderBy%2A> doit trier l'ensemble de la collection avant de renvoyer le premier élément.</span><span class="sxs-lookup"><span data-stu-id="1010a-119">For example, the <xref:System.Linq.Enumerable.OrderBy%2A> method has to sort the entire collection before it returns the first element.</span></span>

<span data-ttu-id="1010a-120">L'évaluation différée procure en général de meilleures performances car elle répartit le traitement de surcharge de manière égale durant l'évaluation de la collection et limite l'utilisation des données temporaires.</span><span class="sxs-lookup"><span data-stu-id="1010a-120">Lazy evaluation usually yields better performance because it distributes overhead processing evenly throughout the evaluation of the collection and minimizes the use of temporary data.</span></span> <span data-ttu-id="1010a-121">Bien entendu, pour certaines opérations il est impossible d'éviter la matérialisation de résultats intermédiaires.</span><span class="sxs-lookup"><span data-stu-id="1010a-121">Of course, for some operations, there is no other option than to materialize intermediate results.</span></span>

<span data-ttu-id="1010a-122">Pour obtenir un exemple de programmation de l’exécution différée en C# et Visual Basic, consultez [exemple d’exécution différée](deferred-execution-example.md) .</span><span class="sxs-lookup"><span data-stu-id="1010a-122">See [Deferred execution example](deferred-execution-example.md) for an example of programming deferred execution in C# and Visual Basic.</span></span>
