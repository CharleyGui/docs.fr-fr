# <a name="exceptions-in-managed-threads"></a><span data-ttu-id="7ab6f-104">Exceptions dans les threads managés</span><span class="sxs-lookup"><span data-stu-id="7ab6f-104">Exceptions in managed threads</span></span>

<span data-ttu-id="7ab6f-105">Le common language runtime permet à la plupart des exceptions non gérées dans les threads de continuer naturellement.</span><span class="sxs-lookup"><span data-stu-id="7ab6f-105">The common language runtime allows most unhandled exceptions in threads to proceed naturally.</span></span> <span data-ttu-id="7ab6f-106">Dans la plupart des cas, cela signifie que l’exception non prise en charge provoque l’arrêt de l’application.</span><span class="sxs-lookup"><span data-stu-id="7ab6f-106">In most cases this means that the unhandled exception causes the application to terminate.</span></span>
  
<span data-ttu-id="7ab6f-107">Le common language runtime représente une protection pour certaines exceptions non prises en charge qui sont utilisées pour contrôler le flux du programme :</span><span class="sxs-lookup"><span data-stu-id="7ab6f-107">The common language runtime provides a backstop for certain unhandled exceptions that are used for controlling program flow:</span></span>
