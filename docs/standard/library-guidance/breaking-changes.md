## <a name="types-of-breaking-changes"></a><span data-ttu-id="121c2-122">Types de changements cassants</span><span class="sxs-lookup"><span data-stu-id="121c2-122">Types of breaking changes</span></span>

<span data-ttu-id="121c2-123">Les changements cassants sont répartis en différentes catégories et n’ont pas tous le même impact.</span><span class="sxs-lookup"><span data-stu-id="121c2-123">Breaking changes fall into different categories and aren't equally impactful.</span></span>

### <a name="source-breaking-change"></a><span data-ttu-id="121c2-124">Changement cassant source</span><span class="sxs-lookup"><span data-stu-id="121c2-124">Source breaking change</span></span>

<span data-ttu-id="121c2-125">Un changement cassant source n’affecte pas l’exécution du programme, mais provoque des erreurs de compilation la prochaine fois que l’application est recompilée.</span><span class="sxs-lookup"><span data-stu-id="121c2-125">A source breaking change doesn't affect program execution but will cause compilation errors the next time the application is recompiled.</span></span> <span data-ttu-id="121c2-126">Par exemple, une nouvelle surcharge peut créer une ambiguïté dans les appels de méthode qui étaient précédemment non équivoques, ou un paramètre renommé peut casser des appelants qui utilisent des paramètres nommés.</span><span class="sxs-lookup"><span data-stu-id="121c2-126">For example, a new overload can create an ambiguity in method calls that were unambiguous previously, or a renamed parameter will break callers that use named parameters.</span></span>

```csharp
public class Task
{
    // Adding a type called Task could conflict with System.Threading.Tasks.Task at compilation
}
```

<span data-ttu-id="121c2-127">Comme un changement cassant source est uniquement nuisible quand les développeurs recompilent leurs applications, il s’agit de celui qui entraîne le moins de perturbations.</span><span class="sxs-lookup"><span data-stu-id="121c2-127">Because a source breaking change is only harmful when developers recompile their applications, it's the least disruptive breaking change.</span></span> <span data-ttu-id="121c2-128">Les développeurs peuvent facilement résoudre leur propre code source rompu.</span><span class="sxs-lookup"><span data-stu-id="121c2-128">Developers can fix their own broken source code easily.</span></span>

### <a name="behavior-breaking-change"></a><span data-ttu-id="121c2-129">Changement cassant de comportement</span><span class="sxs-lookup"><span data-stu-id="121c2-129">Behavior breaking change</span></span>

<span data-ttu-id="121c2-130">Les changements de comportement sont le type le plus courant de changement cassant : presque n’importe quel changement de comportement peut impacter une personne.</span><span class="sxs-lookup"><span data-stu-id="121c2-130">Behavior changes are the most common type of breaking change: almost any change in behavior could break someone.</span></span> <span data-ttu-id="121c2-131">Les changements apportés à votre bibliothèque, comme les signatures de méthode, les exceptions levées ou les formats de données d’entrée ou de sortie, peuvent tous avoir un impact négatif sur les consommateurs de celle-ci.</span><span class="sxs-lookup"><span data-stu-id="121c2-131">Changes to your library, such as method signatures, exceptions thrown or input or output data formats, could all negatively impact your library consumers.</span></span> <span data-ttu-id="121c2-132">Même un correctif de bogue peut être considéré comme un changement cassant si les utilisateurs s’appuyaient sur le comportement précédemment rompu.</span><span class="sxs-lookup"><span data-stu-id="121c2-132">Even a bug fix can qualify as a breaking change if users relied on the previously broken behavior.</span></span>

<span data-ttu-id="121c2-133">L’ajout de fonctionnalités et l’amélioration des comportements médiocres sont une bonne chose, mais la mise à niveau peut s’avérer très difficile pour les utilisateurs existants sans les précautions nécessaires.</span><span class="sxs-lookup"><span data-stu-id="121c2-133">Adding features and improving bad behaviors is a good thing, but without care it can make it very hard for existing users to upgrade.</span></span> <span data-ttu-id="121c2-134">Une approche pour aider les développeurs à gérer les changements cassants de comportement consiste à les masquer derrière des paramètres.</span><span class="sxs-lookup"><span data-stu-id="121c2-134">One approach to helping developers deal with behavior breaking changes is to hide them behind settings.</span></span> <span data-ttu-id="121c2-135">Les paramètres permettent aux développeurs d’effectuer la mise à jour vers la dernière version de votre bibliothèque tout en choisissant d’adhérer ou non aux changements cassants.</span><span class="sxs-lookup"><span data-stu-id="121c2-135">Settings let developers update to the latest version of your library while at the same time choosing to opt in or opt out of breaking changes.</span></span> <span data-ttu-id="121c2-136">Cette stratégie permet aux développeurs de rester à jour tout en permettant à leur code de consommation de s’adapter au fil du temps.</span><span class="sxs-lookup"><span data-stu-id="121c2-136">This strategy lets developers stay up to date while letting their consuming code adapt over time.</span></span>

<span data-ttu-id="121c2-137">Par exemple, ASP.NET Core MVC présente le concept d’une [version de compatibilité](/aspnet/core/mvc/compatibility-version) qui modifie les fonctionnalités activées et désactivées sur `MvcOptions`.</span><span class="sxs-lookup"><span data-stu-id="121c2-137">For example, ASP.NET Core MVC has the concept of a [compatibility version](/aspnet/core/mvc/compatibility-version) that modifies the features enabled and disabled on `MvcOptions`.</span></span>

<span data-ttu-id="121c2-138">**✔️ À ENVISAGER** : Omettre de nouvelles fonctionnalités par défaut si elles affectent les utilisateurs existants et permettre aux développeurs d’adhérer à la fonctionnalité avec un paramètre.</span><span class="sxs-lookup"><span data-stu-id="121c2-138">**✔️ CONSIDER** leaving new features off by default, if they affect existing users, and let developers opt in to the feature with a setting.</span></span>

### <a name="binary-breaking-change"></a><span data-ttu-id="121c2-139">Changement cassant binaire</span><span class="sxs-lookup"><span data-stu-id="121c2-139">Binary breaking change</span></span>

<span data-ttu-id="121c2-140">Un changement cassant binaire se produit quand vous modifiez l’API publique de votre bibliothèque ; les assemblys compilés avec des versions antérieures de votre bibliothèque ne sont plus en mesure d’appeler l’API.</span><span class="sxs-lookup"><span data-stu-id="121c2-140">A binary breaking change happens when you change the public API of your library, so assemblies compiled against older versions of your library are no longer able to call the API.</span></span> <span data-ttu-id="121c2-141">Par exemple, la modification de la signature d’une méthode en ajoutant un nouveau paramètre amène les assemblys compilés avec la version antérieure de la bibliothèque à lever une <xref:System.MissingMethodException>.</span><span class="sxs-lookup"><span data-stu-id="121c2-141">For example, changing a method's signature by adding a new parameter will cause assemblies compiled against the older version of the library to throw a <xref:System.MissingMethodException>.</span></span>

<span data-ttu-id="121c2-142">Un changement cassant binaire peut également casser un **assembly entier**.</span><span class="sxs-lookup"><span data-stu-id="121c2-142">A binary breaking change can also break an **entire assembly**.</span></span> <span data-ttu-id="121c2-143">Le renommage d’un assembly avec `AssemblyName` change l’identité de l’assembly, de même que l’ajout, la suppression ou le changement de la clé d’affectation de noms forts de l’assembly.</span><span class="sxs-lookup"><span data-stu-id="121c2-143">Renaming an assembly with `AssemblyName` will change the assembly's identity, as will adding, removing, or changing the assembly's strong naming key.</span></span> <span data-ttu-id="121c2-144">Un changement de l’identité d’un assembly va casser tout le code compilé qui l’utilise.</span><span class="sxs-lookup"><span data-stu-id="121c2-144">A change of an assembly's identity will break all compiled code that uses it.</span></span>

<span data-ttu-id="121c2-145">**❌ ne modifiez pas** un nom d’assembly.</span><span class="sxs-lookup"><span data-stu-id="121c2-145">**❌ DO NOT** change an assembly name.</span></span>

<span data-ttu-id="121c2-146">**❌ n’ajoutez pas, ne** supprimez pas ou ne modifiez pas la clé de nom fort.</span><span class="sxs-lookup"><span data-stu-id="121c2-146">**❌ DO NOT** add, remove, or change the strong naming key.</span></span>

<span data-ttu-id="121c2-147">**✔️ À ENVISAGER** : Utiliser des classes de base abstraites plutôt que des interfaces.</span><span class="sxs-lookup"><span data-stu-id="121c2-147">**✔️ CONSIDER** using abstract base classes instead of interfaces.</span></span>

> <span data-ttu-id="121c2-148">L’ajout de n’importe quel élément à une interface entraîne l’échec des types existants qui l’implémentent.</span><span class="sxs-lookup"><span data-stu-id="121c2-148">Adding anything to an interface will cause existing types that implement it to fail.</span></span> <span data-ttu-id="121c2-149">Une classe de base abstraite vous permet d’ajouter une implémentation virtuelle par défaut.</span><span class="sxs-lookup"><span data-stu-id="121c2-149">An abstract base class allows you to add a default virtual implementation.</span></span>

<span data-ttu-id="121c2-150">**✔️ À ENVISAGER** : Placer le <xref:System.ObsoleteAttribute> sur les types et membres que vous envisagez de supprimer.</span><span class="sxs-lookup"><span data-stu-id="121c2-150">**✔️ CONSIDER** placing the <xref:System.ObsoleteAttribute> on types and members that you intend to remove.</span></span> <span data-ttu-id="121c2-151">L’attribut doit comporter des instructions pour la mise à jour du code afin de ne plus utiliser l’API obsolète.</span><span class="sxs-lookup"><span data-stu-id="121c2-151">The attribute should have instructions for updating code to no longer use the obsolete API.</span></span>

> <span data-ttu-id="121c2-152">Le code qui appelle des types et méthodes avec le <xref:System.ObsoleteAttribute> génère un avertissement de génération avec le message fourni à l’attribut.</span><span class="sxs-lookup"><span data-stu-id="121c2-152">Code that calls types and methods with the <xref:System.ObsoleteAttribute> will generate a build warning with the message supplied to the attribute.</span></span> <span data-ttu-id="121c2-153">Les avertissements donnent aux personnes qui utilisent la surface d’API obsolète le temps de migrer afin que la plupart ne l’utilisent plus quand elle est supprimée.</span><span class="sxs-lookup"><span data-stu-id="121c2-153">The warnings give people who use the obsolete API surface time to migrate so that when the obsolete API is removed, most are no longer using it.</span></span>

```csharp
public class Document
{
    [Obsolete("LoadDocument(string) is obsolete. Use LoadDocument(Uri) instead.")]
    public static Document LoadDocument(string uri)
    {
        return LoadDocument(new Uri(uri));
    }

    public static Document LoadDocument(Uri uri)
    {
        // Load the document
    }
}
```

<span data-ttu-id="121c2-154">**✔️ À ENVISAGER** : Conserver les types et méthodes avec le <xref:System.ObsoleteAttribute> indéfiniment dans les bibliothèques de bas niveau et de niveau intermédiaire.</span><span class="sxs-lookup"><span data-stu-id="121c2-154">**✔️ CONSIDER** keeping types and methods with the <xref:System.ObsoleteAttribute> indefinitely in low and middle-level libraries.</span></span>

> <span data-ttu-id="121c2-155">La suppression des API est un changement cassant binaire.</span><span class="sxs-lookup"><span data-stu-id="121c2-155">Removing APIs is a binary breaking change.</span></span> <span data-ttu-id="121c2-156">Envisagez de conserver les méthodes et types obsolètes si leur mise à jour présente un faible coût et n’ajoute pas beaucoup de dette technique à votre bibliothèque.</span><span class="sxs-lookup"><span data-stu-id="121c2-156">Considering keeping obsolete types and methods if maintaining them is low cost and doesn't add lot of technical debt to your library.</span></span> <span data-ttu-id="121c2-157">Le fait de ne pas supprimer les types et méthodes peut permettre d’éviter les pires scénarios mentionnés ci-dessus.</span><span class="sxs-lookup"><span data-stu-id="121c2-157">Not removing types and methods can help avoid the worst-case scenarios mentioned above.</span></span>
