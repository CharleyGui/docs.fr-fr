<span data-ttu-id="afb95-103">Vous pouvez optimiser les performances des applications qui utilisent très souvent des expressions régulières en comprenant comment le moteur d’expression régulière compile les expressions et comment les expressions régulières sont mises en cache.</span><span class="sxs-lookup"><span data-stu-id="afb95-103">You can optimize the performance of applications that make extensive use of regular expressions by understanding how the regular expression engine compiles expressions and by understanding how regular expressions are cached.</span></span> <span data-ttu-id="afb95-104">Cette rubrique décrit la compilation et la mise en cache.</span><span class="sxs-lookup"><span data-stu-id="afb95-104">This topic discusses both compilation and caching.</span></span>  
  
## <a name="compiled-regular-expressions"></a><span data-ttu-id="afb95-105">Expressions régulières compilées</span><span class="sxs-lookup"><span data-stu-id="afb95-105">Compiled Regular Expressions</span></span>  
 <span data-ttu-id="afb95-106">Par défaut, le moteur d’expression régulière compile une expression régulière en une séquence d’instructions internes (il s’agit des codes généraux différents de MSIL, ou Microsoft Intermediate Language).</span><span class="sxs-lookup"><span data-stu-id="afb95-106">By default, the regular expression engine compiles a regular expression to a sequence of internal instructions (these are high-level codes that are different from Microsoft intermediate language, or MSIL).</span></span> <span data-ttu-id="afb95-107">Quand le moteur exécute une expression régulière, il interprète les codes internes.</span><span class="sxs-lookup"><span data-stu-id="afb95-107">When the engine executes a regular expression, it interprets the internal codes.</span></span>  
  
 <span data-ttu-id="afb95-108">Si un objet <xref:System.Text.RegularExpressions.Regex> est construit avec l’option <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType>, il compile l’expression régulière en code MSIL explicite plutôt qu’en instructions internes d’expression régulière générale.</span><span class="sxs-lookup"><span data-stu-id="afb95-108">If a <xref:System.Text.RegularExpressions.Regex> object is constructed with the <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option, it compiles the regular expression to explicit MSIL code instead of high-level regular expression internal instructions.</span></span> <span data-ttu-id="afb95-109">Ainsi, le compilateur juste-à-temps (JIT) de .NET convertit l’expression en code machine natif pour de meilleures performances.</span><span class="sxs-lookup"><span data-stu-id="afb95-109">This allows .NET's just-in-time (JIT) compiler to convert the expression to native machine code for higher performance.</span></span>  <span data-ttu-id="afb95-110">Le coût de <xref:System.Text.RegularExpressions.Regex> construction de l’objet peut être plus élevé, mais le coût d’exécution des allumettes avec elle est susceptible d’être beaucoup plus faible.</span><span class="sxs-lookup"><span data-stu-id="afb95-110">The cost of constructing the <xref:System.Text.RegularExpressions.Regex> object may be higher, but the cost of performing matches with it is likely to be much smaller.</span></span>

 <span data-ttu-id="afb95-111">Une alternative serait d’utiliser des expressions régulières précompilées.</span><span class="sxs-lookup"><span data-stu-id="afb95-111">An alternative is to use precompiled regular expressions.</span></span> <span data-ttu-id="afb95-112">Vous pouvez compiler l’ensemble de vos expressions dans une DLL réutilisable à l’aide de la méthode <xref:System.Text.RegularExpressions.Regex.CompileToAssembly%2A>.</span><span class="sxs-lookup"><span data-stu-id="afb95-112">You can compile all of your expressions into a reusable DLL by using the <xref:System.Text.RegularExpressions.Regex.CompileToAssembly%2A> method.</span></span> <span data-ttu-id="afb95-113">Cela évite d’avoir à compiler au moment de l’exécution tout en continuant à tirer parti de la rapidité des expressions régulières compilées.</span><span class="sxs-lookup"><span data-stu-id="afb95-113">This avoids the need to compile at runtime while still benefiting from the speed of compiled regular expressions.</span></span>  
  
## <a name="the-regular-expressions-cache"></a><span data-ttu-id="afb95-114">Cache des expressions régulières</span><span class="sxs-lookup"><span data-stu-id="afb95-114">The Regular Expressions Cache</span></span>  
 <span data-ttu-id="afb95-115">Pour améliorer les performances, le moteur d’expression régulière gère un cache à l’échelle de l’application des expressions régulières compilées.</span><span class="sxs-lookup"><span data-stu-id="afb95-115">To improve performance, the regular expression engine maintains an application-wide cache of compiled regular expressions.</span></span> <span data-ttu-id="afb95-116">Ce cache stocke les modèles d’expression régulière utilisés uniquement dans les appels de méthode statique.</span><span class="sxs-lookup"><span data-stu-id="afb95-116">The cache stores regular expression patterns that are used only in static method calls.</span></span> <span data-ttu-id="afb95-117">(Les modèles d’expression réguliers fournis aux méthodes d’instance ne sont pas mis en cache.) Cela évite la nécessité de réparer une expression dans le code byte de haut niveau chaque fois qu’il est utilisé.</span><span class="sxs-lookup"><span data-stu-id="afb95-117">(Regular expression patterns supplied to instance methods are not cached.) This avoids the need to reparse an expression into high-level byte code each time it is used.</span></span>  
  
 <span data-ttu-id="afb95-118">Le nombre maximal d’expressions régulières mises en cache est déterminé par la valeur de la propriété <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType>`static` (`Shared` en Visual Basic).</span><span class="sxs-lookup"><span data-stu-id="afb95-118">The maximum number of cached regular expressions is determined by the value of the `static` (`Shared` in Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="afb95-119">Par défaut, le moteur d’expression régulière met en cache jusqu’à 15 expressions régulières compilées.</span><span class="sxs-lookup"><span data-stu-id="afb95-119">By default, the regular expression engine caches up to 15 compiled regular expressions.</span></span> <span data-ttu-id="afb95-120">Si le nombre d’expressions régulières compilées dépasse la taille du cache, l’expression régulière la plus anciennement utilisée est ignorée et la nouvelle expression régulière est mise en cache.</span><span class="sxs-lookup"><span data-stu-id="afb95-120">If the number of compiled regular expressions exceeds the cache size, the least recently used regular expression is discarded and the new regular expression is cached.</span></span>  
  
 <span data-ttu-id="afb95-121">Votre application peut réutiliser des expressions régulières de l’une des deux façons suivantes :</span><span class="sxs-lookup"><span data-stu-id="afb95-121">Your application can reuse regular expressions in one of the following two ways:</span></span>  
  
- <span data-ttu-id="afb95-122">En utilisant une méthode statique de l’objet <xref:System.Text.RegularExpressions.Regex> pour définir l’expression régulière.</span><span class="sxs-lookup"><span data-stu-id="afb95-122">By using a static method of the <xref:System.Text.RegularExpressions.Regex> object to define the regular expression.</span></span> <span data-ttu-id="afb95-123">Si vous utilisez un modèle d’expression régulier qui a déjà été défini par un autre appel de méthode statique, le moteur d’expression régulière va essayer de le récupérer à partir du cache.</span><span class="sxs-lookup"><span data-stu-id="afb95-123">If you're using a regular expression pattern that has already been defined by another static method call, the regular expression engine will try to retrieve it from the cache.</span></span> <span data-ttu-id="afb95-124">S’il n’est pas disponible dans le cache, le moteur compilera l’expression régulière et l’ajoutera au cache.</span><span class="sxs-lookup"><span data-stu-id="afb95-124">If it's not available in the cache, the engine will compile the regular expression and add it to the cache.</span></span>
  
- <span data-ttu-id="afb95-125">En réutilisant un objet <xref:System.Text.RegularExpressions.Regex> existant tant que son modèle d’expression régulière est nécessaire.</span><span class="sxs-lookup"><span data-stu-id="afb95-125">By reusing an existing <xref:System.Text.RegularExpressions.Regex> object as long as its regular expression pattern is needed.</span></span>  
  
 <span data-ttu-id="afb95-126">En raison de la surcharge liée à l’instanciation d’objets et à la compilation d’expressions régulières, la création et la destruction rapide d’un grand nombre d’objets <xref:System.Text.RegularExpressions.Regex> est un processus très coûteux.</span><span class="sxs-lookup"><span data-stu-id="afb95-126">Because of the overhead of object instantiation and regular expression compilation, creating and rapidly destroying numerous <xref:System.Text.RegularExpressions.Regex> objects is a very expensive process.</span></span> <span data-ttu-id="afb95-127">Pour les applications qui utilisent un grand nombre d’expressions régulières différentes, vous pouvez optimiser les performances en utilisant des appels de méthodes `Regex` statiques et en augmentant éventuellement la taille du cache des expressions régulières.</span><span class="sxs-lookup"><span data-stu-id="afb95-127">For applications that use a large number of different regular expressions, you can optimize performance by using calls to static `Regex` methods and possibly by increasing the size of the regular expression cache.</span></span>
