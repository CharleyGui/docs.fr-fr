# <a name="compilation-and-reuse-in-regular-expressions"></a><span data-ttu-id="868a8-103">Compilation et réutilisation dans les expressions régulières</span><span class="sxs-lookup"><span data-stu-id="868a8-103">Compilation and Reuse in Regular Expressions</span></span>

<span data-ttu-id="868a8-104">Vous pouvez optimiser les performances des applications qui utilisent très souvent des expressions régulières en comprenant comment le moteur d’expression régulière compile les expressions et comment les expressions régulières sont mises en cache.</span><span class="sxs-lookup"><span data-stu-id="868a8-104">You can optimize the performance of applications that make extensive use of regular expressions by understanding how the regular expression engine compiles expressions and by understanding how regular expressions are cached.</span></span> <span data-ttu-id="868a8-105">Cette rubrique décrit la compilation et la mise en cache.</span><span class="sxs-lookup"><span data-stu-id="868a8-105">This topic discusses both compilation and caching.</span></span>  
  
## <a name="compiled-regular-expressions"></a><span data-ttu-id="868a8-106">Expressions régulières compilées</span><span class="sxs-lookup"><span data-stu-id="868a8-106">Compiled Regular Expressions</span></span>  

 <span data-ttu-id="868a8-107">Par défaut, le moteur d’expression régulière compile une expression régulière en une séquence d’instructions internes (il s’agit des codes généraux différents de MSIL, ou Microsoft Intermediate Language).</span><span class="sxs-lookup"><span data-stu-id="868a8-107">By default, the regular expression engine compiles a regular expression to a sequence of internal instructions (these are high-level codes that are different from Microsoft intermediate language, or MSIL).</span></span> <span data-ttu-id="868a8-108">Quand le moteur exécute une expression régulière, il interprète les codes internes.</span><span class="sxs-lookup"><span data-stu-id="868a8-108">When the engine executes a regular expression, it interprets the internal codes.</span></span>  
  
 <span data-ttu-id="868a8-109">Si un objet <xref:System.Text.RegularExpressions.Regex> est construit avec l’option <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType>, il compile l’expression régulière en code MSIL explicite plutôt qu’en instructions internes d’expression régulière générale.</span><span class="sxs-lookup"><span data-stu-id="868a8-109">If a <xref:System.Text.RegularExpressions.Regex> object is constructed with the <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option, it compiles the regular expression to explicit MSIL code instead of high-level regular expression internal instructions.</span></span> <span data-ttu-id="868a8-110">Ainsi, le compilateur juste-à-temps (JIT) de .NET convertit l’expression en code machine natif pour de meilleures performances.</span><span class="sxs-lookup"><span data-stu-id="868a8-110">This allows .NET's just-in-time (JIT) compiler to convert the expression to native machine code for higher performance.</span></span>  <span data-ttu-id="868a8-111">Le coût de la construction de l' <xref:System.Text.RegularExpressions.Regex> objet peut être plus élevé, mais le coût lié à l’exécution de correspondances est probablement plus petit.</span><span class="sxs-lookup"><span data-stu-id="868a8-111">The cost of constructing the <xref:System.Text.RegularExpressions.Regex> object may be higher, but the cost of performing matches with it is likely to be much smaller.</span></span>

 <span data-ttu-id="868a8-112">Une alternative serait d’utiliser des expressions régulières précompilées.</span><span class="sxs-lookup"><span data-stu-id="868a8-112">An alternative is to use precompiled regular expressions.</span></span> <span data-ttu-id="868a8-113">Vous pouvez compiler l’ensemble de vos expressions dans une DLL réutilisable à l’aide de la méthode <xref:System.Text.RegularExpressions.Regex.CompileToAssembly%2A>.</span><span class="sxs-lookup"><span data-stu-id="868a8-113">You can compile all of your expressions into a reusable DLL by using the <xref:System.Text.RegularExpressions.Regex.CompileToAssembly%2A> method.</span></span> <span data-ttu-id="868a8-114">Cela évite de devoir compiler au moment de l’exécution tout en bénéficiant de la vitesse des expressions régulières compilées.</span><span class="sxs-lookup"><span data-stu-id="868a8-114">This avoids the need to compile at run time while still benefiting from the speed of compiled regular expressions.</span></span>  
  
## <a name="the-regular-expressions-cache"></a><span data-ttu-id="868a8-115">Cache des expressions régulières</span><span class="sxs-lookup"><span data-stu-id="868a8-115">The Regular Expressions Cache</span></span>  

 <span data-ttu-id="868a8-116">Pour améliorer les performances, le moteur d’expression régulière gère un cache à l’échelle de l’application des expressions régulières compilées.</span><span class="sxs-lookup"><span data-stu-id="868a8-116">To improve performance, the regular expression engine maintains an application-wide cache of compiled regular expressions.</span></span> <span data-ttu-id="868a8-117">Ce cache stocke les modèles d’expression régulière utilisés uniquement dans les appels de méthode statique.</span><span class="sxs-lookup"><span data-stu-id="868a8-117">The cache stores regular expression patterns that are used only in static method calls.</span></span> <span data-ttu-id="868a8-118">(Les modèles d’expression régulière fournis aux méthodes d’instance ne sont pas mis en cache.) Cela évite de devoir réanalyser une expression en code d’octet de haut niveau chaque fois qu’elle est utilisée.</span><span class="sxs-lookup"><span data-stu-id="868a8-118">(Regular expression patterns supplied to instance methods are not cached.) This avoids the need to reparse an expression into high-level byte code each time it is used.</span></span>  
  
 <span data-ttu-id="868a8-119">Le nombre maximal d’expressions régulières mises en cache est déterminé par la valeur de la propriété <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType>`static` (`Shared` en Visual Basic).</span><span class="sxs-lookup"><span data-stu-id="868a8-119">The maximum number of cached regular expressions is determined by the value of the `static` (`Shared` in Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="868a8-120">Par défaut, le moteur d’expression régulière met en cache jusqu’à 15 expressions régulières compilées.</span><span class="sxs-lookup"><span data-stu-id="868a8-120">By default, the regular expression engine caches up to 15 compiled regular expressions.</span></span> <span data-ttu-id="868a8-121">Si le nombre d’expressions régulières compilées dépasse la taille du cache, l’expression régulière la plus anciennement utilisée est ignorée et la nouvelle expression régulière est mise en cache.</span><span class="sxs-lookup"><span data-stu-id="868a8-121">If the number of compiled regular expressions exceeds the cache size, the least recently used regular expression is discarded and the new regular expression is cached.</span></span>  
  
 <span data-ttu-id="868a8-122">Votre application peut réutiliser des expressions régulières de l’une des deux manières suivantes :</span><span class="sxs-lookup"><span data-stu-id="868a8-122">Your application can reuse regular expressions in one of the following two ways:</span></span>  
  
- <span data-ttu-id="868a8-123">En utilisant une méthode statique de l’objet <xref:System.Text.RegularExpressions.Regex> pour définir l’expression régulière.</span><span class="sxs-lookup"><span data-stu-id="868a8-123">By using a static method of the <xref:System.Text.RegularExpressions.Regex> object to define the regular expression.</span></span> <span data-ttu-id="868a8-124">Si vous utilisez un modèle d’expression régulière qui a déjà été défini par un autre appel de méthode statique, le moteur d’expression régulière essaiera de le récupérer à partir du cache.</span><span class="sxs-lookup"><span data-stu-id="868a8-124">If you're using a regular expression pattern that has already been defined by another static method call, the regular expression engine will try to retrieve it from the cache.</span></span> <span data-ttu-id="868a8-125">S’il n’est pas disponible dans le cache, le moteur compile l’expression régulière et l’ajoute au cache.</span><span class="sxs-lookup"><span data-stu-id="868a8-125">If it's not available in the cache, the engine will compile the regular expression and add it to the cache.</span></span>
  
- <span data-ttu-id="868a8-126">En réutilisant un objet <xref:System.Text.RegularExpressions.Regex> existant tant que son modèle d’expression régulière est nécessaire.</span><span class="sxs-lookup"><span data-stu-id="868a8-126">By reusing an existing <xref:System.Text.RegularExpressions.Regex> object as long as its regular expression pattern is needed.</span></span>  
  
 <span data-ttu-id="868a8-127">En raison de la surcharge liée à l’instanciation d’objets et à la compilation d’expressions régulières, la création et la destruction rapide d’un grand nombre d’objets <xref:System.Text.RegularExpressions.Regex> est un processus très coûteux.</span><span class="sxs-lookup"><span data-stu-id="868a8-127">Because of the overhead of object instantiation and regular expression compilation, creating and rapidly destroying numerous <xref:System.Text.RegularExpressions.Regex> objects is a very expensive process.</span></span> <span data-ttu-id="868a8-128">Pour les applications qui utilisent un grand nombre d’expressions régulières différentes, vous pouvez optimiser les performances en utilisant des appels de méthodes `Regex` statiques et en augmentant éventuellement la taille du cache des expressions régulières.</span><span class="sxs-lookup"><span data-stu-id="868a8-128">For applications that use a large number of different regular expressions, you can optimize performance by using calls to static `Regex` methods and possibly by increasing the size of the regular expression cache.</span></span>
