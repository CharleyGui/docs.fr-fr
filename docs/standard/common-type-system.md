## <a name="common-language-specification"></a><span data-ttu-id="2b3aa-132">CLS (Common Language Specification)</span><span class="sxs-lookup"><span data-stu-id="2b3aa-132">Common Language Specification</span></span>

<span data-ttu-id="2b3aa-133">Pour permettre des scénarios d’interopérabilité complète, tous les objets qui sont créés dans le code doivent s’appuyer sur certains points communs entre les langages qui les consomment (leurs _appelants_).</span><span class="sxs-lookup"><span data-stu-id="2b3aa-133">To enable full interoperability scenarios, all objects that are created in code must rely on some commonality in the languages that are consuming them (are their _callers_).</span></span> <span data-ttu-id="2b3aa-134">Comme il existe de nombreux langages différents, .NET a défini ces points communs dans ce que l’on appelle la **spécification CLS (Common Language Specification)**.</span><span class="sxs-lookup"><span data-stu-id="2b3aa-134">Since there are numerous different languages, .NET has specified those commonalities in something called the **Common Language Specification** (CLS).</span></span> <span data-ttu-id="2b3aa-135">CLS définit un ensemble de fonctionnalités nécessaires pour de nombreuses applications courantes.</span><span class="sxs-lookup"><span data-stu-id="2b3aa-135">CLS defines a set of features that are needed by many common applications.</span></span> <span data-ttu-id="2b3aa-136">Il fournit également une sorte de recette pour n’importe quel langage implémenté sur .NET qui explique ce qu’il doit prendre en charge.</span><span class="sxs-lookup"><span data-stu-id="2b3aa-136">It also provides a sort of recipe for any language that is implemented on top of .NET on what it needs to support.</span></span>

<span data-ttu-id="2b3aa-137">CLS est un sous-ensemble du système CTS.</span><span class="sxs-lookup"><span data-stu-id="2b3aa-137">CLS is a subset of the CTS.</span></span> <span data-ttu-id="2b3aa-138">Cela signifie que toutes les règles du CTS s’appliquent également à la spécification CLS, sauf si les règles CLS sont plus strictes.</span><span class="sxs-lookup"><span data-stu-id="2b3aa-138">This means that all of the rules in the CTS also apply to the CLS, unless the CLS rules are more strict.</span></span> <span data-ttu-id="2b3aa-139">Si un composant est créé uniquement à l’aide des règles CLS, autrement dit, s’il expose uniquement les fonctionnalités CLS dans son API, il est considéré comme étant **conforme CLS**.</span><span class="sxs-lookup"><span data-stu-id="2b3aa-139">If a component is built using only the rules in the CLS, that is, it exposes only the CLS features in its API, it is said to be **CLS-compliant**.</span></span> <span data-ttu-id="2b3aa-140">Par exemple, les `<framework-librares>` sont conformes CLS justement parce qu’ils ont besoin d’utiliser tous les langages pris en charge sur .NET.</span><span class="sxs-lookup"><span data-stu-id="2b3aa-140">For instance, the `<framework-librares>` are CLS-compliant precisely because they need to work across all of the languages that are supported on .NET.</span></span>
