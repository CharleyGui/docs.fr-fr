Les développeurs qui travaillent sur Windows peuvent créer des images pour les conteneurs Linux ou Windows.</span><span class="sxs-lookup"><span data-stu-id="0902c-115">(Developers working on the Mac can edit code or run the Docker command-line interface (CLI) from macOS, but as of this writing, containers don't run directly on macOS.) <span data-ttu-id="0902c-119">Les [conteneurs Windows](/virtualization/windowscontainers/about/) fonctionnent avec deux types de runtime :</span><span class="sxs-lookup"><span data-stu-id="0902c-119">To run [Windows Containers](/virtualization/windowscontainers/about/), there are two types of runtimes:</span></span>

- <span data-ttu-id="0902c-122">Les **conteneurs Hyper-V** étendent l’isolation fournie par les conteneurs Windows Server en exécutant chaque conteneur sur une machine virtuelle hautement optimisée.</span><span class="sxs-lookup"><span data-stu-id="0902c-122">**Hyper-V Containers** expand on the isolation provided by Windows Server Containers by running each container in a highly optimized virtual machine.</span></span> <span data-ttu-id="0902c-123">Dans cette configuration, le noyau de l’hôte conteneur n’est pas partagé avec les conteneurs Hyper-V, ce qui garantit une meilleure isolation.</span><span class="sxs-lookup"><span data-stu-id="0902c-123">In this configuration, the kernel of the container host isn't shared with the Hyper-V Containers, providing better isolation.</span></span>

<span data-ttu-id="0902c-124">Les images de ces deux types de conteneurs sont créées et fonctionnent exactement de la même façon.</span><span class="sxs-lookup"><span data-stu-id="0902c-124">The images for these containers are created and work just the same way.</span></span> <span data-ttu-id="0902c-125">La différence réside dans la façon dont le conteneur est créé à partir de l’image, l’exécution d’un conteneur Hyper-V nécessitant un paramètre supplémentaire.</span><span class="sxs-lookup"><span data-stu-id="0902c-125">The difference is in how the container is created from the image—running a Hyper-V Container requires an extra parameter.</span></span> <span data-ttu-id="0902c-126">Pour plus d’informations, consultez [Conteneurs Hyper-V](/virtualization/windowscontainers/manage-containers/hyperv-container).</span><span class="sxs-lookup"><span data-stu-id="0902c-126">For details, see [Hyper-V Containers](/virtualization/windowscontainers/manage-containers/hyperv-container).</span></span>

<span data-ttu-id="0902c-130">**Figure 1-3**.</span><span class="sxs-lookup"><span data-stu-id="0902c-130">**Figure 1-3**.</span></span> <span data-ttu-id="0902c-131">Comparaison entre les machines virtuelles traditionnelles et les conteneurs Docker</span><span class="sxs-lookup"><span data-stu-id="0902c-131">Comparison of traditional virtual machines to Docker containers</span></span>

<span data-ttu-id="0902c-132">Comme indiqué dans le diagramme ci-dessus, pour les machines virtuelles, il existe trois couches de base dans le serveur hôte.</span><span class="sxs-lookup"><span data-stu-id="0902c-132">As shown in the above diagram, for VMs, there are three base layers in the host server.</span></span> <span data-ttu-id="0902c-133">Du bas vers le haut : infrastructure, système d’exploitation hôte et hyperviseur.</span><span class="sxs-lookup"><span data-stu-id="0902c-133">From the bottom-up: Infrastructure, Host Operating System, and a Hypervisor.</span></span> <span data-ttu-id="0902c-134">Au-dessus de tout cela, chaque machine virtuelle possède son propre système d’exploitation et toutes les bibliothèques nécessaires.</span><span class="sxs-lookup"><span data-stu-id="0902c-134">On top of all that, each VM has its own OS and all necessary libraries.</span></span> <span data-ttu-id="0902c-135">D’un autre côté, pour l’arrimeur, le serveur hôte dispose uniquement de l’infrastructure et du système d’exploitation.</span><span class="sxs-lookup"><span data-stu-id="0902c-135">On the other hand, for Docker, the host server only has the Infrastructure and the OS.</span></span> <span data-ttu-id="0902c-136">En plus de cela, le moteur de conteneur conserve les conteneurs isolés, mais les autorise à partager les services d’un système d’exploitation de base unique.</span><span class="sxs-lookup"><span data-stu-id="0902c-136">On top of that, the container engine keeps containers isolated, but lets them share the single base OS's services.</span></span>

<span data-ttu-id="0902c-137">Du fait que les conteneurs nécessitent beaucoup moins de ressources (par exemple, ils n’ont pas besoin d’un système d’exploitation complet), leur démarrage est rapide et leur déploiement est simple.</span><span class="sxs-lookup"><span data-stu-id="0902c-137">Because containers require far fewer resources (for example, they don't need a full OS), they're easy to deploy and they start fast.</span></span> <span data-ttu-id="0902c-138">Cela vous permet d’avoir une densité plus élevée, ce qui vous permet d’exécuter plus de services sur la même unité matérielle, limitant les coûts.</span><span class="sxs-lookup"><span data-stu-id="0902c-138">This allows you to have higher density, meaning that it allows you to run more services on the same hardware unit, thereby reducing costs.</span></span>

<span data-ttu-id="0902c-139">Comme effet secondaire de l’exécution sur le même noyau, vous obtenez une moins bonne isolation que sur les machines virtuelles.</span><span class="sxs-lookup"><span data-stu-id="0902c-139">As a side effect of running on the same kernel, you get less isolation than VMs.</span></span>

<span data-ttu-id="0902c-140">L’objectif principal d’une image est de garantir un environnement (dépendances) identique entre les différents déploiements.</span><span class="sxs-lookup"><span data-stu-id="0902c-140">The main goal of an image is to ensure the same environment (dependencies) across different deployments.</span></span> <span data-ttu-id="0902c-141">Vous pouvez ainsi la déboguer sur votre machine et la déployer sur une autre machine, en étant sûr de garder le même environnement.</span><span class="sxs-lookup"><span data-stu-id="0902c-141">This means that you can debug it on your machine and then deploy it to another machine, the same environment guaranteed.</span></span>

<span data-ttu-id="0902c-142">Une image conteneur est un moyen d’empaqueter une application ou un service et de les déployer ensuite d’une façon fiable et reproductible.</span><span class="sxs-lookup"><span data-stu-id="0902c-142">A container image is a way to package an app or service and deploy it in a reliable and reproducible way.</span></span> <span data-ttu-id="0902c-143">Plus qu’une simple technologie, Docker peut également être vu comme une philosophie et un processus.</span><span class="sxs-lookup"><span data-stu-id="0902c-143">You could say that Docker isn't only a technology but also a philosophy and a process.</span></span>

<span data-ttu-id="0902c-144">Les développeurs qui utilisent Docker ne disent pas : « Cela fonctionne sur ma machine, alors pourquoi pas en production ? »</span><span class="sxs-lookup"><span data-stu-id="0902c-144">When using Docker, you won't hear developers say, "It works on my machine, why not in production?"</span></span> <span data-ttu-id="0902c-145">Ils peuvent simplement dire « Cela s’exécute sur Docker », car l’application Docker empaquetée peut être exécutée sur n’importe quel environnement Docker pris en charge et elle s’exécute comme prévu sur toutes les cibles de déploiement (par exemple développement, assurance qualité, préproduction et production).</span><span class="sxs-lookup"><span data-stu-id="0902c-145">They can just say, "It runs on Docker", because the packaged Docker application can be executed on any supported Docker environment, and it runs the way it was intended to on all deployment targets (such as Dev, QA, staging, and production).</span></span>

## <a name="a-simple-analogy"></a><span data-ttu-id="0902c-146">Une simple analogie</span><span class="sxs-lookup"><span data-stu-id="0902c-146">A simple analogy</span></span>

<span data-ttu-id="0902c-147">Peut-être qu’une analogie simple peut aider à bien comprendre le concept de base de Docker.</span><span class="sxs-lookup"><span data-stu-id="0902c-147">Perhaps a simple analogy can help getting the grasp of the core concept of Docker.</span></span>

<span data-ttu-id="0902c-148">Retournons dans les années 1950 un instant.</span><span class="sxs-lookup"><span data-stu-id="0902c-148">Let's go back in time to the 1950s for a moment.</span></span> <span data-ttu-id="0902c-149">Le traitement de texte n’existait pas et les photocopieurs étaient utilisés partout (enfin, presque).</span><span class="sxs-lookup"><span data-stu-id="0902c-149">There were no word processors, and the photocopiers were used everywhere (well, kind of).</span></span>

<span data-ttu-id="0902c-150">Imaginez que vous devez émettre rapidement des lots de lettres à envoyer aux clients, avec du papier et des enveloppes, et les livrer physiquement à l’adresse de chaque client (l’e-mail n’existait pas à l’époque).</span><span class="sxs-lookup"><span data-stu-id="0902c-150">Imagine you're responsible for quickly issuing batches of letters as required, to mail them to customers, using real paper and envelopes, to be delivered physically to each customer's address (there was no email back then).</span></span>

<span data-ttu-id="0902c-151">À un moment donné, vous réalisez que les lettres sont simplement une composition d’un grand ensemble de paragraphes, qui sont organisés en fonction des besoins, selon l’objectif de la lettre, donc vous concevez un système permettant d’émettre des lettres rapidement, afin d’obtenir une augmentation importante.</span><span class="sxs-lookup"><span data-stu-id="0902c-151">At some point, you realize the letters are just a composition of a large set of paragraphs, which are picked and arranged as needed, according to the purpose of the letter, so you devise a system to issue letters quickly, expecting to get a hefty raise.</span></span>

<span data-ttu-id="0902c-152">Le système est simple :</span><span class="sxs-lookup"><span data-stu-id="0902c-152">The system is simple:</span></span>

1. <span data-ttu-id="0902c-153">Vous commencez avec un jeu de feuilles transparentes contenant chacune un paragraphe.</span><span class="sxs-lookup"><span data-stu-id="0902c-153">You begin with a deck of transparent sheets containing one paragraph each.</span></span>

2. <span data-ttu-id="0902c-154">Pour émettre un ensemble de lettres, vous choisissez les feuilles avec les paragraphes dont vous avez besoin, vous les empilez et les alignez afin de pouvoir les lire correctement.</span><span class="sxs-lookup"><span data-stu-id="0902c-154">To issue a set of letters, you pick the sheets with the paragraphs you need, then you stack and align them so they look and read fine.</span></span>

3. <span data-ttu-id="0902c-155">Enfin, vous placez l’ensemble dans la photocopieuse, puis appuyez sur démarrer pour produire autant de lettres que nécessaire.</span><span class="sxs-lookup"><span data-stu-id="0902c-155">Finally, you place the set in the photocopier and press start to produce as many letters as required.</span></span>

<span data-ttu-id="0902c-156">C’est cela l’idée fondamentale de Docker (en version simplifiée).</span><span class="sxs-lookup"><span data-stu-id="0902c-156">So, simplifying, that's the core idea of Docker.</span></span>

<span data-ttu-id="0902c-157">Dans Docker, chaque couche est un jeu de résultats des modifications qui se produisent dans le système de fichiers après l’exécution d’une commande, par exemple, l’installation d’un programme.</span><span class="sxs-lookup"><span data-stu-id="0902c-157">In Docker, each layer is the resulting set of changes that happen to the filesystem after executing a command, such as, installing a program.</span></span>

<span data-ttu-id="0902c-158">Par conséquent, lorsque vous « regardez » le système de fichiers une fois que la couche a été copiée, vous voyez tous les fichiers, y compris la couche lorsque le programme a été installé.</span><span class="sxs-lookup"><span data-stu-id="0902c-158">So, when you "look" at the filesystem after the layer has been copied, you see all the files, included the layer when the program was installed.</span></span>

<span data-ttu-id="0902c-159">Vous pouvez considérer une image en tant que disque dur en lecture seule auxiliaire prêt à être installé dans un « ordinateur » où le système d’exploitation est déjà installé.</span><span class="sxs-lookup"><span data-stu-id="0902c-159">You can think of an image as an auxiliary read-only hard disk ready to be installed in a "computer" where the operating system is already installed.</span></span>

<span data-ttu-id="0902c-160">De même, vous pouvez considérer un conteneur comme « l’ordinateur » avec le disque dur de l’image installé.</span><span class="sxs-lookup"><span data-stu-id="0902c-160">Similarly, you can think of a container as the "computer" with the image hard disk installed.</span></span> <span data-ttu-id="0902c-161">Le conteneur, tout comme un ordinateur, peut être activé ou désactivé.</span><span class="sxs-lookup"><span data-stu-id="0902c-161">The container, just like a computer, can be powered on or off.</span></span>
