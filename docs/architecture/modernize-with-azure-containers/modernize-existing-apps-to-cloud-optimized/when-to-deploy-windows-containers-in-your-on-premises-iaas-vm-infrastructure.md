# <a name="when-to-deploy-windows-containers-in-your-on-premises-iaas-vm-infrastructure"></a>Quand déployer des conteneurs Windows dans votre infrastructure de machines virtuelles IaaS locale

- Votre organisation n’est peut-être pas prête à migrer vers le Cloud, ou il est possible qu’elle ne soit pas en mesure de se déplacer vers le Cloud pour une raison commerciale. Toutefois, vous pouvez toujours bénéficier des avantages de l’utilisation de conteneurs Windows dans vos propres centres de centres.

- Vous pouvez avoir d’autres artefacts en cours d’utilisation en local, et ce qui peut vous ralentir quand vous essayez de vous déplacer vers le Cloud. Par exemple, les dépendances de sécurité ou d’authentification avec les Active Directory Windows Server locaux, ou toute autre ressource locale.

- Si vous commencez à utiliser des conteneurs Windows aujourd’hui, vous pouvez effectuer une migration échelonnée vers le Cloud demain à partir d’une position bien meilleure. Les conteneurs Windows deviennent une unité de déploiement pour n’importe quel Cloud, sans verrou.
