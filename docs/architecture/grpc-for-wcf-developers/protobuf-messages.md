## <a name="declaring-a-message"></a><span data-ttu-id="232d5-106">Déclaration d’un message</span><span class="sxs-lookup"><span data-stu-id="232d5-106">Declaring a message</span></span>

<span data-ttu-id="232d5-107">Dans WCF, une classe de `Stock` pour une application commerciale boursière peut être définie comme dans l’exemple suivant :</span><span class="sxs-lookup"><span data-stu-id="232d5-107">In WCF, a `Stock` class for a stock market trading application might be defined like the following example:</span></span>

```csharp
namespace TraderSys
{
    [DataContract]
    public class Stock
    {
        [DataMember]
        public int Id { get; set;}
        [DataMember]
        public string Symbol { get; set;}
        [DataMember]
        public string DisplayName { get; set;}
        [DataMember]
        public int MarketId { get; set; }
    }
}
```

<span data-ttu-id="232d5-108">Pour implémenter la classe équivalente dans Protobuf, elle doit être déclarée dans le fichier `.proto`.</span><span class="sxs-lookup"><span data-stu-id="232d5-108">To implement the equivalent class in Protobuf, it must be declared in the `.proto` file.</span></span> <span data-ttu-id="232d5-109">Le compilateur `protoc` générera ensuite la classe .NET dans le cadre du processus de génération.</span><span class="sxs-lookup"><span data-stu-id="232d5-109">The `protoc` compiler will then generate the .NET class as part of the build process.</span></span>

```protobuf
syntax "proto3";

option csharp_namespace = "TraderSys";

message Stock {

    int32 id = 1;
    string symbol = 2;
    string display_name = 3;
    int32 market_id = 4;

}  
```

<span data-ttu-id="232d5-110">La première ligne déclare la version de syntaxe utilisée.</span><span class="sxs-lookup"><span data-stu-id="232d5-110">The first line declares the syntax version being used.</span></span> <span data-ttu-id="232d5-111">La version 3 de la langue a été publiée en 2016 et est la version recommandée pour les services gRPC.</span><span class="sxs-lookup"><span data-stu-id="232d5-111">Version 3 of the language was released in 2016 and is the recommended version for gRPC services.</span></span>

<span data-ttu-id="232d5-112">La ligne `option csharp_namespace` spécifie l’espace de noms à utiliser pour C# les types générés.</span><span class="sxs-lookup"><span data-stu-id="232d5-112">The `option csharp_namespace` line specifies the namespace to be used for the generated C# types.</span></span> <span data-ttu-id="232d5-113">Cette option est ignorée lorsque le fichier `.proto` est compilé pour d’autres langues.</span><span class="sxs-lookup"><span data-stu-id="232d5-113">This option will be ignored when the `.proto` file is compiled for other languages.</span></span> <span data-ttu-id="232d5-114">Il est courant que les fichiers Protobuf contiennent des options spécifiques à la langue pour plusieurs langues.</span><span class="sxs-lookup"><span data-stu-id="232d5-114">It is common for Protobuf files to contain language-specific options for several languages.</span></span>

<span data-ttu-id="232d5-115">La définition de message `Stock` spécifie quatre champs, chacun avec un type, un nom et un numéro de champ.</span><span class="sxs-lookup"><span data-stu-id="232d5-115">The `Stock` message definition specifies four fields, each with a type, a name, and a field number.</span></span>

## <a name="field-numbers"></a><span data-ttu-id="232d5-116">Numéros de champ</span><span class="sxs-lookup"><span data-stu-id="232d5-116">Field numbers</span></span>

<span data-ttu-id="232d5-117">Les numéros de champ constituent une partie importante de Protobuf.</span><span class="sxs-lookup"><span data-stu-id="232d5-117">Field numbers are an important part of Protobuf.</span></span> <span data-ttu-id="232d5-118">Elles sont utilisées pour identifier les champs dans les données binaires codées, ce qui signifie qu’elles ne peuvent pas passer d’une version à une version de votre service.</span><span class="sxs-lookup"><span data-stu-id="232d5-118">They're used to identify fields in the binary encoded data, which means they can't change from version to version of your service.</span></span> <span data-ttu-id="232d5-119">L’avantage est que la compatibilité ascendante et descendante est possible.</span><span class="sxs-lookup"><span data-stu-id="232d5-119">The advantage is that backward and forward compatibility is possible.</span></span> <span data-ttu-id="232d5-120">Les clients et les services ignorent simplement les numéros de champ qu’ils ne connaissent pas, à condition que la possibilité de valeurs manquantes soit gérée.</span><span class="sxs-lookup"><span data-stu-id="232d5-120">Clients and services will simply ignore field numbers they don't know about, as long as the possibility of missing values is handled.</span></span>

<span data-ttu-id="232d5-121">Dans le format binaire, le numéro de champ est associé à un identificateur de type.</span><span class="sxs-lookup"><span data-stu-id="232d5-121">In the binary format, the field number is combined with a type identifier.</span></span> <span data-ttu-id="232d5-122">Les numéros de champ de 1 à 15 peuvent être encodés avec leur type comme un octet unique ; les nombres compris entre 16 et 2047 prennent 2 octets.</span><span class="sxs-lookup"><span data-stu-id="232d5-122">Field numbers from 1 to 15 can be encoded with their type as a single byte; numbers from 16 to 2047 take 2 bytes.</span></span> <span data-ttu-id="232d5-123">Vous pouvez aller plus haut si vous avez besoin de plus de 2047 champs sur un message pour une raison quelconque.</span><span class="sxs-lookup"><span data-stu-id="232d5-123">You can go higher if you need more than 2047 fields on a message for any reason.</span></span> <span data-ttu-id="232d5-124">Les identificateurs à un seul octet pour les numéros de champ 1 à 15 offrent de meilleures performances. vous devez donc les utiliser pour les champs les plus basiques et les plus fréquemment utilisés.</span><span class="sxs-lookup"><span data-stu-id="232d5-124">The single byte identifiers for field numbers 1 to 15 offer better performance, so you should use them for the most basic, frequently used fields.</span></span>
