## <a name="declaring-a-message"></a><span data-ttu-id="75b64-106">Déclaration d’un message</span><span class="sxs-lookup"><span data-stu-id="75b64-106">Declaring a message</span></span>

<span data-ttu-id="75b64-107">Dans WCF, une `Stock` classe pour une application commerciale boursière peut être définie comme dans l’exemple suivant :</span><span class="sxs-lookup"><span data-stu-id="75b64-107">In WCF, a `Stock` class for a stock market trading application might be defined like the following example:</span></span>

```csharp
namespace TraderSys
{
    [DataContract]
    public class Stock
    {
        [DataMember]
        public int Id { get; set;}
        [DataMember]
        public string Symbol { get; set;}
        [DataMember]
        public string DisplayName { get; set;}
        [DataMember]
        public int MarketId { get; set; }
    }
}
```

<span data-ttu-id="75b64-108">Pour implémenter la classe équivalente dans Protobuf, elle doit être déclarée dans le `.proto` fichier.</span><span class="sxs-lookup"><span data-stu-id="75b64-108">To implement the equivalent class in Protobuf, it must be declared in the `.proto` file.</span></span> <span data-ttu-id="75b64-109">Le `protoc` compilateur génère ensuite la classe .net dans le cadre du processus de génération.</span><span class="sxs-lookup"><span data-stu-id="75b64-109">The `protoc` compiler will then generate the .NET class as part of the build process.</span></span>

```protobuf
syntax "proto3";

option csharp_namespace = "TraderSys";

message Stock {

    int32 id = 1;
    string symbol = 2;
    string displayName = 3;
    int32 marketId = 4;

}  
```

<span data-ttu-id="75b64-110">La première ligne déclare la version de syntaxe utilisée.</span><span class="sxs-lookup"><span data-stu-id="75b64-110">The first line declares the syntax version being used.</span></span> <span data-ttu-id="75b64-111">La version 3 de la langue a été publiée en 2016 et est la version recommandée pour les services gRPC.</span><span class="sxs-lookup"><span data-stu-id="75b64-111">Version 3 of the language was released in 2016 and is the recommended version for gRPC services.</span></span>

<span data-ttu-id="75b64-112">La `option csharp_namespace` ligne spécifie l’espace de noms à utiliser pour C# les types générés.</span><span class="sxs-lookup"><span data-stu-id="75b64-112">The `option csharp_namespace` line specifies the namespace to be used for the generated C# types.</span></span> <span data-ttu-id="75b64-113">Cette option est ignorée lorsque le `.proto` fichier est compilé pour d’autres langues.</span><span class="sxs-lookup"><span data-stu-id="75b64-113">This option will be ignored when the `.proto` file is compiled for other languages.</span></span> <span data-ttu-id="75b64-114">Il est courant que les fichiers Protobuf contiennent des options spécifiques à la langue pour plusieurs langues.</span><span class="sxs-lookup"><span data-stu-id="75b64-114">It is common for Protobuf files to contain language-specific options for several languages.</span></span>

<span data-ttu-id="75b64-115">La `Stock` définition de message spécifie quatre champs, chacun avec un type, un nom et un numéro de champ.</span><span class="sxs-lookup"><span data-stu-id="75b64-115">The `Stock` message definition specifies four fields, each with a type, a name, and a field number.</span></span>

## <a name="field-numbers"></a><span data-ttu-id="75b64-116">Numéros de champ</span><span class="sxs-lookup"><span data-stu-id="75b64-116">Field numbers</span></span>

<span data-ttu-id="75b64-117">Les numéros de champ constituent une partie importante de Protobuf.</span><span class="sxs-lookup"><span data-stu-id="75b64-117">Field numbers are an important part of Protobuf.</span></span> <span data-ttu-id="75b64-118">Elles sont utilisées pour identifier les champs dans les données binaires codées, ce qui signifie qu’elles ne peuvent pas passer d’une version à une version de votre service.</span><span class="sxs-lookup"><span data-stu-id="75b64-118">They're used to identify fields in the binary encoded data, which means they can't change from version to version of your service.</span></span> <span data-ttu-id="75b64-119">L’avantage est que la compatibilité ascendante et descendante est possible.</span><span class="sxs-lookup"><span data-stu-id="75b64-119">The advantage is that backward and forward compatibility is possible.</span></span> <span data-ttu-id="75b64-120">Les clients et les services ignorent simplement les numéros de champ qu’ils ne connaissent pas, à condition que la possibilité de valeurs manquantes soit gérée.</span><span class="sxs-lookup"><span data-stu-id="75b64-120">Clients and services will simply ignore field numbers they don't know about, as long as the possibility of missing values is handled.</span></span>
