# <a name="sos-debugging-extension"></a>Extension de débogage SOS

L’extension de débogage SOS vous permet d’afficher des informations sur le code qui s’exécute dans le Runtime .NET Core, à la fois sur les processus en temps réel et les vidages. L’extension est préinstallée avec [dotnet-dump](dotnet-dump.md) et [WinDbg/dbg](/windows-hardware/drivers/debugger/debugger-download-tools), et peut être [téléchargée](dotnet-sos.md) pour être utilisée avec LLDB.  L’extension de débogage SOS est utile pour collecter des informations sur le tas managé, Rechercher des altérations de tas, afficher des types de données internes utilisés par le runtime et afficher des informations sur l’ensemble du code managé s’exécutant au sein du Runtime.

## <a name="syntax"></a>Syntax

### <a name="windows"></a>Windows

`![command] [options]`

### <a name="linux-and-macos"></a>Linux et macOS

`sos [command] [options]`

La plupart des commandes ont des alias ou des raccourcis sous lldb :

`clrstack [options]`

### <a name="windows-debugger"></a>Débogueur Windows

Vous pouvez également utiliser l’extension de débogage SOS en la chargeant dans le [débogueur WinDbg/dbg](/windows-hardware/drivers/debugger/debugger-download-tools) et en exécutant des commandes dans le débogueur Windows.  Les commandes SOS peuvent être utilisées sur des processus en direct ou des vidages.

Pour charger l’extension de débogage SOS dans le débogueur Windows, exécutez la commande suivante dans l’outil :

```console
.loadby sos clr
```

WinDbg.exe et Visual Studio utilisent une version de SOS.dll qui correspond à la version de Mscorwks.dll actuellement utilisée. Par défaut, vous devez utiliser la version de SOS.dll qui correspond à la version actuelle de Mscorwks.dll.

Pour utiliser un fichier dump créé sur un autre ordinateur, vérifiez que le fichier Mscorwks.dll fourni avec cette installation se trouve dans votre chemin d'accès aux symboles, et chargez la version correspondante de SOS.dll.

Pour charger une version spécifique de SOS.dll, entrez la commande suivante dans le débogueur Windows :

```console
.load <full path to sos.dll>
```

### <a name="lldb-debugger"></a>Débogueur LLDB

Par défaut, vous pouvez accéder à toutes les commandes SOS en entrant : `sos [command\_name]` . Toutefois, les commandes courantes ont été affectées d’un alias afin que vous n’ayez pas besoin du `sos` préfixe :

| ------------------------------------- | ---------------------------------------------------------------------------------------------

|    `clru`                             | Affiche un code machine annoté d’une méthode managée.
|    `dso`                              | Affiche tous les objets managés recherchés dans les limites de la pile actuelle.

|    `dumpclass`                        | Affiche des informations sur la `EEClass` structure à l’adresse spécifiée.
|    `dumpdomain`                       | Affiche des informations sur tous les AppDomains et tous les assemblys au sein du domaine spécifié.
|    `dumpheap`                         | Affiche des informations sur le tas récupéré par le garbage collector et des statistiques de collection concernant les objets.
|    `dumpil`                           | Affiche le Microsoft Intermediate Language (MSIL) associé à une méthode managée.
|    `dumplog`                          | Écrit le contenu d'un journal de contrainte en mémoire dans le fichier spécifié.
|    `dumpmd`                           | Affiche des informations sur la `MethodDesc` structure à l’adresse spécifiée.
|    `dumpmodule`                       | Affiche des informations sur le module à l’adresse spécifiée.
|    `dumpmt`                           | Affiche des informations sur la table de méthodes à l’adresse spécifiée.
|    `dumpobj`                          | Affiche des informations sur l’objet à l’adresse spécifiée.
|    `dumpstack`                        | Affiche une trace de la pile native et managée.
|    `eeheap`                           | Affiche des informations sur la mémoire de processus consommée par les structures de données de Runtime internes.
|    `eestack`                          | S’exécute `dumpstack` sur tous les threads du processus.
|    `gcroot`                           | Affiche des informations sur les références (ou racines) à l’objet à l’adresse spécifiée.

|    `loadsymbols`                      | Chargez les symboles du module natif .NET Core.

|    `pe`                               | Affiche et met en forme les champs de tous les objets dérivés de la classe <xref:System.Exception> à l'adresse spécifiée.
